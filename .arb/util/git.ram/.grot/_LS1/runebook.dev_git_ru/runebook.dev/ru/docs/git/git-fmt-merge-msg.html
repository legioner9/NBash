git-fmt-merge-msg

Name

git-fmt-merge-msg — создать сообщение о фиксации слияния

Synopsis

git fmt-merge-msg [-m <message>] [--into-name <branch>] [--log[=<n>] | --no-log]
git fmt-merge-msg [-m <message>] [--log[=<n>] | --no-log] -F <file>

Description

Берет список объединенных объектов на стандартном вводе и создает подходящее сообщение о фиксации, которое будет использоваться для фиксации слияния, которое обычно передается в качестве аргумента <merge-message> для git merge .

Эта команда предназначена в основном для внутреннего использования сценариями, автоматически вызывающими git merge .

Options

--log[=<n>]

Помимо названий ветвей, заполните сообщение журнала однострочными описаниями фактических коммитов, которые объединяются. Будет использоваться не более <n> коммитов от каждого родителя слияния (20, если <n> опущен). Это переопределяет переменную конфигурации merge.log .

--no-log

Не указывайте однострочные описания фактически объединяемых коммитов.

--[no-]summary

Синонимы --log и --no-log; они устарели и будут удалены в будущем.

-m <message>
--message <message>

Используйте <message> вместо имен ветвей для первой строки сообщения журнала. Для использования с --log .

--into-name <branch>

Подготовьте сообщение о слиянии так, как будто происходит слияние с веткой <branch> , вместо имени реальной ветки, с которой производится слияние.

-F <file>
--file <file>

Возьмите список объединенных объектов из <file> вместо стандартного ввода.

Configuration

merge.branchdesc

Помимо имен ветвей, заполните сообщение журнала текстом описания ветки, связанным с ними. По умолчанию ложь.

merge.log

Помимо имен ветвей, заполните сообщение журнала не более чем указанным количеством однострочных описаний из фактических коммитов, которые объединяются. По умолчанию установлено значение false, а true является синонимом 20.

merge.suppressDest

При добавлении glob, который соответствует именам ветвей интеграции, к этой многозначной переменной конфигурации, сообщение слияния по умолчанию, вычисляемое для слияний с этими ветвями интеграции, будет опускать "into <branch name>" в своем заголовке.

Элемент с пустым значением можно использовать для очистки списка glob, накопленного из предыдущих записей конфигурации. Если переменная merge.suppressDest не определена, для обратной совместимости используется значение по умолчанию master .

merge.summary

Синоним merge.log ; это устарело и будет удалено в будущем.

Examples

$ git fetch origin master
$ git fmt-merge-msg --log <$GIT_DIR/FETCH_HEAD

Распечатайте сообщение журнала, описывающее слияние ветки "master" с удаленным "origin" .
