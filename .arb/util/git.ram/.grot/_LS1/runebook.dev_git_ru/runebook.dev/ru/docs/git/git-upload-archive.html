git-upload-archive

Name

git-upload-archive — отправить архив обратно в git-archive

Synopsis

git upload-archive <repository>

Description

Вызывается git archive --remote и отправляет сгенерированный архив на другой конец по протоколу Git .

Эта команда обычно не вызывается конечным пользователем напрямую. Пользовательский интерфейс протокола находится на стороне git archive , а программная пара предназначена для получения архива из удаленного репозитория.

Security

Чтобы защитить конфиденциальность объектов, которые были удалены из истории, но, возможно, еще не были удалены, git-upload-archive избегает обслуживания архивов для коммитов и деревьев, которые недоступны из ссылок репозитория. Однако, поскольку вычисление достижимости объекта требует больших вычислительных затрат, git-upload-archive реализует более строгий, но более простой для проверки набор правил:

  1. Клиенты могут запросить фиксацию или дерево, на которое указывает непосредственно ссылка. Например, git archive --remote=origin v1.0 .

  2. Клиенты могут запросить поддерево внутри коммита или дерева, используя синтаксис ref:path . Например, git archive --remote=origin v1.0:Documentation .

  3. Клиенты могут not использовать другие выражения sha1, даже если конечный результат достижим. Например, ни относительная фиксация, такая как master^ , ни буквальный sha1, такая как abcd1234 , не разрешены, даже если результат достижим из ссылок.

Обратите внимание, что правило 3 запрещает многие случаи, которые не имеют каких-либо последствий для конфиденциальности. Эти правила могут быть изменены в будущих версиях git, и сервер, к которому обращается git archive --remote , может следовать или не следовать этим точным правилам.

Если параметр конфигурации uploadArchive.allowUnreachable имеет значение true, эти правила игнорируются, и клиенты могут использовать произвольные выражения sha1. Это полезно, если вас не волнует конфиденциальность недоступных объектов или если ваша база данных объектов уже общедоступна для доступа через не-smart-http.

Options

<repository>

Репозиторий, из которого можно получить tar-архив.
