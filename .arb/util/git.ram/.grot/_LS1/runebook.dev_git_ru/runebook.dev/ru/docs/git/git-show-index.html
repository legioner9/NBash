git-show-index

Name

git-show-index — Показать индекс упакованного архива

Synopsis

git show-index [--object-format=<hash-algorithm>]

Description

Считайте файл .idx для пак-файла Git (созданного с помощью git-pack-objects[1] или git-index-pack[1] ) со стандартного ввода и выгрузите его содержимое. Вывод состоит из одного объекта в строке, причем каждая строка содержит два или три столбца, разделенных пробелами:

  • первый столбец — это смещение в байтах объекта в соответствующем пакетном файле.

  • второй столбец — это идентификатор объекта

  • если версия индекса равна 2 или выше, третий столбец содержит CRC32 данных объекта.

Объекты выводятся в том порядке, в котором они находятся в индексном файле, который (в правильно построенном файле) должен быть отсортирован по идентификатору объекта.

Обратите внимание, что дополнительную информацию о пакетном файле можно получить, позвонив по телефону git-verify-pack[1] . Однако, поскольку эта команда учитывает только сам индексный файл, она быстрее и гибче.

Options

--object-format=<hash-algorithm>

Укажите данный формат объекта (алгоритм хеширования) для индексного файла. Допустимые значения: sha1 и (если включено) sha256 . По умолчанию используется алгоритм для текущего репозитория (устанавливается extensions.objectFormat ) или sha1 , если значение не установлено или находится за пределами репозитория.

Примечание. В настоящее время между репозиториями SHA-256 и SHA-1 отсутствует совместимость.

Исторически мы предупреждали, что репозиториям SHA-256 впоследствии могут потребоваться изменения, несовместимые с предыдущими версиями, когда мы введем такие функции взаимодействия. Сегодня мы ожидаем только совместимых изменений. Более того, если такие изменения окажутся необходимыми, можно ожидать, что репозитории SHA-256, созданные с помощью сегодняшнего Git , можно будет использовать в будущих версиях Git без потери данных.
