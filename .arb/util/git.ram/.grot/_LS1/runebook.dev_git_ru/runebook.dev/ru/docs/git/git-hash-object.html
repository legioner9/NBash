git-hash-object

Name

git-hash-object — вычисляет идентификатор объекта и, при необходимости, создает объект из файла.

Synopsis

git hash-object [-t <type>] [-w] [--path=<file> | --no-filters]
                [--stdin [--literally]] [--] <file>…​
git hash-object [-t <type>] [-w] --stdin-paths [--no-filters]

Description

Вычисляет значение идентификатора объекта для объекта указанного типа с содержимым именованного файла (который может находиться за пределами рабочего дерева) и при необходимости записывает полученный объект в базу данных объектов. Сообщает идентификатор своего объекта на стандартный вывод. Если <тип> не указан, по умолчанию используется "blob" .

Options

-t <type>

Укажите тип создаваемого объекта (по умолчанию: "blob" ). Возможные значения: commit , tree , blob и tag .

-w

Фактически запишите объект в базу данных объектов.

--stdin

Считайте объект из стандартного ввода, а не из файла.

--stdin-paths

Считайте имена файлов из стандартного ввода, по одному в строке, а не из командной строки.

--path

Хешировать объект, как если бы он находился по заданному пути. Расположение файла не влияет напрямую на значение хеш-функции, но путь используется для определения того, какие фильтры Git следует применить к объекту, прежде чем его можно будет поместить в базу данных объектов. В результате применения фильтров фактический блоб, помещенный в объектную базу данных, может отличаться от данного файла. Эта опция в основном полезна для хеширования временных файлов, расположенных за пределами рабочего каталога, или файлов, считанных из стандартного ввода.

--no-filters

Хешируйте содержимое как есть, игнорируя любые входные фильтры, которые были бы выбраны механизмом атрибутов, включая преобразование конца строки. Если файл считывается со стандартного ввода, это всегда подразумевается, если не указана опция --path .

--literally

Разрешить --stdin хэшировать любой мусор в свободный объект, который в противном случае не мог бы пройти стандартный анализ объектов или проверки git-fsck. Полезно для стресс-тестирования самого Git или воспроизведения характеристик поврежденных или поддельных объектов, встречающихся в дикой природе.
