git-check-ignore

Name

git-check-ignore — отладка gitignore/исключение файлов

Synopsis

git check-ignore [<options>] <pathname>…​
git check-ignore [<options>] --stdin

Description

Для каждого пути, заданного через командную строку или из файла через --stdin , проверьте, исключен ли файл с помощью .gitignore (или других входных файлов для механизма исключения), и выведите путь, если он исключен.

По умолчанию отслеживаемые файлы вообще не отображаются, поскольку на них не распространяются правила исключения; но см. «--no-index».

Options

-q, --quiet

Ничего не выводите, просто установите статус выхода. Это справедливо только для одного пути.

-v, --verbose

Вместо печати исключенных путей для каждого пути, соответствующего шаблону исключения, распечатайте шаблон исключения вместе с путем. (Соответствие шаблону исключения обычно означает, что путь исключен, но если шаблон начинается с " ! " , то это отрицательный шаблон, и его совпадение означает, что путь НЕ исключен.)

Правила приоритета внутри и между источниками исключения см. в gitignore[5] .

--stdin

Считайте имена путей из стандартного ввода, по одному в каждой строке, а не из командной строки.

-z

Формат вывода изменяется для машинного анализа (см. ниже). Если также указан --stdin , входные пути разделяются символом NUL вместо символа перевода строки.

-n, --non-matching

Показать заданные пути, которые не соответствуют ни одному шаблону. Это имеет смысл только тогда, когда включен --verbose , иначе было бы невозможно отличить пути, соответствующие шаблону, от путей, которые не соответствуют шаблону.

--no-index

Не смотрите в индекс при проведении проверок. Это можно использовать для отладки того, почему путь отслеживается, например, git add . , и не игнорируется правилами, как ожидал пользователь, или при разработке шаблонов, включая отрицание, для соответствия пути, ранее добавленному с помощью git add -f .

Output

По умолчанию будет выведено любое из заданных путей, соответствующих шаблону игнорирования, по одному в строке. Если ни один шаблон не соответствует данному пути, для этого пути ничего не будет выведено; это означает, что путь не будет игнорироваться.

Если указан --verbose , выходные данные представляют собой серию строк вида:

<источник> <Двоеточие> <номер строки> <Двоеточие> <шаблон> <HT> <путь>

<pathname> — это путь к запрашиваемому файлу, <pattern> — соответствующий шаблон, <source> — исходный файл шаблона, а <linenum> — номер строки шаблона в этом источнике. Если шаблон содержал префикс " ! " или суффикс " / " , он будет сохранен в выходных данных. <source> будет абсолютным путем при ссылке на файл, настроенный core.excludesFile , или относительно корня репозитория при ссылке на .git/info/exclude или файл исключения для каждого каталога.

Если указан -z , имена путей в выходных данных разделяются символом null ; если также указан --verbose , то вместо двоеточий и жестких табуляций также используются символы null :

<источник> <NULL> <номер строки> <NULL> <шаблон> <NULL> <путь> <NULL>

Если указаны -n или --non-matching , несовпадающие пути также будут выведены, и в этом случае все поля в каждой выходной записи, кроме <pathname>, будут пустыми. Это может быть полезно при работе в неинтерактивном режиме, так что файлы можно постепенно передавать в поток STDIN длительного процесса проверки-игнорирования, и для каждого из этих файлов STDOUT будет указывать, соответствует ли этот файл шаблону или нет. (Без этой опции было бы невозможно определить, означает ли отсутствие вывода для данного файла, что он не соответствует какому-либо шаблону, или что вывод еще не был сгенерирован.)

Буферизация происходит, как описано в опции GIT_FLUSH в git[1] . Вызывающая сторона отвечает за предотвращение взаимоблокировок, вызванных переполнением входного буфера или чтением из пустого выходного буфера.

Exit status

0

Один или несколько предоставленных путей игнорируются.

1

None предоставленных путей игнорируются.

128

Произошла фатальная ошибка.
