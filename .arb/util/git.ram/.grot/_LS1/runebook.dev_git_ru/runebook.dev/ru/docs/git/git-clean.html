git-clean

Name

git-clean — удалить неотслеживаемые файлы из рабочего дерева

Synopsis

git clean [-d] [-f] [-i] [-n] [-q] [-e <pattern>] [-x | -X] [--] [<pathspec>…​]

Description

Очищает рабочее дерево, рекурсивно удаляя файлы, не находящиеся под контролем версий, начиная с текущего каталога.

Обычно удаляются только файлы, неизвестные Git , но если указана опция -x , игнорируемые файлы также удаляются. Это может быть полезно, например, для удаления всех продуктов сборки.

Если указаны какие-либо необязательные аргументы <pathspec>... , затрагиваются только те пути, которые соответствуют спецификации пути.

Options

-d

Обычно, если <pathspec> не указан, git clean не будет рекурсивно обращаться к неотслеживаемым каталогам, чтобы избежать слишком большого удаления. Укажите -d, чтобы он также выполнял рекурсию в такие каталоги. Если указан <pathspec>, -d не имеет значения; все неотслеживаемые файлы, соответствующие указанным путям (за исключением вложенных каталогов git , упомянутых в разделе --force ), будут удалены.

-f
--force

Если для переменной конфигурации Git clean.requireForce не установлено значение false, git clean откажется удалять файлы или каталоги, если не указано -f или -i. Git откажется изменять неотслеживаемые вложенные репозитории git (каталоги с подкаталогом . git ), если не указан второй ключ -f.

-i
--interactive

Покажите, что будет сделано, и очистите файлы в интерактивном режиме. Подробности см. в разделе «Интерактивный режим».

-n
--dry-run

На самом деле ничего не удаляйте, просто покажите, что будет сделано.

-q
--quiet

Будьте спокойны, сообщайте только об ошибках, а не об успешно удаленных файлах.

-e <pattern>
--exclude=<pattern>

Используйте данный шаблон исключения в дополнение к стандартным правилам игнорирования (см. gitignore[5] ).

-x

Не используйте стандартные правила игнорирования (см. gitignore[5] ), но по-прежнему используйте правила игнорирования, заданные параметрами -e в командной строке. Это позволяет удалить все неотслеживаемые файлы, включая продукты сборки. Это можно использовать (возможно, в сочетании с git restore или git reset ) для создания чистого рабочего каталога для тестирования чистой сборки.

-X

Удалите только файлы, игнорируемые Git. Это может быть полезно для восстановления всего с нуля, но при сохранении файлов, созданных вручную.

Interactive mode

Когда команда переходит в интерактивный режим, она показывает файлы и каталоги, которые необходимо очистить, и переходит в интерактивный командный цикл.

Командный цикл показывает список доступных подкоманд и выдает приглашение "What now> " . В общем, когда приглашение заканчивается одним > , вы можете выбрать только один из предложенных вариантов и ввести return, например:

    *** Commands ***
        1: clean                2: filter by pattern    3: select by numbers
        4: ask each             5: quit                 6: help
    What now> 1

Вы также можете сказать c или clean выше, если выбор уникален.

Основной командный цикл имеет 6 подкоманд.

clean

Начните очистку файлов и каталогов, а затем завершите работу.

фильтровать по шаблону

При этом отображаются файлы и каталоги, которые необходимо удалить, и выдается запрос "Input ignore patterns>>" . Вы можете ввести шаблоны, разделенные пробелами, чтобы исключить файлы и каталоги из удаления. Например, "*.c *.h" исключит из удаления файлы, заканчивающиеся на ".c" и ".h" . Когда вы будете удовлетворены отфильтрованным результатом, нажмите ENTER (пусто) для возврата в главное меню.

выбирать по номерам

При этом отображаются файлы и каталоги, которые необходимо удалить, и выводится приглашение "Select items to delete>>" . Когда приглашение заканчивается двойным >> , как здесь, вы можете сделать более одного выбора, объединенного пробелами или запятыми. Также вы можете сказать диапазоны. Например, "2-5 7,9" , чтобы выбрать 2,3,4,5,7,9 из списка. Если второе число в диапазоне опущено, выбираются все оставшиеся элементы. Например, "7-" , чтобы выбрать 7,8,9 из списка. Вы можете сказать * , чтобы выбрать все. Также, если вы удовлетворены отфильтрованным результатом, нажмите ENTER (пусто) для возврата в главное меню.

ask each

Начнется очистка, и вы должны подтвердить одно за другим, чтобы удалить элементы. Обратите внимание, что это действие не так эффективно, как два действия выше.

quit

Это позволяет вам выйти, не выполняя никакой очистки.

help

Покажите краткое использование интерактивного git-clean.

Configuration

Все, что находится ниже этой строки в этом разделе, выборочно включено из документации git-config[1] . Содержание такое же, как и найденное там:

clean.requireForce

boolean , чтобы заставить git-clean ничего не делать, если не указано -f, -i или -n. По умолчанию истинно.
