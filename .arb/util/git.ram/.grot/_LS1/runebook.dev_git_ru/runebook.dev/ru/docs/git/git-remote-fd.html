git-remote-fd

Name

git-remote-fd — Отражать интеллектуальный транспортный поток обратно вызывающему абоненту

Synopsis

"fd::<infd>[,<outfd>][/<anything>]" (как URL)

Description

Этот помощник использует указанные файловые дескрипторы для подключения к удаленному серверу Git . Это предназначено не для конечных пользователей, а для программ и сценариев, вызывающих выборку, отправку или архивирование git .

Если указан только <infd>, предполагается, что это двунаправленный сокет, подключенный к удаленному серверу Git (git-upload-pack, git-receive-pack или git-upload-archive). Если указаны и <infd>, и <outfd>, предполагается, что они представляют собой каналы, подключенные к удаленному серверу Git (<infd> — это входящий канал, а <outfd> — исходящий канал).

Предполагается, что все процедуры установления связи уже завершены (например, отправка запроса на обслуживание для git://) до запуска этого помощника.

<что-нибудь> может быть любой строкой. Это игнорируется. Он предназначен для предоставления пользователю информации в URL-адресе в случае, если этот URL-адрес отображается в некотором контексте.

Environment variables

GIT_TRANSLOOP_DEBUG

Если установлено, выводит отладочную информацию о различных операциях чтения/записи.

Examples

git fetch fd::17 master

Получите мастер-файл, используя файловый дескриптор № 17 для связи с git-upload-pack.

git fetch fd::17/foo master

То же, что и выше.

git push fd::7,8 master (as URL)

Нажмите master, используя файловый дескриптор №7 для чтения данных из git-receive-pack и файловый дескриптор №8 для записи данных в тот же сервис.

git push fd::7,8/bar master

То же, что и выше.
