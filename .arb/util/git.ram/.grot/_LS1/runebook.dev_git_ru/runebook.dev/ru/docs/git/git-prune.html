git-prune

Name

git-prune — удалить все недоступные объекты из базы данных объектов.

Synopsis

git prune [-n] [-v] [--progress] [--expire <time>] [--] [<head>…​]

Description

Note
 В большинстве случаев пользователям следует запускать git gc, который вызывает сокращение git . См. раздел "NOTES" ниже.

При этом git fsck --unreachable запускается с использованием всех ссылок, доступных в refs/ , при необходимости с дополнительным набором объектов, указанным в командной строке, и удаляет все распакованные объекты, недоступные для любого из этих головных объектов из базы данных объектов. Кроме того, он удаляет распакованные объекты, которые также находятся в пакетах, запустив git prune-packed . Он также удаляет записи из .git/shallow, которые недоступны ни для одной ссылки.

Обратите внимание, что недоступные упакованные объекты останутся. Если это нежелательно, см. git-repack[1] .

Options

-n
--dry-run

Ничего не удаляйте; просто сообщите, что он удалит.

-v
--verbose

Сообщайте обо всех удаленных объектах.

--progress

Show progress.

--expire <time>

Срок действия истекает только у незакрепленных объектов старше <time>.

--

Не интерпретируйте дополнительные аргументы как варианты.

<head>…

В дополнение к объектам, доступным по любой из наших ссылок, оставьте объекты доступными из перечисленных <head>.

Examples

Чтобы удалить объекты, не используемые вашим репозиторием или другим, который заимствует из вашего репозитория через его .git/objects/info/alternates :

$ git prune $(cd ../another && git rev-parse --all)

Notes

В большинстве случаев пользователям не нужно будет напрямую вызывать git prune , а вместо этого следует вызвать git gc , который выполняет обрезку наряду со многими другими служебными задачами.

Описание объектов, подлежащих сокращению, см. в параметре --unreachable git fsck .
