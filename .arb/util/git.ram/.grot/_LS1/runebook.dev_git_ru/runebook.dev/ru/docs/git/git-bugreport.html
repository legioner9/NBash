git-bugreport

Name

git-bugreport — собирать информацию, чтобы пользователь мог отправить отчет об ошибке.

Synopsis

git bugreport [(-o | --output-directory) <path>] [(-s | --suffix) <format>]
                [--diagnose[=<mode>]]

Description

Собирает информацию о компьютере пользователя, клиенте Git и состоянии репозитория, а также форму, запрашивающую информацию о поведении, наблюдаемом пользователем, и сохраняет ее в одном текстовом файле, которым пользователь затем может поделиться, например, в списке рассылки Git . , чтобы сообщить о замеченной ошибке.

У пользователя запрашивается следующая информация:

  • Reproduction steps

  • Expected behavior

  • Actual behavior

Следующая информация фиксируется автоматически:

  • git version --build-options

  • uname системное имя, выпуск, версия и строки машины

  • Информационная строка, специфичная для компилятора

  • Список включенных хуков

  • $SHELL

Дополнительную информацию можно собрать в отдельный zip-архив с помощью опции --diagnose и прикрепить к документу с отчетом об ошибке, чтобы предоставить читателям дополнительный контекст.

Этот инструмент вызывается посредством типичного процесса установки Git , а это означает, что в некоторых случаях его запуск может быть невозможен, например, если соответствующий файл конфигурации не читается. В таком случае может быть полезно вручную собрать информацию, указанную выше, при обращении за помощью вручную.

Options

-o <path>
--output-directory <path>

Поместите полученный файл отчета об ошибке в <path> вместо текущего каталога.

-s <format>
--suffix <format>

Укажите альтернативный суффикс для имени отчета об ошибке, чтобы создать файл с именем git-bugreport-<formatted suffix> . Это должно иметь форму строки формата strftime(3); будет использоваться текущее местное время.

--no-diagnose
--diagnose[=<mode>]

Создайте zip-архив с дополнительной информацией о компьютере пользователя, клиенте Git и состоянии репозитория. Архив записывается в тот же выходной каталог, что и отчет об ошибке, и называется git-diagnostics-<formatted suffix> .

Если не указан mode , диагностический архив будет содержать набор статистики по умолчанию, сообщаемый git diagnose . Можно указать дополнительное значение mode , чтобы изменить информацию, включаемую в архив. См. git-diagnose[1] для получения списка допустимых значений для mode и подробностей об их использовании.
