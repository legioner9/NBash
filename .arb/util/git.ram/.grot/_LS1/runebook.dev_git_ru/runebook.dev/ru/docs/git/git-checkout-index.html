git-checkout-index

Name

git-checkout-index — копировать файлы из индекса в рабочее дерево

Synopsis

git checkout-index [-u] [-q] [-a] [-f] [-n] [--prefix=<string>]
                   [--stage=<number>|all]
                   [--temp]
                   [--ignore-skip-worktree-bits]
                   [-z] [--stdin]
                   [--] [<file>…​]

Description

Копирует все перечисленные файлы из индекса в рабочий каталог (не перезаписывая существующие файлы).

Options

-u
--index

обновить информацию о статистике для извлеченных записей в индексном файле.

-q
--quiet

молчать, если файлы существуют или их нет в индексе

-f
--force

принудительно перезаписать существующие файлы

-a
--all

проверяет все файлы в индексе, за исключением тех, у которых установлен бит пропуска рабочего дерева (см. --ignore-skip-worktree-bits ). Невозможно использовать вместе с явными именами файлов.

-n
--no-create

Не извлекайте новые файлы, а только обновляйте уже извлеченные файлы.

--prefix=<string>

При создании файлов добавьте <строку> (обычно каталог, включающий в конце /).

--stage=<number>|all

Вместо извлечения несвязанных записей скопируйте файлы из именованного этапа. <число> должно быть от 1 до 3. Примечание. --stage=all автоматически подразумевает --temp.

--temp

Вместо копирования файлов в рабочий каталог запишите содержимое во временные файлы. Временные ассоциации имен будут записаны в стандартный вывод.

--ignore-skip-worktree-bits

Проверьте все файлы, включая те, у которых установлен бит Skip-worktree.

--stdin

Вместо того, чтобы брать список путей из командной строки, прочитайте список путей из стандартного ввода. По умолчанию пути разделяются LF (т.е. один путь в строке).

-z

Имеет смысл только с --stdin ; пути разделяются символом NUL вместо LF.

--

Не интерпретируйте дополнительные аргументы как варианты.

Раньше порядок флагов имел значение, но теперь это не так.

Просто выполнение git checkout-index ничего не дает. Вы, вероятно, имели в виду git checkout-index -a . А если вы хотите заставить его, вам нужен git checkout-index -f -a .

Интуитивность здесь не является целью. Повторяемость есть. Причина поведения "no arguments means no work" заключается в том, что из сценариев вы должны иметь возможность выполнять:

$ find . -name '*.h' -print0 | xargs -0 git checkout-index -f --

что приведет к замене всех существующих файлов *.h их кэшированными копиями. Если бы пустая командная строка подразумевала "all" , то это привело бы к принудительному обновлению всего в индексе, что не имело смысла. Но поскольку git checkout-index принимает --stdin, было бы быстрее использовать:

$ find . -name '*.h' -print0 | git checkout-index -f -z --stdin

-- – это просто хорошая идея, если вы знаете, что остальное будет именами файлов; это предотвратит проблемы с именем файла, например, -a . Использование -- , вероятно, является хорошей политикой в ​​сценариях.

Использование --temp или --stage=all

Когда используется --temp (или подразумевается --stage=all ), git checkout-index создает временный файл для каждой извлекаемой записи индекса. Индекс не будет обновляться статистической информацией. Эти параметры могут быть полезны, если вызывающей стороне нужны все этапы всех неслитых записей, чтобы несвязанные файлы могли быть обработаны внешним инструментом слияния.

В стандартный вывод будет записан листинг, обеспечивающий связь имен временных файлов с именами отслеживаемых путей. Формат листинга имеет два варианта:

  1. имя_времени TAB путь RS

    Первый формат используется, когда --stage опущен или не является --stage=all . Поле tempname — это имя временного файла, содержащее содержимое файла, а path — это имя отслеживаемого пути в индексе. Выводятся только запрошенные записи.

  2. stage1temp SP stage2temp SP stage3tmp TAB путь RS

    Второй формат используется, когда --stage=all . В трех временных полях этапа (stage1temp, stage2temp, stage3temp) указывается имя временного файла, если в индексе есть запись этапа, или . , если записи этапа нет. Пути, которые имеют только запись стадии 0, всегда будут исключены из вывода.

В обоих форматах RS (разделитель записей) по умолчанию представляет собой новую строку, но будет байтом null , если в командной строке был передан параметр -z. Имена временных файлов всегда представляют собой безопасные строки; они никогда не будут содержать разделители каталогов или символы пробелов. Поле пути всегда указывается относительно текущего каталога, а имена временных файлов всегда относятся к каталогу верхнего уровня.

Если объект, копируемый во временный файл, является символической ссылкой, содержимое ссылки будет записано в обычный файл. Использование этой информации зависит от конечного пользователя или компании Porcelain.

Examples

Чтобы обновить и обновить только уже извлеченные файлы
$ git checkout-index -n -f -a && git update-index --ignore-missing --refresh
Использование индекса проверки git для "export an entire tree"

Возможность использования префикса упрощает использование git checkout-index в качестве функции "export as tree" . Просто прочитайте нужное дерево в индекс и выполните:

$ git checkout-index --prefix=git-export-dir/ -a

git checkout-index будет "export" индексировать указанный каталог.

Последний "/" важен. Экспортированное имя буквально начинается с указанной строки. Сравните это со следующим примером.

Экспортировать файлы с префиксом
$ git checkout-index --prefix=.merged- Makefile

Это проверит текущую кэшированную копию Makefile в файл .merged-Makefile .
