git-cvsexportcommit

Name

git-cvsexportcommit — экспортировать один коммит в кассу CVS.

Synopsis

git cvsexportcommit [-h] [-u] [-v] [-c] [-P] [-p] [-a] [-d <cvsroot>]
        [-w <cvs-workdir>] [-W] [-f] [-m <msgprefix>] [<parent-commit>] <commit-id>

Description

Экспортирует фиксацию из Git в хранилище CVS, что упрощает объединение исправлений из репозитория Git в репозиторий CVS.

Укажите имя извлечения CVS с помощью ключа -w или выполните его из корня рабочей копии CVS. В последнем случае необходимо определить GIT_DIR. См. примеры ниже.

Он делает все возможное, чтобы обеспечить безопасность: он проверит, что файлы не изменены и не обновлены при проверке CVS, и по умолчанию не выполняет автоматическую фиксацию.

Поддерживает добавление, удаление и фиксацию файлов, влияющих на двоичные файлы.

Если фиксация является фиксацией слияния, вы должны указать git cvsexportcommit , против какого родителя следует выполнить набор изменений.

Options

-c

Зафиксируйте автоматически, если патч применился правильно. Он не будет зафиксирован, если какие-либо фрагменты не будут применены или возникнут другие проблемы.

-p

Будьте педантичны (параноиком) при применении патчей. Вызывает патч с --fuzz=0

-a

Добавьте информацию об авторстве. Добавляет в сообщение строку «Автор» и «Коммиттер» (если отличается от «Автор»).

-d

Установите альтернативный CVSROOT для использования. Это соответствует параметру CVS -d. Обычно пользователи не захотят устанавливать это, за исключением случаев асимметричного использования CVS.

-f

Принудительно выполнить слияние, даже если файлы не обновлены.

-P

Принудительно зафиксировать родительский элемент, даже если он не является прямым родителем.

-m

Добавьте к сообщению фиксации указанный префикс. Полезно для серий патчей и тому подобного.

-u

Перед попыткой экспорта обновите затронутые файлы из репозитория CVS.

-k

Отмените расширение ключевого слова CVS (например, $Revision: 1.2.3.4$ становится $Revision$) при рабочей проверке CVS перед применением исправления.

-w

Укажите расположение оформления CVS, которое будет использоваться для экспорта. Этот параметр не требует установки GIT_DIR перед выполнением, если текущий каталог находится в репозитории Git . По умолчанию используется значение cvsexportcommit.cvsdir .

-W

Сообщите cvsexportcommit, что текущий рабочий каталог — это не только проверка Git , но и проверка CVS. Поэтому Git сбросит рабочий каталог до родительского коммита, прежде чем продолжить.

-v

Verbose.

Configuration

cvsexportcommit.cvsdir

Местоположение по умолчанию оформления заказа CVS, которое будет использоваться для экспорта.

Examples

Объединить один патч в CVS
$ export GIT_DIR=~/project/.git
$ cd ~/project_cvs_checkout
$ git cvsexportcommit -v <commit-sha1>
$ cvs commit -F .msg <files>
Объедините один патч в CVS (опции -c и -w). Рабочий каталог находится в репозитории Git .
        $ git cvsexportcommit -v -c -w ~/project_cvs_checkout <commit-sha1>
Автоматически объединяйте ожидающие исправления в CVS — только если вы действительно знаете, что делаете.
$ export GIT_DIR=~/project/.git
$ cd ~/project_cvs_checkout
$ git cherry cvshead myhead | sed -n 's/^+ //p' | xargs -l1 git cvsexportcommit -c -p -v
