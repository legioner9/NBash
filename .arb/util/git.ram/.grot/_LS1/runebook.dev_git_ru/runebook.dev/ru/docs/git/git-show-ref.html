git-show-ref

Name

git-show-ref — список ссылок в локальном репозитории.

Synopsis

git show-ref [--head] [-d | --dereference]
             [-s | --hash[=<n>]] [--abbrev[=<n>]] [--tags]
             [--heads] [--] [<pattern>…​]
git show-ref --verify [-q | --quiet] [-d | --dereference]
             [-s | --hash[=<n>]] [--abbrev[=<n>]]
             [--] [<ref>…​]
git show-ref --exclude-existing[=<pattern>]
git show-ref --exists <ref>

Description

Отображает ссылки, доступные в локальном репозитории, вместе со связанными идентификаторами фиксации. Результаты можно фильтровать с использованием шаблона, а теги можно разыменовывать в идентификаторы объектов. Кроме того, его можно использовать для проверки существования конкретной ссылки.

По умолчанию отображаются теги, заголовки и удаленные ссылки.

Форма --exclude-existing представляет собой фильтр, который делает обратное. Он считывает ссылки из стандартного ввода, по одной ссылке на строку, и показывает те, которых нет в локальном репозитории.

Форма --exists может использоваться для проверки существования одной ссылки. Эта форма не проверяет, разрешается ли ссылка на реальный объект.

Использование этой утилиты рекомендуется для прямого доступа к файлам в каталоге .git .

Options

--head

Покажите ссылку HEAD, даже если она обычно отфильтровывается.

--heads
--tags

Ограничьтесь "refs/heads" и "refs/tags" соответственно. Эти варианты не являются взаимоисключающими; если заданы оба параметра, отображаются ссылки, хранящиеся в "refs/heads" и "refs/tags" .

-d
--dereference

Разыменовывание тегов также в идентификаторы объектов. Они будут показаны с добавлением ^{} .

-s
--hash[=<n>]

Покажите только OID, а не имя ссылки. В сочетании с --dereference разыменованный тег по-прежнему будет отображаться после OID.

--verify

Включите более строгую проверку ссылок, требуя точный путь к ссылке. Помимо возврата кода ошибки 1, он также выведет сообщение об ошибке, если --quiet не был указан.

--exists

Проверьте, существует ли данная ссылка. Возвращает код выхода 0, если да, 2, если он отсутствует, и 1, если поиск ссылки не выполнен из-за ошибки, отличной от отсутствия ссылки.

--abbrev[=<n>]

Сократите имя объекта. При использовании --hash не обязательно произносить --hash --abbrev ; --hash=n подойдет.

-q
--quiet

Не выводить результаты на стандартный вывод. Может использоваться с --verify для автоматической проверки существования ссылки.

--exclude-existing[=<pattern>]

Заставьте git show-ref действовать как фильтр, который считывает ссылки из стандартного ввода формы ^(?:<anything>\s)?<refname>(?:\^{})?$ и выполняет над каждым из них следующие действия: (1) удаляет ^{} в конце строки, если таковой имеется; (2) игнорировать, если указан шаблон и он не соответствует имени ссылки; (3) предупреждать, если refname не является корректным, и пропускать; (4) игнорировать, если refname является ссылкой, существующей в локальном репозитории; (5) в противном случае выведите строку.

<pattern>…

Показать ссылки, соответствующие одному или нескольким шаблонам. Шаблоны сопоставляются с конца полного имени, и сопоставляются только полные части, например, master соответствует refs/heads/master , refs/remotes/origin/master , refs/tags/jedi/master , но не соответствует refs/heads/mymaster или refs/remotes/master/jedi .

Output

Вывод имеет формат:

<oid> SP <ref> LF

For example,

$ git show-ref --head --dereference
832e76a9899f560a90ffd62ae2ce83bbeff58f54 HEAD
832e76a9899f560a90ffd62ae2ce83bbeff58f54 refs/heads/master
832e76a9899f560a90ffd62ae2ce83bbeff58f54 refs/heads/origin
3521017556c5de4159da4615a39fa4d5d2c279b5 refs/tags/v0.99.9c
6ddc0964034342519a87fe013781abf31c6db6ad refs/tags/v0.99.9c^{}
055e4ae3ae6eb344cbabf2a5256a49ea66040131 refs/tags/v1.0rc4
423325a2d24638ddcc82ce47be5e40be550f4507 refs/tags/v1.0rc4^{}
...

При использовании --hash (а не --dereference ) выходные данные имеют формат:

<oid> LF

For example,

$ git show-ref --heads --hash
2e3ba0114a1f52b47df29743d6915d056be13278
185008ae97960c8d551adcd9e23565194651b5d1
03adf42c988195b50e1a1935ba5fcbc39b2b029b
...

Examples

Чтобы отобразить все ссылки с именем "master" , будь то теги, заголовки или что-то еще, независимо от того, насколько глубоко они находятся в иерархии именования ссылок, используйте:

        git show-ref master

Это покажет "refs/heads/master" , а также "refs/remote/other-repo/master" , если такие ссылки существуют.

При использовании флага --verify для команды требуется точный путь:

        git show-ref --verify refs/heads/master

будет соответствовать только точной ветке с именем "master" .

Если ничего не совпадает, git show-ref вернет код ошибки 1, а в случае проверки покажет сообщение об ошибке.

Для сценариев вы можете попросить его сохранять тишину с помощью флага --quiet , который позволяет вам делать такие вещи, как

        git show-ref --quiet --verify -- "refs/heads/$headname" ||
                echo "$headname is not a valid branch"

чтобы проверить, существует ли конкретная ветка или нет (обратите внимание, что на самом деле мы не хотим показывать какие-либо результаты и хотим использовать для нее полное имя ссылки, чтобы не вызывать проблемы с неоднозначными частичными совпадениями).

Чтобы отображать только теги или только правильные заголовки ветвей, используйте --tags и/или --heads соответственно (использование обоих означает, что отображаются теги и заголовки, но не другие случайные ссылки в подкаталоге refs/).

Чтобы выполнить автоматическое разыменование объекта тега, используйте флаг -d или --dereference .

        git show-ref --tags --dereference

чтобы получить список всех тегов вместе с тем, что они разыменовывают.
