git-patch-id

Name

git-patch-id — вычислить уникальный идентификатор патча

Synopsis

git patch-id [--stable | --unstable | --verbatim]

Description

Считайте патч из стандартного ввода и вычислите для него идентификатор патча.

"patch ID" — это не что иное, как сумма SHA-1 различий файлов, связанных с патчем, без учета номеров строк. По сути, это "reasonably stable" , но в то же время он достаточно уникален, т. е. два патча, имеющих один и тот же "patch ID" , почти гарантированно будут одним и тем же.

Основной вариант использования этой команды — поиск вероятных повторяющихся коммитов.

При работе с выводом git diff-tree он использует тот факт, что патч имеет префикс с именем объекта фиксации, и выводит две 40-байтовые шестнадцатеричные строки. Первая строка — это идентификатор исправления, а вторая строка — идентификатор фиксации. Это можно использовать для сопоставления идентификатора патча с идентификатором фиксации.

Options

--verbatim

Вычислите идентификатор патча для ввода, как он задан, не удаляя пробелы.

This is the default if patchid.verbatim is true.
--stable

Используйте сумму хэшей "stable" в качестве идентификатора исправления. С помощью этой опции:

  • Изменение порядка различий файлов, составляющих патч, не влияет на идентификатор. В частности, два патча, созданные путем сравнения одних и тех же двух деревьев с двумя разными настройками для "-O<orderfile>" , приводят к одной и той же подписи идентификатора патча, что позволяет использовать вычисленный результат в качестве ключа для индексации некоторой метаинформации об изменении между двумя деревьями. ;

  • Результат отличается от значения, полученного git 1.9 и старше или полученного, когда настроен хеш "unstable" (см. --unstable ниже) - даже при использовании на выходе diff, полученном без использования "-O<orderfile>" , что приводит к тому, что существующие базы данных хранят такие "unstable" или исторические идентификаторы патчей непригодны для использования.

  • Все пробелы в патче игнорируются и не влияют на идентификатор.

    This is the default if patchid.stable is set to true.
--unstable

Используйте хэш "unstable" в качестве идентификатора исправления. При использовании этой опции полученный результат совместим со значением patch-id, созданным git 1.9 и старше, а пробелы игнорируются. Пользователи с уже существующими базами данных, хранящими идентификаторы исправлений, созданные git 1.9 и старше (которые не имеют дело с измененными исправлениями), могут захотеть использовать эту опцию.

This is the default.
