git-http-push

Name

git-http-push — отправлять объекты через HTTP/DAV в другой репозиторий.

Synopsis

git http-push [--all] [--dry-run] [--force] [--verbose] <URL> <ref> [<ref>…​]

Description

Отправляет недостающие объекты в удаленный репозиторий и обновляет удаленную ветку.

ПРИМЕЧАНИЕ. Эта команда временно отключена, если ваша библиотека libcurl старше 7.16, поскольку сообщается, что эта комбинация не работает и иногда повреждает репозиторий.

Options

--all

Не предполагайте, что удаленный репозиторий завершен в своем текущем состоянии, и убедитесь, что все объекты во всей истории локальных ссылок существуют в удаленном репозитории.

--force

Обычно команда отказывается обновлять удаленную ссылку, которая не является предком локальной ссылки, использованной для ее перезаписи. Этот флаг отключает проверку. Это означает, что удаленный репозиторий может потерять коммиты; используйте его с осторожностью.

--dry-run

Делайте все, кроме фактической отправки обновлений.

--verbose

Сообщите список объектов, просматриваемых локально, и список объектов, успешно отправленных в удаленный репозиторий.

-d
-D

Удалите <ref> из удаленного репозитория. Указанная ветка не может быть удаленным HEAD. Если указан -d, также должны быть выполнены следующие другие условия:

  • Удаленный HEAD должен разрешаться в объект, существующий локально.

  • Указанная ветвь разрешается в объект, существующий локально.

  • Указанная ветка является предком удаленного HEAD.

<ref>…

Удаленные ссылки для обновления.

Указание ссылок

Спецификация <ref> может представлять собой либо один шаблон, либо пару таких шаблонов, разделенных двоеточием ":" (это означает, что имя ссылки не может содержать двоеточие). Один шаблон <name> — это просто сокращение от <name>:<name> .

Каждая пара шаблонов <src>:<dst> состоит из стороны источника (до двоеточия) и стороны назначения (после двоеточия). Ссылка для отправки определяется путем поиска совпадения, соответствующего исходной стороне, а место ее отправки определяется с использованием стороны назначения.

  • Если <src> не соответствует точно одной из локальных ссылок, это ошибка.

  • Если <dst> не соответствует ни одному удаленному заданию, либо

    • он должен начинаться с "refs/" ; В этом случае <dst> используется в качестве пункта назначения буквально.

    • <src> == <dst> и ссылка, соответствующая <src>, не должна существовать в наборе удаленных ссылок; ссылка, соответствующая <src> локально, используется в качестве имени места назначения.

Без --force ссылка <src> сохраняется на удаленном компьютере, только если <dst> не существует или <dst> является правильным подмножеством (т. е. предком) <src>. Эта проверка, известная как "fast-forward check" , выполняется во избежание случайной перезаписи удаленной ссылки и потери оттуда коммитов других людей.

В --force проверка перемотки вперед отключена для всех ссылок.

При желании параметр <ref> может иметь префикс со знаком плюс + , чтобы отключить проверку быстрой перемотки вперед только для этой ссылки.
