git-archive

Name

git-archive — Создать архив файлов из именованного дерева.

Synopsis

git archive [--format=<fmt>] [--list] [--prefix=<prefix>/] [<extra>]
              [-o <file> | --output=<file>] [--worktree-attributes]
              [--remote=<repo> [--exec=<git-upload-archive>]] <tree-ish>
              [<path>…​]

Description

Создает архив указанного формата, содержащий древовидную структуру именованного дерева, и записывает его в стандартный вывод. Если указан <префикс>, он добавляется к именам файлов в архиве.

git archive ведет себя по-разному, если ему присвоен идентификатор дерева, а не идентификатор фиксации или идентификатор тега. Если указан идентификатор дерева, текущее время используется как время изменения каждого файла в архиве. С другой стороны, когда предоставляется идентификатор фиксации или идентификатор тега, вместо этого используется время фиксации, записанное в указанном объекте фиксации. Кроме того, идентификатор фиксации сохраняется в глобальном расширенном заголовке pax, если используется формат tar; его можно извлечь с помощью git get-tar-commit-id . В ZIP-файлах он хранится как комментарий к файлу.

Options

--format=<fmt>

Формат полученного архива. Возможные значения: tar , zip , tar.gz , tgz и любой формат, определенный с помощью параметра конфигурации tar.<format>.command . Если --format не указан и указан выходной файл, формат выводится из имени файла, если это возможно (например, запись в foo.zip приводит к тому, что выходные данные будут в формате zip ). В противном случае выходной формат — tar .

-l
--list

Показать все доступные форматы.

-v
--verbose

Сообщайте о ходе выполнения в stderr.

--prefix=<prefix>/

Добавьте <префикс>/ к путям в архиве. Можно повторить; его самое правое значение используется для всех отслеживаемых файлов. Ниже показано, какое значение используется --add-file и --add-virtual-file .

-o <file>
--output=<file>

Запишите архив в <file> вместо стандартного вывода.

--add-file=<file>

Добавьте в архив неотслеживаемый файл. Можно повторить, чтобы добавить несколько файлов. Путь к файлу в архиве создается путем объединения значения последней опции --prefix (если она есть) перед этой --add-file и базового имени <file>.

--add-virtual-file=<path>:<content>

Добавьте указанное содержимое в архив. Можно повторить, чтобы добавить несколько файлов. Путь к файлу в архиве строится путем объединения значения последней опции --prefix (если она есть) перед этой --add-virtual-file и <path> .

Аргумент <path> может начинаться и заканчиваться символом двойной кавычки; содержащееся имя файла интерпретируется как строка в стиле C, т. е. обратная косая черта интерпретируется как escape-символ. Путь должен быть заключен в кавычки, если он содержит двоеточие, чтобы двоеточие не было ошибочно интерпретировано как разделитель между путем и содержимым, или если путь начинается или заканчивается символом двойной кавычки.

Файловый режим ограничен обычным файлом, и на этот параметр могут распространяться ограничения командной строки, зависящие от платформы. В нетривиальных случаях запишите неотслеживаемый файл и используйте вместо него --add-file .

--worktree-attributes

Также ищите атрибуты в файлах .gitattributes в рабочем дереве (см. ATTRIBUTES ).

--mtime=<time>

Установите время изменения записей архива. Без этой опции используется время коммиттера, если <tree-ish> является фиксацией или тегом, и текущее время, если это дерево.

<extra>

Это могут быть любые параметры, которые понимает серверная часть архиватора. См. следующий раздел.

--remote=<repo>

Вместо создания tar-архива из локального репозитория извлеките tar-архив из удаленного репозитория. Обратите внимание, что удаленный репозиторий может налагать ограничения на то, какие выражения sha1 могут быть разрешены в <tree-ish> . Подробности см. в git-upload-archive[1] .

--exec=<git-upload-archive>

Используется с --remote для указания пути к git-upload-archive на удаленной стороне.

<tree-ish>

Дерево или коммит, для которого создается архив.

<path>

Без необязательного параметра пути в архив включаются все файлы и подкаталоги текущего рабочего каталога. Если указаны один или несколько путей, включаются только они.

Дополнительные опции бэкэнда

zip

-<digit>

Укажите уровень сжатия. Большие значения позволяют команде тратить больше времени на сжатие до меньшего размера. Поддерживаемые значения: от -0 (только для магазина) до -9 (лучшее соотношение). По умолчанию — -6 , если не указано иное.

tar

-<number>

Укажите уровень сжатия. Значение будет передано команде сжатия, настроенной в tar.<format>.command . Список поддерживаемых уровней и уровень по умолчанию, если этот параметр не указан, см. на странице руководства настроенной команды.

Configuration

tar.umask

Эту переменную можно использовать для ограничения битов разрешений записей архива tar. По умолчанию установлено значение 0002, которое отключает бит мировой записи. Специальное значение "user" указывает, что вместо этого будет использоваться маска пользователя, осуществляющего архивацию. Подробности смотрите в umask(2). Если используется --remote , то действует только конфигурация удаленного репозитория.

tar.<format>.command

Эта переменная указывает команду оболочки, через которую должен передаваться вывод tar, сгенерированный git archive . Команда выполняется с использованием оболочки с сгенерированным файлом tar на стандартном вводе и должна выдавать конечный результат на стандартный вывод. В команду будут переданы любые параметры уровня сжатия (например, -9 ).

Форматы tar.gz и tgz определяются автоматически и по умолчанию используют волшебную команду git archive gzip , которая вызывает внутреннюю реализацию gzip.

tar.<format>.remote

Если true , включите формат для использования удаленными клиентами через git-upload-archive[1] . По умолчанию используется false для пользовательских форматов, но true для форматов tar.gz и tgz .

Attributes

export-ignore

Файлы и каталоги с атрибутом «экспорт-игнор» не будут добавлены в архивные файлы. Подробности см. в gitattributes[5] .

export-subst

Если для файла установлен атрибут Export-subst, то Git расширит несколько заполнителей при добавлении этого файла в архив. Подробности см. в gitattributes[5] .

Обратите внимание, что атрибуты по умолчанию берутся из файлов .gitattributes в архивируемом дереве. Если вы хотите настроить способ генерации вывода постфактум (например, вы зафиксировали без добавления соответствующего экспорта-игнорирования в его .gitattributes ), отредактируйте извлеченный файл .gitattributes по мере необходимости и используйте опцию --worktree-attributes . Альтернативно вы можете сохранить необходимые атрибуты, которые должны применяться при архивировании любого дерева в файле $GIT_DIR/info/attributes .

Examples

git archive --format=tar --prefix=junk/ HEAD | (cd /var/tmp/ && tar xf -)

Создайте tar-архив, содержащий содержимое последней фиксации в текущей ветке, и извлеките его в каталог /var/tmp/junk .

git archive --format=tar --prefix=git-1.4.0/ v1.4.0 | gzip >git-1.4.0.tar.gz

Создайте сжатый архив для версии v1.4.0.

git archive --format=tar.gz --prefix=git-1.4.0/ v1.4.0 >git-1.4.0.tar.gz

То же, что и выше, но с использованием встроенной обработки tar.gz.

git archive --prefix=git-1.4.0/ -o git-1.4.0.tar.gz v1.4.0

То же, что и выше, но формат определяется из выходного файла.

git archive --format=tar --prefix=git-1.4.0/ v1.4.0^{tree} | gzip >git-1.4.0.tar.gz

Создайте сжатый архив для версии 1.4.0, но без глобального расширенного заголовка pax.

git archive --format=zip --prefix=git-docs/ HEAD:Documentation/ > git-1.4.0-docs.zip

Поместите все из каталога Documentation/ текущей главы в git-1.4.0-docs.zip с префиксом git-docs/ .

git archive -o latest.zip HEAD

Создайте Zip-архив, содержащий содержимое последнего коммита в текущей ветке. Обратите внимание, что выходной формат определяется расширением выходного файла.

git archive -o latest.tar --prefix=build/ --add-file=configure --prefix= HEAD

Создает tar-архив, содержащий содержимое последней фиксации в текущей ветке без префикса и неотслеживаемый файл configure с префиксом build/ .

git config tar.tar.xz.command "xz -c"

Настройте формат "tar.xz" для создания tar-файлов, сжатых LZMA. Вы можете использовать его, указав --format=tar.xz или создав выходной файл, например -o foo.tar.xz .
