git-ls-remote

Name

git-ls-remote — список ссылок в удаленном репозитории.

Synopsis

git ls-remote [--heads] [--tags] [--refs] [--upload-pack=<exec>]
              [-q | --quiet] [--exit-code] [--get-url] [--sort=<key>]
              [--symref] [<repository> [<patterns>…​]]

Description

Отображает ссылки, доступные в удаленном репозитории, вместе со связанными идентификаторами фиксации.

Options

-h
--heads
-t
--tags

Ограничьтесь только ссылками/заголовками и ссылками/тегами соответственно. Эти опции not являются взаимоисключающими; если заданы оба параметра, отображаются ссылки, хранящиеся в refs/heads и refs/tags. Обратите внимание, что git ls-remote -h , используемый без каких-либо других команд в командной строке, дает справку, согласующуюся с другими подкомандами git .

--refs

Не показывайте в выводе очищенные теги или псевдоссылки, такие как HEAD .

-q
--quiet

Не выводить удаленный URL-адрес в stderr.

--upload-pack=<exec>

Укажите полный путь к git-upload-pack на удаленном хосте. Это позволяет выводить список ссылок из репозиториев, к которым осуществляется доступ через SSH, и где демон SSH не использует PATH, настроенный пользователем.

--exit-code

Выйдите со статусом "2" , если в удаленном репозитории не найдены соответствующие ссылки. Обычно команда завершается со статусом "0" , чтобы указать, что она успешно связалась с удаленным репозиторием и обнаружила ли она какие-либо соответствующие ссылки.

--get-url

Расширьте URL-адрес данного удаленного репозитория, принимая во внимание любые настройки конфигурации "url.<base>.insteadOf" (см. git-config[1] ) и выйдите, не разговаривая с удаленным устройством.

--symref

В дополнение к объекту, на который он указывает, покажите базовую ссылку, на которую он указывает, при отображении символической ссылки. В настоящее время пакет загрузки показывает только HEAD symref, поэтому ls-remote будет отображать только его.

--sort=<key>

Сортировка по указанному ключу. Префикс - для сортировки в порядке убывания значения. Поддерживает "version:refname" или "v:refname" (имена тегов считаются версиями). На порядок сортировки "version:refname" также может влиять переменная конфигурации "versionsort.suffix" . Дополнительные параметры сортировки см. в git-for-each-ref[1] , но имейте в виду, что такие ключи, как committerdate , которые требуют доступа к самим объектам, не будут работать для ссылок, объекты которых еще не были получены с удаленного устройства, и выдадут ошибку missing object .

-o <option>
--server-option=<option>

Передайте данную строку на сервер при обмене данными по протоколу версии 2. Данная строка не должна содержать символ NUL или LF. Если задано несколько --server-option=<option> , все они отправляются другой стороне в порядке, указанном в командной строке.

<repository>

Репозиторий "remote" для запроса. Этот параметр может быть либо URL-адресом, либо именем удаленного устройства (см. разделы «URL-адреса GIT » и «REMOTES» документа git-fetch[1] ).

<patterns>…

Если не указано, отображаются все ссылки после фильтрации с помощью --heads и --tags. Если указаны <patterns>…, отображаются только ссылки, соответствующие одному или нескольким заданным шаблонам. Каждый шаблон интерпретируется как glob (см. glob в gitglossary[7] ), который сопоставляется с "tail" ссылки, начиная либо с начала ссылки (так что полное имя, например refs/heads/foo , соответствует), либо с разделителя косой черты (так что bar соответствует refs/heads/bar ). но не refs/heads/foobar ).

Output

Вывод имеет формат:

<oid> TAB <ref> LF

При отображении аннотированного тега, если не указан --refs , отображаются две такие строки: одна с именем ссылки для самого тега как <ref> , а другая с <ref> , за которой следует ^{} . <oid> в последней строке показывает имя объекта, на который указывает тег.

Examples

  • Перечислите все ссылки (включая символику и псевдоссылки), очистив теги:

    $ git ls-remote
27d43aaaf50ef0ae014b88bba294f93658016a2e        HEAD
950264636c68591989456e3ba0a5442f93152c1a        refs/heads/main
d9ab777d41f92a8c1684c91cfb02053d7dd1046b        refs/heads/next
d4ca2e3147b409459955613c152220f4db848ee1        refs/tags/v2.40.0
73876f4861cd3d187a4682290ab75c9dccadbc56        refs/tags/v2.40.0^{}

  • Перечислите все ссылки, соответствующие заданным шаблонам:

    $ git ls-remote http://www.kernel.org/pub/scm/git/git.git master seen rc
5fe978a5381f1fbad26a80e682ddd2a401966740        refs/heads/master
c781a84b5204fb294c9ccc79f8b3baceeb32c061        refs/heads/seen

  • Перечислите только теги, соответствующие заданному шаблону подстановочных знаков:

    $ git ls-remote --tags http://www.kernel.org/pub/scm/git/git.git v\*
485a869c64a68cc5795dd99689797c5900f4716d        refs/tags/v2.39.2
cbf04937d5b9fcf0a76c28f69e6294e9e3ecd7e6        refs/tags/v2.39.2^{}
d4ca2e3147b409459955613c152220f4db848ee1        refs/tags/v2.40.0
73876f4861cd3d187a4682290ab75c9dccadbc56        refs/tags/v2.40.0^{}
