git-pack-redundant

Name

git-pack-redundant — найти избыточные файлы пакетов.

Synopsis

git pack-redundant [--verbose] [--alt-odb] (--all | <pack-filename>…​)

Warning

git pack-redundant устарел, и его планируется удалить в будущей версии Git. Поскольку он может удалять только целые повторяющиеся пакеты, а не отдельные повторяющиеся объекты, обычно он не является полезным инструментом для уменьшения размера репозитория. Для этого лучше использовать git gc , который поместит объекты в новый пакет, удалив дубликаты.

В этом выпуске запуск pack-redundant без флага --i-still-use-this не удастся. Если вы считаете, что у вас есть вариант использования, для которого pack-redundant лучше подходит, и вы возражаете против такого удаления, обратитесь в список рассылки Git по адресу [email protected] . Дополнительную информацию о списке можно получить по адресу https://git-scm.com/community .

Description

Эта программа вычисляет, какие пакеты в вашем репозитории являются избыточными. Вывод подходит для передачи на xargs rm , если вы находитесь в корне репозитория.

git pack-redundant принимает список объектов на стандартный ввод. Любые заданные объекты будут игнорироваться при проверке необходимых пакетов. Это делает следующую команду полезной при необходимости удалить пакеты, содержащие недоступные объекты.

git fsck --full --недоступно | вырезать -d ' ' -f3 | \ git пакетно-резервный --all | xargs РМ

Options

--all

Обрабатывает все пакеты. Любые имена файлов в командной строке игнорируются.

--alt-odb

Не требуйте, чтобы объекты, присутствующие в пакетах из каталогов альтернативной базы данных объектов (odb), присутствовали в локальных пакетах.

--verbose

Выводит некоторую статистику в stderr. Имеет небольшой штраф за производительность.
