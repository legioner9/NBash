git-help

Name

git-help — показать справочную информацию о Git .

Synopsis

git help [-a|--all] [--[no-]verbose] [--[no-]external-commands] [--[no-]aliases]
git help [[-i|--info] [-m|--man] [-w|--web]] [<command>|<doc>]
git help [-g|--guides]
git help [-c|--config]
git help [--user-interfaces]
git help [--developer-interfaces]

Description

Если параметры и <command> или <doc> не заданы, краткое описание команды git и список наиболее часто используемых команд Git выводятся на стандартный вывод.

Если указана опция --all или -a , все доступные команды выводятся на стандартный вывод.

Если указана опция --guides или -g , список концептуальных руководств Git также печатается на стандартном выходе.

Если дана команда или другая документация, откроется соответствующая страница руководства. По умолчанию для этой цели используется программа man , но ее можно переопределить другими параметрами или переменными конфигурации.

Если задан псевдоним, git отображает определение псевдонима в стандартном выводе. Чтобы получить страницу руководства для команды с псевдонимом, используйте git <command> --help .

Обратите внимание, что git --help ... идентичен git help ... , поскольку первый внутренне преобразуется во второй.

Чтобы отобразить справочную страницу git[1] , используйте git help git .

Эту страницу можно отобразить с помощью git help help или git help --help .

Options

-a
--all

Выведите все доступные команды на стандартный вывод.

--no-external-commands

При использовании с --all исключите список внешних команд "git-*" , имеющихся в $PATH .

--no-aliases

При использовании с --all исключите список настроенных псевдонимов.

--verbose

При использовании с --all выводит описание всех распознанных команд. Это значение по умолчанию.

-c
--config

Перечислите все доступные переменные конфигурации. Это краткое изложение списка в git-config[1] .

-g
--guides

Напечатайте список концептуальных руководств Git на стандартном выходе.

--user-interfaces

Выведите список документации по репозиторию, командным и файловым интерфейсам на стандартный вывод.

Здесь документированы файловые интерфейсы внутри репозитория, такие как .git/info/exclude (см. gitrepository-layout[5] ), а также внутридеревьевая конфигурация, например .mailmap (см. gitmailmap[5] ).

В этом разделе документации также рассматриваются общие или широко распространенные соглашения пользовательского интерфейса (например, gitcli[7] ) и псевдоконфигурации, такие как файловый интерфейс .git/hooks/* , описанный в githooks[5] .

--developer-interfaces

Выведите список форматов файлов, протоколов и другую документацию по интерфейсам разработчика на стандартный вывод.

-i
--info

Отображение страницы руководства для команды в формате info . Для этой цели будет использоваться программа info .

-m
--man

Отображение страницы руководства для команды в формате man . Эту опцию можно использовать для переопределения значения, установленного в переменной конфигурации help.format .

По умолчанию программа man будет использоваться для отображения страницы руководства, но переменная конфигурации man.viewer может использоваться для выбора других программ отображения (см. ниже).

-w
--web

Отображение страницы руководства для команды в формате web (HTML). Для этой цели будет использоваться веб-браузер.

Веб-браузер можно указать с помощью переменной конфигурации help.browser или web.browser , если первая не установлена. Если ни одна из этих переменных конфигурации не установлена, вспомогательный сценарий git web--browse (вызываемый git help ) выберет подходящее значение по умолчанию. См. git-web--browse[1] для получения дополнительной информации об этом.

Configuration variables

help.format

Если параметр командной строки не указан, будет проверена переменная конфигурации help.format . Для этой переменной поддерживаются следующие значения; они заставляют git help вести себя как соответствующий параметр командной строки:

  • "man" соответствует -m|--man ,

  • "info" соответствует -i|--info ,

  • "web" или "html" соответствуют -w|--web .

help.browser, web.browser и браузер.<инструмент>.путь

help.browser , web.browser и browser.<tool>.path также будут проверены, если выбран формат web (либо с помощью параметра командной строки, либо с помощью переменной конфигурации). См. -w|--web в разделе ОПЦИИ выше и git-web--browse[1] .

man.viewer

Переменная конфигурации man.viewer будет проверена, если выбран формат man . В настоящее время поддерживаются следующие значения:

  • "man" : используйте программу man как обычно,

  • "woman" : используйте emacsclient для запуска режима "woman" в emacs (это работает только начиная с версии emacsclient 22),

  • "konqueror" : используйте kfmclient , чтобы открыть справочную страницу в новой вкладке Konqueror (см. Note about konqueror ниже).

Значения для других инструментов можно использовать, если имеется соответствующая запись конфигурации man.<tool>.cmd (см. ниже).

Переменной конфигурации man.viewer можно присвоить несколько значений. Соответствующие им программы будут опробованы в порядке, указанном в файле конфигурации.

Например, эта конфигурация:

        [man]
                viewer = konqueror
                viewer = woman

сначала попробую использовать konqueror. Но это может потерпеть неудачу (например, если DISPLAY не установлен), и в этом случае будет опробован женский режим emacs.

Если все не удалось или программа просмотра не настроена, будет опробована программа просмотра, указанная в переменной среды GIT_MAN_VIEWER . Если и это не поможет, программа man все равно будет опробована.

man.<tool>.path

Вы можете явно указать полный путь к предпочитаемой вами программе просмотра man, установив переменную конфигурации man.<tool>.path . Например, вы можете настроить абсолютный путь к konqueror, задав man.konqueror.path . В противном случае git help предполагает, что инструмент доступен в PATH.

man.<tool>.cmd

Если программа просмотра man, указанная переменными конфигурации man.viewer , не входит в число поддерживаемых, будет выполнен поиск соответствующей переменной конфигурации man.<tool>.cmd . Если эта переменная существует, то указанный инструмент будет рассматриваться как пользовательская команда, и для запуска команды будет использоваться оболочка eval с передачей страницы руководства в качестве аргументов.

Примечание о победителе

Когда в переменной конфигурации man.viewer указан konqueror , мы запускаем kfmclient , чтобы попытаться открыть справочную страницу уже открытого konqueror в новой вкладке, если это возможно.

Для единообразия мы также попробуем такой трюк, если для man.konqueror.path установлено что-то вроде A_PATH_TO/konqueror . Это значит, что вместо него мы попробуем запустить A_PATH_TO/kfmclient .

Если вы действительно хотите использовать konqueror , вы можете использовать что-то вроде следующего:

        [man]
                viewer = konq

        [man "konq"]
                cmd = A_PATH_TO/konqueror

Примечание о конфигурации git --global

Обратите внимание, что все эти переменные конфигурации, вероятно, следует установить с помощью флага --global , например так:

$ git config --global help.format веб-сайт
$ git config --global web.browser Firefox

поскольку они, вероятно, более специфичны для пользователя, чем для репозитория. См. git-config[1] для получения дополнительной информации об этом.
