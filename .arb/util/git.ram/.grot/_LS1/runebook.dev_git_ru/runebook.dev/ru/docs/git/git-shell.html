git-shell

Name

git-shell — Ограниченная оболочка входа для доступа по SSH только для Git.

Synopsis

chsh -s $(command -v git-shell) <user>
git clone <user>@localhost:/path/to/repo.git
ssh <user>@localhost

Description

Это оболочка входа для учетных записей SSH, обеспечивающая ограниченный доступ к Git . Он разрешает выполнение только серверных команд Git , реализующих функцию извлечения и отправки, а также пользовательских команд, присутствующих в подкаталоге с именем git-shell-commands в домашнем каталоге пользователя.

Commands

git shell принимает следующие команды после опции -c :

git пакет приема <аргумент>
git пакет загрузки <аргумент>
git загрузка-архив <аргумент>

Вызовите соответствующую серверную команду для поддержки запроса клиента git push , git fetch или git archive --remote .

cvs server

Имитируйте сервер CVS. См. git-cvsserver[1] .

Если каталог ~/git-shell-commands присутствует, git shell также будет обрабатывать другие пользовательские команды, запуская " git-shell-commands/<command> <arguments> " из домашнего каталога пользователя.

Interactive use

По умолчанию приведенные выше команды могут выполняться только с опцией -c ; оболочка не является интерактивной.

Если каталог ~/git-shell-commands присутствует, git shell также можно запускать в интерактивном режиме (без аргументов). Если в каталоге git-shell-commands присутствует команда help , она запускается, чтобы предоставить пользователю обзор разрешенных действий. Затем отображается приглашение "git> " , в котором можно ввести любую команду из каталога git-shell-commands или exit для закрытия соединения.

Обычно этот режим используется в качестве административного интерфейса, позволяющего пользователям составлять список репозиториев, к которым у них есть доступ, создавать, удалять или переименовывать репозитории, а также изменять описания репозиториев и разрешения.

Если команда no-interactive-login существует, она запускается и интерактивная оболочка прерывается.

Examples

Чтобы отключить интерактивный вход в систему и вместо этого отображать приветствие:

$ chsh -s /usr/bin/git-shell
$ mkdir $HOME/git-shell-commands
$ cat >$HOME/git-shell-commands/no-interactive-login <<\EOF
#!/bin/sh
printf '%s\n' "Hi $USER! You've successfully authenticated, but I do not"
printf '%s\n' "provide interactive shell access."
exit 128
EOF
$ chmod +x $HOME/git-shell-commands/no-interactive-login

Чтобы включить доступ к git-cvsserver (который обычно должен иметь приведенный выше пример no-interactive-login в качестве предварительного условия, поскольку создание каталога git-shell-commands позволяет осуществлять интерактивный вход в систему):

$ cat >$HOME/git-shell-commands/cvs <<\EOF
if ! test $# = 1 && test "$1" = "server"
then
        echo >&2 "git-cvsserver only handles \"server\""
        exit 1
fi
exec git cvsserver server
EOF
$ chmod +x $HOME/git-shell-commands/cvs
