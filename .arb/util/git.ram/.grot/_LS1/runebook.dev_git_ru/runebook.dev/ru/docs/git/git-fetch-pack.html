git-fetch-pack

Name

git-fetch-pack — Получить недостающие объекты из другого репозитория

Synopsis

git fetch-pack [--all] [--quiet|-q] [--keep|-k] [--thin] [--include-tag]
        [--upload-pack=<git-upload-pack>]
        [--depth=<n>] [--no-progress]
        [-v] <repository> [<refs>…​]

Description

Обычно вместо этого вы предпочитаете использовать git fetch , который является оболочкой более высокого уровня этой команды.

Вызывает git-upload-pack в возможно удаленном репозитории и просит его отправить объекты, отсутствующие в этом репозитории, для обновления именованных головок. Список доступных локально коммитов определяется путем сканирования локальных ссылок/иерархии и отправляется на git-upload-pack , работающий на другом конце.

Эта команда вырождается в загрузку всего, чтобы завершить запрошенные ссылки с удаленной стороны, когда на локальной стороне нет общего предка.

Options

--all

Получить все удаленные ссылки.

--stdin

Возьмите список ссылок из стандартного ввода, по одной в строке. Если помимо этой опции в командной строке указаны ссылки, то ссылки из стандартного ввода обрабатываются после ссылок в командной строке.

Если вместе с этой опцией указан --stateless-rpc , то список ссылок должен быть в пакетном формате (pkt-line). Каждая ссылка должна находиться в отдельном пакете, а список должен заканчиваться флеш-пакетом.

-q
--quiet

Передать флаг -q в git unpack-objects ; это делает процесс клонирования менее многословным.

-k
--keep

Не вызывайте git unpack-objects для полученных данных, а вместо этого создайте из них один пакетный файл и сохраните его в базе данных объектов. Если указано дважды, пакет заблокирован от переупаковки.

--thin

Получите пакет "thin" , который записывает объекты в разделенной форме на основе объектов, не включенных в пакет, чтобы уменьшить сетевой трафик.

--include-tag

Если удаленная сторона поддерживает это, объекты аннотированных тегов будут загружаться по тому же соединению, что и другие объекты, если загружен объект, на который ссылается тег. В противном случае вызывающая сторона должна определить теги, которые эта опция делает доступными.

--upload-pack=<git-upload-pack>

Используйте это, чтобы указать путь к git-upload-pack на удаленной стороне, если он не найден в вашем $PATH. При установке sshd игнорируются сценарии настройки среды пользователя для оболочек входа в систему (например, .bash_profile), и ваш частный установленный git может не быть найден в системном $PATH по умолчанию. Другой предлагаемый обходной путь — настроить $PATH в ".bashrc" , но этот флаг предназначен для людей, которые не хотят платить накладные расходы за неинтерактивные оболочки, имея простой файл .bashrc (большую часть вещей они настраивают в .bash_profile). ).

--exec=<git-upload-pack>

То же, что --upload-pack=<git-upload-pack>.

--depth=<n>

Ограничьте выборку цепочек предков длиной не более n. git-upload-pack рассматривает специальную глубину 2147483647 как бесконечную, даже если существует такая длинная цепочка предков.

--shallow-since=<date>

Углубите или сократите историю мелкого репозитория, чтобы включить все доступные коммиты после <дата>.

--shallow-exclude=<revision>

Углубите или сократите историю мелкого репозитория, чтобы исключить коммиты, доступные из указанной удаленной ветки или тега. Эту опцию можно указать несколько раз.

--deepen-relative

Аргумент --глубина указывает количество коммитов от текущей мелкой границы, а не от кончика истории каждой удаленной ветки.

--refetch

Пропускает согласование коммитов с сервером, чтобы получить все соответствующие объекты. Используйте для повторного применения нового фильтра BLOB-объектов/дерева частичного клонирования.

--no-progress

Не показывайте прогресс.

--check-self-contained-and-connected

Выведите "connectivity-ok" , если полученный пакет автономен и подключен.

-v

Run verbosely.

<repository>

URL-адрес удаленного репозитория.

<refs>…

Удаленные головки для обновления. Это относительно $GIT_DIR (например, "HEAD" , "refs/heads/master" ). Если не указано, обновляется со всех головок, имеющихся на удаленной стороне.

Если на пульте дистанционного управления включены опции uploadpack.allowTipSHA1InWant , uploadpack.allowReachableSHA1InWant или uploadpack.allowAnySHA1InWant , в качестве альтернативы они могут быть 40-шестнадцатеричными sha1, присутствующими на пульте дистанционного управления.
