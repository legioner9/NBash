git-grep

Name

git-grep — вывести строки, соответствующие шаблону

Synopsis

git grep [-a | --text] [-I] [--textconv] [-i | --игнорировать регистр] [-w | --word-regexp]
           [-v | --invert-match] [-h|-H] [--полное-имя]
           [-E | --extended-regexp] [-G | --basic-regexp]
           [-P | --perl-regexp]
           [-F | --fixed-strings] [-n | --номер-строки] [--столбец]
           [-l | --файлы-с-соответствиями] [-L | --files-without-match]
           [(-O | --open-files-in-pager) [<pager>]]
           [-z | --null]
           [ -o | --only-matching ] [-c | --count] [--все совпадения] [-q | --тихий]
           [--max-глубина <глубина>] [--[нет-]рекурсивный]
           [--color[=<когда>] | --no-цвет]
           [--break] [--heading] [-p | --show-функция]
           [-A <post-context>] [-B <pre-context>] [-C <context>]
           [-W | --function-context]
           [(-m | --max-count) <num>]
           [--threads <num>]
           [-f <file>] [-e] <pattern>
           [--and|--or|--not|(|)|-e <pattern>…​]
           [--recurse-submodules] [--parent-basename <базовое имя>]
           [ [--[no-]exclude-standard] [--cached | --no-индекс | --без отслеживания] | <дерево>…​]
           [--] [<pathspec>…​]

Description

Ищите указанные шаблоны в отслеживаемых файлах рабочего дерева, больших двоичных объектах, зарегистрированных в индексном файле, или больших двоичных объектах в заданных объектах дерева. Шаблоны представляют собой списки одного или нескольких поисковых выражений, разделенных символами новой строки. Пустая строка в качестве выражения поиска соответствует всем строкам.

Options

--cached

Вместо поиска отслеживаемых файлов в рабочем дереве поиск BLOB-объектов, зарегистрированных в индексном файле.

--no-index

Найдите файлы в текущем каталоге, которым не управляет Git.

--untracked

Помимо поиска в отслеживаемых файлах в рабочем дереве, ищите также и в неотслеживаемых файлах.

--no-exclude-standard

Также выполните поиск в игнорируемых файлах, не используя механизм .gitignore . Полезно только с --untracked .

--exclude-standard

Не обращайте внимания на игнорируемые файлы, указанные через механизм .gitignore . Полезно только при поиске файлов в текущем каталоге с помощью --no-index .

--recurse-submodules

Рекурсивный поиск в каждом активном и извлеченном подмодуле в репозитории. При использовании в сочетании с опцией <tree> префиксом всех выходных данных подмодуля будет имя объекта <tree> родительского проекта. Эта опция не имеет эффекта, если задан --no-index .

-a
--text

Обрабатывайте двоичные файлы так, как если бы они были текстовыми.

--textconv

Соблюдайте настройки фильтра textconv.

--no-textconv

Не соблюдайте настройки фильтра textconv. Это значение по умолчанию.

-i
--ignore-case

Игнорируйте различия в регистре между шаблонами и файлами.

-I

Не соответствует шаблону в двоичных файлах.

--max-depth <depth>

Для каждого <pathspec>, заданного в командной строке, спуститесь не более чем на <глубину> уровней каталогов. Значение -1 означает отсутствие ограничений. Этот параметр игнорируется, если <pathspec> содержит активные подстановочные знаки. Другими словами, если "a*" соответствует каталогу с именем "a*" , "*" сопоставляется буквально, поэтому --max-глубина по-прежнему эффективна.

-r
--recursive

То же, что --max-depth=-1 ; это значение по умолчанию.

--no-recursive

То же, что --max-depth=0 .

-w
--word-regexp

Сопоставление шаблона только на границе слова (либо начинается в начале строки, либо ему предшествует символ, не являющийся словом; заканчивается в конце строки, или за ним следует символ, не являющийся словом).

-v
--invert-match

Выберите несовпадающие строки.

-h
-H

По умолчанию команда показывает имя файла для каждого совпадения. Опция -h используется для подавления этого выхода. -H используется для полноты и не делает ничего, кроме переопределения -h , заданного ранее в командной строке.

--full-name

При запуске из подкаталога команда обычно выводит пути относительно текущего каталога. Эта опция заставляет выводить пути относительно верхнего каталога проекта.

-E
--extended-regexp
-G
--basic-regexp

Используйте расширенное/базовое регулярное выражение POSIX для шаблонов. По умолчанию используется базовое регулярное выражение.

-P
--perl-regexp

Используйте Perl-совместимые регулярные выражения для шаблонов.

Поддержка этих типов регулярных выражений является дополнительной зависимостью времени компиляции. Если Git не был скомпилирован с их поддержкой, предоставление этой опции приведет к его прекращению.

-F
--fixed-strings

Используйте фиксированные строки для шаблонов (не интерпретируйте шаблон как регулярное выражение).

-n
--line-number

Префикс номера строки для соответствующих строк.

--column

Префикс 1-индексированного смещения байтов первого совпадения от начала соответствующей строки.

-l
--files-with-matches
--name-only
-L
--files-without-match

Вместо того, чтобы показывать каждую совпавшую строку, покажите только имена файлов, которые содержат (или не содержат) совпадения. Для лучшей совместимости с git diff --name-only является синонимом --files-with-matches .

-O[<pager>]
--open-files-in-pager[=<pager>]

Откройте соответствующие файлы в пейджере (не вывод grep ). Если пейджером является "less" или "vi" и пользователь указал только один шаблон, первый файл автоматически размещается в первом совпадении. Аргумент pager не является обязательным; если указано, оно должно быть прикреплено к опции без пробела. Если pager не указан, будет использоваться пейджер по умолчанию (см. core.pager в git-config[1] ).

-z
--null

Используйте \0 в качестве разделителя имен путей в выходных данных и печатайте их дословно. Без этой опции имена путей с символами "unusual" заключаются в кавычки, как описано для переменной конфигурации core.quotePath (см. git-config[1] ).

-o
--only-matching

Печатайте только совпадающие (непустые) части совпадающей строки, причем каждая такая часть находится в отдельной выходной строке.

-c
--count

Вместо того, чтобы показывать каждую совпавшую строку, покажите количество совпадающих строк.

--color[=<when>]

Покажите цветные спички. Значение должно быть всегда (по умолчанию), никогда или автоматически.

--no-color

Отключите подсветку совпадений, даже если в файле конфигурации по умолчанию указан цветной вывод. То же, что --color=never .

--break

Выведите пустую строку между совпадениями из разных файлов.

--heading

Показывать имя файла над совпадениями в этом файле, а не в начале каждой отображаемой строки.

-p
--show-function

Покажите предыдущую строку, содержащую имя соответствующей функции, если только совпадающая строка не является именем функции. Имя определяется так же, как git diff обрабатывает заголовки фрагментов патчей (см. Defining a custom hunk-header в gitattributes[5] ).

-<num>
-C <num>
--context <num>

Покажите <num> начальную и конечную строки и поместите строку, содержащую -- , между смежными группами совпадений.

-A <num>
--after-context <num>

Покажите <num> завершающих строк и поместите строку, содержащую -- , между смежными группами совпадений.

-B <num>
--before-context <num>

Покажите <num> начальных строк и поместите строку, содержащую -- , между смежными группами совпадений.

-W
--function-context

Покажите окружающий текст от предыдущей строки, содержащей имя функции, до имени перед следующим именем функции, эффективно показывая всю функцию, в которой было найдено совпадение. Имена функций определяются так же, как git diff обрабатывает заголовки фрагментов патчей (см. Defining a custom hunk-header в gitattributes[5] ).

-m <num>
--max-count <num>

Ограничьте количество совпадений на файл. При использовании опции -v или --invert-match поиск прекращается после указанного количества несоответствий. Значение -1 вернет неограниченное количество результатов (по умолчанию). Значение 0 приведет к немедленному выходу с ненулевым статусом.

--threads <num>

Количество используемых рабочих потоков grep. См. grep.threads в CONFIGURATION для получения дополнительной информации.

-f <file>

Чтение шаблонов из <файла>, по одному в строке.

Передача шаблона через <file> позволяет предоставить шаблон поиска, содержащий \0.

Не все типы шаблонов поддерживают шаблоны, содержащие \0. Git выдаст ошибку, если данный тип шаблона не поддерживает такой шаблон. Тип шаблона --perl-regexp при компиляции с серверной частью PCRE v2 имеет самую широкую поддержку этих типов шаблонов.

В версиях Git до 2.23.0 шаблоны, содержащие \0, автоматически считались исправленными. Это никогда не было задокументировано, также были странные и недокументированные взаимодействия между, например, шаблонами, отличными от ASCII, содержащими \0 и --ignore-case .

В будущих версиях мы, возможно, научимся поддерживать шаблоны, содержащие \0, для большего количества серверов поиска, а до тех пор мы умрем, когда рассматриваемый тип шаблона не будет их поддерживать.

-e

Следующий параметр — шаблон. Эту опцию следует использовать для шаблонов, начинающихся с - , и в сценариях, передающих пользовательский ввод в grep. or объединяет несколько шаблонов.

--and
--or
--not
( …)

Укажите способ объединения нескольких шаблонов с помощью выражений Boolean . --or — оператор по умолчанию. --and имеет более высокий приоритет, чем --or . -e необходимо использовать для всех шаблонов.

--all-match

При предоставлении нескольких выражений шаблона в сочетании с --or этот флаг указывается, чтобы ограничить совпадение файлами, в которых есть строки, соответствующие всем из них.

-q
--quiet

Не выводить совпадающие строки; вместо этого выйдите со статусом 0, если совпадение есть, и с ненулевым статусом, если его нет.

<tree>…

Вместо поиска отслеживаемых файлов в рабочем дереве ищите объекты в заданных деревьях.

--

Сигнализирует об окончании вариантов; остальные параметры являются ограничителями <pathspec>.

<pathspec>…

Если задано, ограничьте поиск путями, соответствующими хотя бы одному шаблону. Поддерживаются как ведущие пути, так и шаблоны glob(7).

Дополнительные сведения о синтаксисе <pathspec> см. в записи pathspec в gitglossary[7] .

Examples

git grep 'time_t' -- '*.[ch]'

Ищет time_t во всех отслеживаемых файлах .c и .h в рабочем каталоге и его подкаталогах.

git grep -e '#define' --and \( -e MAX_PATH -e PATH_MAX \)

Ищет строку, содержащую #define и MAX_PATH или PATH_MAX .

git grep --all-match -e NODE -e Unexpected

Ищет строку, содержащую NODE или Unexpected , в файлах, в которых есть строки, соответствующие обеим.

git grep solution -- :^Documentation

Ищет solution , исключая файлы в Documentation .

Примечания к темам

Опция --threads (и конфигурация grep.threads) будут игнорироваться при использовании --open-files-in-pager , что приводит к однопоточному выполнению.

При поиске хранилища объектов (с помощью --cached или предоставлении объектов дерева) работа с несколькими потоками может выполняться медленнее, чем с однопоточным, если задан --textconv и выполняется слишком много преобразований текста. Поэтому, если в этом случае вы испытываете низкую производительность, возможно, будет желательно использовать --threads=1 .

Configuration

Все, что находится ниже этой строки в этом разделе, выборочно включено из документации git-config[1] . Содержание такое же, как и найденное там:

grep.lineNumber

Если установлено значение true, включите опцию -n по умолчанию.

grep.column

Если установлено значение true, включите опцию --column по умолчанию.

grep.patternType

Установите поведение сопоставления по умолчанию. Использование значения basic , extended , fixed или perl включит опцию --basic-regexp , --extended-regexp , --fixed-strings или --perl-regexp соответственно, а значение default будет использовать опцию grep.extendedRegexp для выбора между basic и extended .

grep.extendedRegexp

Если установлено значение true, включите опцию --extended-regexp по умолчанию. Этот параметр игнорируется, если для параметра grep.patternType установлено значение, отличное от default .

grep.threads

Количество используемых рабочих потоков grep. Если этот параметр не установлен (или установлен на 0), Git будет использовать столько потоков, сколько доступно логических ядер.

grep.fullName

Если установлено значение true, включите опцию --full-name по умолчанию.

grep.fallbackToNoIndex

Если установлено значение true, вернитесь к git grep --no-index, если git grep выполняется вне репозитория git . По умолчанию ложь.
