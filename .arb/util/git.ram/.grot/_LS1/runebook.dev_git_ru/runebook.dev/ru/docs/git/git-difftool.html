git-difftool

Name

git-difftool — Показать изменения с помощью обычных инструментов сравнения

Synopsis

git difftool [<options>] [<commit> [<commit>]] [--] [<path>…​]

Description

git difftool — это команда Git , которая позволяет сравнивать и редактировать файлы разных версий с помощью обычных инструментов сравнения. git difftool является интерфейсом git diff и принимает те же параметры и аргументы. См. git-diff[1] .

Options

-d
--dir-diff

Скопируйте измененные файлы во временное расположение и выполните для них разницу каталогов. В этом режиме никогда не появляется запрос перед запуском инструмента сравнения.

-y
--no-prompt

Не запрашивайте запрос перед запуском инструмента сравнения.

--prompt

Запрашивать перед каждым вызовом инструмента сравнения. Это поведение по умолчанию; предоставляется возможность переопределить любые параметры конфигурации.

--rotate-to=<file>

Начните показывать разницу для данного пути, пути до него перейдут в конец и выведутся.

--skip-to=<file>

Начните показывать разницу для данного пути, пропуская все пути перед ним.

-t <tool>
--tool=<tool>

Используйте инструмент сравнения, указанный в <tool>. Допустимые значения: «emerging», «kompare», «meld» и «vimdiff». Запустите git difftool --tool-help для получения списка допустимых настроек <tool>.

Если инструмент сравнения не указан, git difftool будет использовать переменную конфигурации diff.tool . Если переменная конфигурации diff.tool не установлена, git difftool выберет подходящее значение по умолчанию.

Вы можете явно указать полный путь к инструменту, задав переменную конфигурации difftool.<tool>.path . Например, вы можете настроить абсолютный путь к kdiff3, задав difftool.kdiff3.path . В противном случае git difftool предполагает, что инструмент доступен в PATH.

Вместо запуска одного из известных инструментов сравнения git difftool можно настроить для запуска альтернативной программы, указав командную строку для вызова в переменной конфигурации difftool.<tool>.cmd .

Когда git difftool вызывается с помощью этого инструмента (либо через параметр -t или --tool , либо через конфигурационную переменную diff.tool ), настроенная командная строка будет вызываться со следующими доступными переменными: $LOCAL устанавливается на имя временного файла, содержащего содержимое файла diff. предварительное изображение, а для $REMOTE установлено имя временного файла, содержащего содержимое пост-изображения различий. $MERGED — имя сравниваемого файла. $BASE предусмотрен для совместимости с пользовательскими командами инструмента слияния и имеет то же значение, что и $MERGED .

--tool-help

Распечатайте список инструментов сравнения, которые можно использовать с --tool .

--[no-]symlinks

Поведение git difftool по умолчанию заключается в создании символических ссылок на рабочее дерево при запуске в режиме --dir-diff , а правая часть сравнения дает то же содержимое, что и файл в рабочем дереве.

Если указать --no-symlinks , вместо этого git difftool будет создавать копии. --no-symlinks используется по умолчанию в Windows.

-x <command>
--extcmd=<command>

Укажите собственную команду для просмотра различий. git-difftool игнорирует настроенные значения по умолчанию и запускает $command $LOCAL $REMOTE , если указан этот параметр. Кроме того, в среде установлен $BASE .

-g
--[no-]gui

Когда git-difftool вызывается с опцией -g или --gui , инструмент сравнения по умолчанию будет считываться из настроенной переменной diff.guitool вместо diff.tool . Это можно выбрать автоматически с помощью переменной конфигурации difftool.guiDefault . Опцию --no-gui можно использовать для отмены этих настроек. Если diff.guitool не установлен, мы будем выполнять возврат в порядке merge.guitool , diff.tool , merge.tool , пока не будет найден инструмент.

--[no-]trust-exit-code

git-difftool вызывает инструмент сравнения индивидуально для каждого файла. Ошибки, о которых сообщает инструмент сравнения, по умолчанию игнорируются. Используйте --trust-exit-code , чтобы заставить git-difftool завершить работу, когда вызванный инструмент сравнения возвращает ненулевой код выхода.

git-difftool пересылает код завершения вызванного инструмента при использовании --trust-exit-code .

Полный список поддерживаемых опций см. в git-diff[1] .

Configuration

git difftool возвращается к переменным конфигурации git mergetool , если эквиваленты difftool не определены.

Все, что находится выше этой строки в этом разделе, не включено в документацию git-config[1] . Следующее содержимое такое же, как и найденное там:

diff.tool

Управляет тем, какой инструмент сравнения используется git-difftool[1] . Эта переменная переопределяет значение, настроенное в merge.tool . В списке ниже показаны допустимые встроенные значения. Любое другое значение рассматривается как пользовательский инструмент сравнения и требует определения соответствующей переменной difftool.<tool>.cmd.

diff.guitool

Определяет, какой инструмент сравнения будет использоваться git-difftool[1] , если указан флаг -g/--gui. Эта переменная переопределяет значение, настроенное в merge.guitool . В списке ниже показаны допустимые встроенные значения. Любое другое значение рассматривается как пользовательский инструмент сравнения и требует определения соответствующей переменной difftool.<guitool>.cmd.

difftool.<tool>.cmd

Укажите команду для вызова указанного инструмента сравнения. Указанная команда оценивается в оболочке со следующими доступными переменными: для LOCAL установлено имя временного файла, содержащего содержимое предварительного изображения различий, а для REMOTE установлено имя временного файла, содержащего содержимое сообщения сравнения. -изображение.

Дополнительную информацию см. в опции --tool=<tool> в git-difftool[1] .

difftool.<tool>.path

Переопределить путь для данного инструмента. Это полезно, если ваш инструмент отсутствует в PATH.

difftool.trustExitCode

Выйдите из difftool, если вызванный инструмент diff возвращает ненулевой статус выхода.

Дополнительную информацию см. в опции --trust-exit-code в git-difftool[1] .

difftool.prompt

Запрашивать перед каждым вызовом инструмента сравнения.

difftool.guiDefault

Установите true для использования diff.guitool по умолчанию (эквивалентно указанию аргумента --gui ) или auto для выбора diff.guitool или diff.tool в зависимости от наличия значения переменной среды DISPLAY . По умолчанию используется false , где аргумент --gui должен быть указан явно, чтобы можно было использовать diff.guitool .
