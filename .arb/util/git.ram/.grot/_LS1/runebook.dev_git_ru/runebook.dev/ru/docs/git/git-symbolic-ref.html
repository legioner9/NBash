git-symbolic-ref

Name

git-symbolic-ref — Чтение, изменение и удаление символических ссылок.

Synopsis

git symbolic-ref [-m <reason>] <name> <ref>
git symbolic-ref [-q] [--short] [--no-recurse] <name>
git symbolic-ref --delete [-q] <name>

Description

Учитывая один аргумент, считывает, на какую голову ветки ссылается данная символическая ссылка, и выводит ее путь относительно каталога .git/ . Обычно вы указываете HEAD в качестве аргумента <name>, чтобы узнать, в какой ветви находится ваше рабочее дерево.

Учитывая два аргумента, создает или обновляет символическую ссылку <name>, чтобы указать на данную ветвь <ref>.

Учитывая --delete и дополнительный аргумент, удаляет данную символическую ссылку.

Символическая ссылка — это обычный файл, в котором хранится строка, начинающаяся с ref: refs/ . Например, ваш .git/HEAD — это обычный файл с содержимым ref: refs/heads/master .

Options

-d
--delete

Удалите символическую ссылку <имя>.

-q
--quiet

Не выдавайте сообщение об ошибке, если <имя> является не символической ссылкой, а отдельным HEAD; вместо этого выйдите с ненулевым статусом молча.

--short

При отображении значения <name> в виде символической ссылки попробуйте сократить значение, например, с refs/heads/master на master .

--recurse
--no-recurse

При отображении значения <name> в качестве символической ссылки, если <name> ссылается на другую символическую ссылку, следуйте такой цепочке символических ссылок до тех пор, пока результат больше не будет указывать на символическую ссылку ( --recurse , значение по умолчанию). --no-recurse останавливается после разыменования только одного уровня символьной ссылки.

-m

Обновите журнал ссылок для <имя>, указав <причина>. Это действительно только при создании или обновлении символической ссылки.

Notes

Раньше .git/HEAD представлял собой символическую ссылку, указывающую на refs/heads/master . Когда мы хотели переключиться на другую ветку, мы делали ln -sf refs/heads/newbranch .git/HEAD , а когда мы хотели узнать, на какой ветке мы находимся, мы делали readlink .git/HEAD . Но символические ссылки не полностью переносимы, поэтому теперь они устарели, а символические ссылки (как описано выше) используются по умолчанию.

git symbolic-ref выйдет со статусом 0, если содержимое символической ссылки было напечатано правильно, со статусом 1, если запрошенное имя не является символической ссылкой, или со статусом 128, если произойдет другая ошибка.
