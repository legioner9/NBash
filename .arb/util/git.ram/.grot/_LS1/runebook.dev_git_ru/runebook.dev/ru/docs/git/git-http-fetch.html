git-http-fetch

Name

git-http-fetch — загрузка из удаленного репозитория Git через HTTP.

Synopsis

git http-fetch [-c] [-t] [-a] [-d] [-v] [-w <filename>] [--recover] [--stdin | --packfile=<hash> | <commit>] <URL>

Description

Загружает удаленный репозиторий Git через HTTP.

Эта команда всегда получает все объекты. Исторически существовало три варианта -a , -c и -t для выбора объектов для загрузки. Сейчас их молча игнорируют.

Options

commit-id

Либо хэш, либо имя файла в [URL]/refs/ для извлечения.

-действовать

Эти варианты игнорируются по историческим причинам.

-v

Сообщите о том, что скачано.

-w <filename>

Записывает идентификатор фиксации в указанное имя файла в $GIT_DIR/refs/<filename> на локальном конце после завершения передачи.

--stdin

Вместо идентификатора фиксации в командной строке (чего в данном случае не ожидается) git http-fetch ожидает строки на стандартном вводе в формате

<commit-id>['\t'<filename-as-in--w>]
--packfile=<hash>

Только для внутреннего использования. Вместо идентификатора фиксации в командной строке (чего в данном случае не ожидается), git http-fetch извлекает файл пакета непосредственно по заданному URL-адресу и использует index-pack для создания соответствующих файлов .idx и .keep. Хэш используется для определения имени временного файла и является произвольным. Вывод index-pack выводится на стандартный вывод. Требуется --index-pack-args.

--index-pack-args=<args>

Только для внутреннего использования. Команда для запуска содержимого загруженного пакета. Аргументы закодированы в URL-адресе и разделены пробелами.

--recover

Убедитесь, что все, что доступно от цели, получено. Используется после прерывания предыдущей выборки.
