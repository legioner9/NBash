git-merge-index

Name

git-merge-index — запустить слияние файлов, нуждающихся в слиянии

Synopsis

git merge-index [-o] [-q] <merge-program> (-a | ( [--] <file>…​) )

Description

При этом выполняется поиск <файла>(ов) в индексе и, если есть какие-либо записи слияния, передается хэш SHA-1 для этих файлов в качестве аргументов 1, 2, 3 (пустой аргумент, если файла нет) и <файл>. в качестве аргумента 4. Режимы файлов для трех файлов передаются в качестве аргументов 5, 6 и 7.

Options

--

Не интерпретируйте дополнительные аргументы как варианты.

-a

Запустите слияние для всех файлов в индексе, которые необходимо объединить.

-o

Вместо того, чтобы останавливаться на первом неудачном слиянии, выполните все действия за один раз — продолжайте слияние, даже если предыдущие слияния возвращали ошибки, и возвращайте код ошибки только после всех слияний.

-q

Не жалуйтесь на неудачную программу слияния (сбой программы слияния обычно указывает на конфликты во время слияния). Это для фарфора, который может захотеть отправлять собственные сообщения.

Если git merge-index вызывается с несколькими <file> (или -a), то он обрабатывает их по очереди, только останавливаясь, если слияние возвращает ненулевой код завершения.

Обычно это запускается с помощью сценария, вызывающего имитацию Git команды merge из пакета RCS.

В дистрибутив включен образец сценария под названием git merge-one-file .

ВНИМАНИЕ ВНИМАНИЕ ВНИМАНИЕ! Git "merge object order" отличается от порядка объектов слияния программы RCS merge . В приведенном выше порядке оригинал стоит первым. Но порядок аргументов программы трехстороннего слияния merge таков: оригинал находится посередине. Не спрашивайте меня, почему.

Examples:

torvalds@ppc970:~/merge-test> git merge-index cat MM
This is MM from the original tree.                # original
This is modified MM in the branch A.                # merge1
This is modified MM in the branch B.                # merge2
This is modified MM in the branch B.                # текущее содержимое

or

torvalds@ppc970:~/merge-test> git merge-index cat AA MM
cat: : No such file or directory
This is added AA in the branch A.
This is added AA in the branch B.
This is added AA in the branch B.
fatal: merge program failed

где последний пример показывает, как git merge-index прекратит попытки слияния, если что-то вернуло ошибку (т. е. cat вернул ошибку для файла AA, поскольку его не было в оригинале, и, следовательно, git merge-index даже не пытался слить вещь ММ).
