git-multi-pack-index

Name

git-multi-pack-index — писать и проверять мультипакетные индексы

Synopsis

git multi-pack-index [--object-dir=<dir>] [--[no-]bitmap] <sub-command>

Description

Запишите или проверьте файл мультипакетного индекса (MIDX).

Options

--object-dir=<dir>

Используйте данный каталог для размещения объектов Git . Мы проверяем <dir>/packs/multi-pack-index на наличие текущего MIDX-файла и <dir>/packs на наличие индексируемых пак-файлов.

<dir> должен быть альтернативой текущему репозиторию.

--[no-]progress

Включите/выключите прогресс явно. Если ни один из них не указан, прогресс отображается, если к терминалу подключена стандартная ошибка. Поддерживается подкомандами write , verify , expire и `repack.

Доступны следующие подкоманды:

write

Запишите новый файл MIDX. Для подкоманды write доступны следующие параметры:

--preferred-pack=<pack>

При необходимости укажите пакет разрешения конфликтов, используемый, когда несколько пакетов содержат один и тот же объект. <pack> должен содержать хотя бы один объект. Если не указано, ничья разрывается в пользу пакета с наименьшим временем mtime.

--[no-]bitmap

Контролируйте, будет ли записываться растровое изображение с несколькими упаковками.

--stdin-packs

Напишите индекс нескольких пакетов, содержащий только набор базовых имен индексов пакетов, разделенных строками, предоставленных через стандартный ввод.

--refs-snapshot=<path>

Для --bitmap дополнительно укажите файл, содержащий "refs snapshot" , снятый до переупаковки.

Эталонный снимок состоит из разделенных строками OID, соответствующих эталонным подсказкам, которые обычно создаются git repack перед созданием нового пакета. Строка может опционально начинаться с символа + , чтобы указать, что ссылкой, соответствующей этому OID, является "preferred" (см. pack.preferBitmapTips git-config[1] ).

Ожидается, что файл, указанный по адресу <path> , доступен для чтения и может содержать дубликаты. (Если данный OID указан более одного раза, он помечается как предпочтительный, если хотя бы один его экземпляр начинается со специального маркера + ).

verify

Проверьте содержимое файла MIDX.

expire

Удалите файлы пакетов, которые отслеживаются файлом MIDX, но не имеют объектов, на которые ссылается MIDX (за исключением пакетов .keep и пакетов Cruft). После этого перезапишите файл MIDX, чтобы удалить все ссылки на эти файлы пакета.

repack

Создайте новый пакетный файл, содержащий объекты в небольших пакетных файлах, на которые ссылается multi-pack-index. Если размер, заданный аргументом --batch-size=<size> , равен нулю, создайте пакет, содержащий все объекты, на которые ссылается индекс multi-pack. Для ненулевого размера пакета выберите файлы пакета, проверив пакеты от самых старых до самых новых, вычислив "expected size" путем подсчета количества объектов в пакете, на которые ссылается индекс нескольких пакетов, а затем разделите их на общее количество. объектов в пачке и умножьте на размер пачки. Мы выбираем пакеты с ожидаемым размером ниже размера пакета до тех пор, пока общий ожидаемый размер набора пакетов не будет равен размеру пакета, или пока не будут рассмотрены все файлы пакетов. Если выбран только один пак-файл, то ничего не делайте. Если создается новый пакетный файл, перепишите мультипакетный индекс, чтобы он ссылался на новый пакетный файл. При более позднем запуске git multi-pack-index expire пак-файлы, входившие в этот пакет, будут удалены.

Если repack.packKeptObjects — false , то любые файлы упаковки со связанным файлом .keep не будут выбраны для переупаковки пакета.

Examples

  • Запишите файл MIDX для файлов пакета в текущем каталоге .git .

    $ git multi-pack-index write

  • Запишите файл MIDX для файлов пакета в текущем каталоге .git с соответствующим растровым изображением.

    $ git multi-pack-index write --preferred-pack=<pack> --bitmap

  • Запишите файл MIDX для паковочных файлов в альтернативном хранилище объектов.

    $ git multi-pack-index --object-dir <alt> write

  • Проверьте файл MIDX на наличие паковочных файлов в текущем каталоге .git .

    $ git multi-pack-index verify

See also

См. The Multi-Pack-Index Design Document и gitformat-pack[5] для получения дополнительной информации о функции мультиупаковки-индекса и ее формате файла.
