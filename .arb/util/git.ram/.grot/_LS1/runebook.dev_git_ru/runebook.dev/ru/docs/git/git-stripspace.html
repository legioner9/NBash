git-stripspace

Name

git-stripspace — удалить ненужные пробелы

Synopsis

git stripspace [-s | --strip-comments]
git stripspace [-c | --comment-lines]

Description

Считайте текст, например сообщения о фиксации, заметки, теги и описания ветвей, из стандартного ввода и очистите его способом, используемым Git.

Без аргументов это будет:

  • удалить конечные пробелы из всех строк

  • свернуть несколько последовательных пустых строк в одну пустую строку

  • удалить пустые строки из начала и конца ввода

  • при необходимости добавьте недостающий \n в последнюю строку.

В случае, когда входные данные полностью состоят из пробельных символов, выходные данные не будут произведены.

ПРИМЕЧАНИЕ. Это предназначено для очистки метаданных. Предпочитайте режим --whitespace=fix вместо git-apply[1] для исправления пробелов в патчах или файлах в репозитории.

Options

-s
--strip-comments

Пропустите и удалите все строки, начинающиеся с символа комментария (по умолчанию # ).

-c
--comment-lines

Добавьте перед каждой строкой символ комментария и пробел. Строки автоматически завершаются новой строкой. В пустых строках будет добавлен только символ комментария.

Examples

Учитывая следующий шумный ввод с $ , указывающим конец строки:

|A brief introduction   $
|   $
|$
|A new paragraph$
|# with a commented-out line    $
|explaining lots of stuff.$
|$
|# Старый абзац, тоже закомментированный. $
|      $
|The end.$
|  $

Используйте git stripspace без аргументов, чтобы получить:

|A brief introduction$
|$
|A new paragraph$
|# с закомментированной строкой$
|explaining lots of stuff.$
|$
|# An old paragraph, also commented-out.$
|$
|The end.$

Используйте git stripspace --strip-comments , чтобы получить:

|A brief introduction$
|$
|A new paragraph$
|explaining lots of stuff.$
|$
|The end.$
