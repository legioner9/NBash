git-mailinfo

Name

git-mailinfo — извлекает патч и авторство из одного сообщения электронной почты.

Synopsis

git mailinfo [-k|-b] [-u | --encoding=<encoding> | -n]
               [--[no-]scissors] [--quoted-cr=<action>]
               <msg> <patch>

Description

Считывает одно сообщение электронной почты из стандартного ввода и записывает сообщение журнала фиксации в файл <msg>, а исправления - в файл <patch>. Имя автора, адрес электронной почты и тема электронного письма выводятся в стандартный вывод, который будет использоваться git am для создания фиксации. Обычно нет необходимости использовать эту команду напрямую. Вместо этого см. git-am[1] .

Options

-k

Обычно программа удаляет мусор электронной почты из строки заголовка «Тема:», чтобы извлечь строку заголовка сообщения журнала фиксации. Эта опция предотвращает подобные искажения и наиболее полезна при использовании для обратного считывания вывода git format-patch -k .

В частности, удаляются следующие, пока из них не останется none :

  • Ведущие и конечные пробелы.

  • Ведущие Re: , re: и : .

  • Ведущие строки в квадратных скобках (между [ и ] , обычно [PATCH] ).

Наконец, пробелы нормализуются до одного символа пробела ASCII.

-b

Если параметр -k не действует, все ведущие строки, заключенные в квадратные скобки парами [ и ] , удаляются. Эта опция ограничивает удаление только парами, строка в квадратных скобках которых содержит слово "PATCH" .

-u

Сообщение журнала фиксации, имя автора и адрес электронной почты автора берутся из электронного письма и после минимального декодирования передаточной кодировки MIME перекодируются в кодировку, указанную i18n.commitEncoding (по умолчанию UTF-8), путем транслитерации. Раньше это было необязательно, но теперь это значение по умолчанию.

Обратите внимание, что патч всегда используется как есть, без преобразования кодировки, даже с этим флагом.

--encoding=<encoding>

Похоже на -у. Но при перекодировании вместо указанной i18n.commitEncoding или UTF-8 используется указанная здесь кодировка.

-n

Отключите перекодирование всех кодировок метаданных.

-m
--message-id

Скопируйте заголовок Message-ID в конце сообщения фиксации. Это полезно для связи коммитов с обсуждениями в списке рассылки.

--scissors

Удалите все элементы корпуса перед ножницеобразной линией (например, "-- >8 --" ). Линия представляет собой ножницы и знаки перфорации и используется для того, чтобы попросить читателя разрезать сообщение на этой линии. Если эта строка появляется в теле сообщения перед патчем, все перед ней (включая саму линию ножниц) игнорируется при использовании этой опции.

Это полезно, если вы хотите начать свое сообщение в ветке обсуждения с комментариев и предложений по сообщению, на которое вы отвечаете, и завершить его отправкой исправления, разделив обсуждение и начало предлагаемого сообщения журнала фиксации ножницами. линия.

Это можно включить по умолчанию с помощью параметра конфигурации mailinfo.scissors.

--no-scissors

Не обращайте внимания на линии ножниц. Полезно для переопределения настроек mailinfo.scissors.

--quoted-cr=<action>

Действие, когда обрабатываются сообщения электронной почты, отправленные с кодировкой base64 или quote-printable, а декодированные строки заканчиваются CRLF вместо простого LF.

Допустимые действия:

  • nowarn : Git ничего не будет делать, если будет найден такой CRLF.

  • warn : Git выдаст предупреждение для каждого сообщения, если будет найден такой CRLF.

  • strip : Git преобразует CRLF в LF.

Действие по умолчанию можно установить с помощью параметра конфигурации mailinfo.quotedCR . Если такая опция конфигурации не установлена, будет использоваться warn .

<msg>

Сообщение журнала фиксации, извлеченное из электронного письма, обычно за исключением строки заголовка, которая поступает из темы электронного письма.

<patch>

Патч извлечен из электронной почты.

Configuration

Все, что находится ниже этой строки в этом разделе, выборочно включено из документации git-config[1] . Содержание такое же, как и найденное там:

mailinfo.scissors

Если это правда, git-mailinfo[1] (и, следовательно, git-am[1] ) действует по умолчанию так, как если бы в командной строке был указан параметр --scissors. Когда эта функция активна, эта функция удаляет все из тела сообщения до разделительной строки (т. е. состоящей в основном из ">8" , "8<" и "-" ).
