git-for-each-ref

Name

git-for-each-ref — Вывод информации по каждой ссылке

Synopsis

git for-each-ref [--count=<count>] [--shell|--perl|--python|--tcl]
                   [(--sort=<key>)…​] [--format=<format>]
                   [ --stdin | <pattern>…]
                   [--points-at=<object>]
                   [--merged[=<object>]] [--no-merged[=<object>]]
                   [--contains[=<object>]] [--no-contains[=<object>]]
                   [--exclude=<pattern> …​]

Description

Перебрать все ссылки, соответствующие <pattern> , и показать их в соответствии с заданным <format> , предварительно отсортировав их в соответствии с заданным набором <key> . Если задан <count> , остановитесь после отображения такого количества ссылок. Интерполированные значения в <format> могут быть дополнительно заключены в кавычки как строковые литералы на указанном основном языке, что позволяет осуществлять их прямую оценку на этом языке.

Options

<pattern>…

Если задан один или несколько шаблонов, отображаются только ссылки, соответствующие хотя бы одному шаблону, либо с помощью fnmatch(3), либо буквально, в последнем случае совпадение полностью или с начала до косой черты.

--stdin

Если указан --stdin , то список шаблонов считывается из стандартного ввода, а не из списка аргументов.

--count=<count>

По умолчанию команда показывает все ссылки, соответствующие <pattern> . Эта опция останавливает работу после показа такого количества ссылок.

--sort=<key>

Имя поля для сортировки. Префикс - для сортировки в порядке убывания значения. Если не указано, используется refname . Вы можете использовать опцию --sort=<key> несколько раз, и в этом случае последний ключ становится первичным ключом.

--format=<format>

Строка, которая интерполирует %(fieldname) на основе отображаемой ссылки и объекта, на который она указывает. Если перед fieldname стоит звездочка ( * ), а ссылка указывает на объект-тег, используйте значение поля в объекте, на которое ссылается объект-тег (вместо поля в объекте-теге). Если не указано, <format> по умолчанию имеет значение %(objectname) SPC %(objecttype) TAB %(refname) . Он также интерполирует %% в % и %xx , где xx — ​​шестнадцатеричные цифры, интерполируемые в символ с шестнадцатеричным кодом xx ; например, %00 интерполирует в \0 (NUL), %09 в \t (TAB) и %0a в \n (LF).

--color[=<when>]

Соблюдайте любые цвета, указанные в опции --format . Поле <when> должно быть одним из always , never или auto (если <when> отсутствует, ведите себя так, как если бы always был указан).

--shell
--perl
--python
--tcl

Если задано, строки, заменяющие заполнители %(fieldname) , заключаются в кавычки как строковые литералы, подходящие для указанного основного языка. Это предназначено для создания скриптлета, который можно напрямую `оценить`.

--points-at=<object>

Перечислите только ссылки, указывающие на данный объект.

--merged[=<object>]

Перечислите только ссылки, советы которых доступны из указанного коммита (HEAD, если не указано).

--no-merged[=<object>]

Перечислите только те ссылки, советы которых недоступны из указанного коммита (HEAD, если не указано).

--contains[=<object>]

Список только тех ссылок, которые содержат указанный коммит (HEAD, если не указан).

--no-contains[=<object>]

Перечислите только ссылки, которые не содержат указанный коммит (HEAD, если не указан).

--ignore-case

Сортировка и фильтрация ссылок нечувствительны к регистру.

--omit-empty

Не печатайте новую строку после форматированных ссылок, где формат расширяется до пустой строки.

--exclude=<pattern>

Если указан один или несколько шаблонов, отображаются только ссылки, которые не соответствуют ни одному исключенному шаблону(ам). Сопоставление выполняется по тем же правилам, что и <pattern> , описанному выше.

Field names

Различные значения из структурированных полей в объектах, на которые ссылаются, можно использовать для интерполяции в результирующий результат или в качестве ключей сортировки.

Для всех объектов могут использоваться следующие имена:

refname

Имя ссылки (часть после $GIT_DIR/). Для однозначного короткого имени ссылки добавьте :short . Опция core.warnAmbigousRefs используется для выбора строгого режима сокращений. Если добавляется lstrip=<N> ( rstrip=<N> ), компоненты пути <N> , разделенные косой чертой, удаляются из передней (задней) части имени ссылки (например, %(refname:lstrip=2) превращает refs/tags/foo в foo , а %(refname:rstrip=2) превращает refs/tags/foo в refs ). Если <N> — отрицательное число, удалите необходимое количество компонентов пути с указанного конца, чтобы оставить компоненты пути -<N> (например, %(refname:lstrip=-2) превращает refs/tags/foo в tags/foo , а %(refname:rstrip=-1) превращает refs/tags/foo в refs ). Если в ссылке недостаточно компонентов, результатом становится пустая строка при удалении с положительным <N> или полное имя ссылки при удалении с отрицательным <N>. И то и другое не является ошибкой.

strip можно использовать как синоним lstrip .

objecttype

Тип объекта ( blob , tree , commit , tag ).

objectsize

Размер объекта (такой же, как сообщает git cat-file -s ). Добавьте :disk , чтобы получить размер в байтах, который объект занимает на диске. См. примечание о размерах диска в разделе CAVEATS ниже.

objectname

Имя объекта (также известное как SHA-1). Для однозначного сокращения имени объекта добавьте :short . Для сокращения имени объекта желаемой длины добавьте :short=<length> , где минимальная длина — MINIMUM_ABBREV. Длина может быть превышена для обеспечения уникальности имен объектов.

deltabase

Это расширяется до имени объекта дельта-базы для данного объекта, если он хранится как дельта. В противном случае оно расширяется до имени объекта null (все нули).

upstream

Имя локальной ссылки, которую можно считать «вышестоящей» по отношению к отображаемой ссылке. Уважает :short , :lstrip и :rstrip так же, как refname выше. Кроме того, учитывается :track для отображения "[ahead N, behind M]" и :trackshort для отображения краткой версии: ">" (впереди), "<" (сзади), "<>" (впереди и сзади) или "=" (синхронно). :track также печатает "[gone]" всякий раз, когда встречается неизвестная восходящая ссылка. Добавьте :track,nobracket , чтобы отобразить информацию об отслеживании без скобок (т. е. "ahead N, behind M" ).

Для любой ветки удаленного отслеживания %(upstream) , %(upstream:remotename) и %(upstream:remoteref) относятся к имени удаленного устройства и имени отслеживаемого удаленного объекта соответственно. Другими словами, ветвь удаленного отслеживания можно обновить явно и индивидуально, используя refspec %(upstream:remoteref):%(upstream) для выборки из %(upstream:remotename) .

Не имеет никакого эффекта, если со ссылкой не связана информация об отслеживании. Все параметры, кроме nobracket , являются взаимоисключающими, но при совместном использовании выбирается последний вариант.

push

Имя локальной ссылки, которое представляет местоположение @{push} для отображаемой ссылки. Учитывает опции :short , :lstrip , :rstrip , :track , :trackshort , :remotename и :remoteref , как и upstream . Создает пустую строку, если ссылка @{push} не настроена.

HEAD

* , если HEAD соответствует текущей ссылке (выданной ветке), ' ' в противном случае.

color

Изменить цвет вывода. За ним следует :<colorname> , где названия цветов описаны в разделе «Значения» в разделе "CONFIGURATION FILE" документа git-config[1] . Например, %(color:bold red) .

align

Выровняйте содержимое по левому, среднему или правому краю между %(align:…​) и %(end). За "align:" следуют width=<width> и position=<position> в любом порядке, разделенные запятой, где <position> — левый, правый или средний, по умолчанию — левый, а <width> — общая длина контента с выравниванием. Для краткости префиксы "width=" и/или "position=" можно опустить и использовать вместо них голые <width> и <position>. Например, %(align:<width>,<position>) . Если длина содержимого больше ширины, выравнивание не выполняется. При использовании с --quote все, что находится между %(align:…​) и %(end), цитируется, но если оно вложенное, то кавычки выполняются только на самом верхнем уровне.

if

Используется как %(if)…​%(then)…​%(end) или %(if)…​%(then)…​%(else)…​%(end). Если после %(if) есть атом со значением или строковым литералом, то печатается всё, что находится после %(then), в противном случае, если используется атом %(else), то печатается всё, что находится после %(else). Мы игнорируем пробел при оценке строки перед %(then), это полезно, когда мы используем атом %(HEAD), который печатает либо "*" , либо " " , и мы хотим применить условие if только к ссылке HEAD . Добавьте ":equals=<string>" или ":notequals=<string>" , чтобы сравнить значение атомов %(if:…​) и %(then) с заданной строкой.

symref

Ссылка, на которую ссылается данная символическая ссылка. Если это не символическая ссылка, ничего не печатается. Уважает параметры :short , :lstrip и :rstrip так же, как refname выше.

signature

GPG-подпись коммита.

signature:grade

Показывать "G" для хорошей (действительной) подписи, "B" для плохой подписи, "U" для хорошей подписи с неизвестной достоверностью, "X" для хорошей подписи, срок действия которой истек, "Y" для хорошей подписи, сделанной ключом с истекшим сроком действия, "R" для хорошей подписи. сделано с помощью отозванного ключа, "E" , если подпись невозможно проверить (например, отсутствует ключ), и "N" , если подпись отсутствует.

signature:signer

Подписавший подпись GPG коммита.

signature:key

Ключ GPG-подписи коммита.

signature:fingerprint

Отпечаток GPG-подписи коммита.

signature:primarykeyfingerprint

Отпечаток первичного ключа подписи GPG коммита.

signature:trustlevel

Уровень доверия GPG-подписи коммита. Возможные выходы: ultimate , fully , marginal , never и undefined .

worktreepath

Абсолютный путь к рабочему дереву, из которого извлекается ссылка, если она извлечена в любом связанном рабочем дереве. В противном случае пустая строка.

ahead-behind:<committish>

Два целых числа, разделенных пробелом, демонстрирующие количество коммитов впереди и позади соответственно при сравнении выходной ссылки с <committish> , указанным в формате.

describe[:options]

Удобочитаемое имя, например git-describe[1] ; пустая строка для неописуемых коммитов. За строкой describe может следовать двоеточие и один или несколько параметров, разделенных запятыми.

tags=<bool-value>

Вместо того, чтобы рассматривать только аннотированные теги, рассмотрите также облегченные теги; подробности см. в соответствующей опции в git-describe[1] .

abbrev=<number>

Используйте не менее <число> шестнадцатеричных цифр; подробности см. в соответствующей опции в git-describe[1] .

match=<pattern>

Учитывайте только теги, соответствующие заданному шаблону glob(7) , за исключением префикса "refs/tags/" ; подробности см. в соответствующей опции в git-describe[1] .

exclude=<pattern>

Не рассматривать теги, соответствующие заданному шаблону glob(7) , за исключением префикса "refs/tags/" ; подробности см. в соответствующей опции в git-describe[1] .

В дополнение к вышесказанному для объектов фиксации и тега имена полей заголовка ( tree , parent , object , type и tag ) могут использоваться для указания значения в поле заголовка. Поля tree и parent также можно использовать с модификаторами :short и :short=<length> , как и objectname .

Для объектов фиксации и тегирования специальные поля creatordate и creator будут соответствовать соответствующей дате или кортежу имени-электронной почты-даты из полей committer или tagger в зависимости от типа объекта. Они предназначены для работы с аннотированными и облегченными тегами.

Поля, которые имеют в качестве значения кортеж name-email-date ( author , committer и tagger ), могут иметь суффиксы name , email и date для извлечения именованного компонента. Для полей электронной почты ( authoremail , committeremail и taggeremail ) можно добавить :trim , чтобы получить электронное письмо без угловых скобок, и :localpart , чтобы получить часть перед символом @ из обрезанного электронного письма. В дополнение к этому можно использовать опцию :mailmap и соответствующие :mailmap,trim и :mailmap,localpart (порядок не имеет значения) для получения значений имени и электронной почты в соответствии с файлом .mailmap или в соответствии с файлом, установленным в mailmap.file или mailmap. Переменная конфигурации .blob (см. gitmailmap[5] ).

Необработанные данные в объекте — raw .

raw:size

Размер необработанных данных объекта.

Обратите внимание, что --format=%(raw) нельзя использовать с --python , --shell , --tcl , поскольку такой язык может не поддерживать произвольные двоичные данные в типе строковой переменной.

Сообщение в коммите или объекте тега — contents , из которого contents:<part> можно использовать для извлечения различных частей:

contents:size

Размер сообщения фиксации или тега в байтах.

contents:subject

Первый абзац сообщения, который обычно представляет собой одну строку, принимается за "subject" сообщения фиксации или тега. Вместо contents:subject для получения тех же результатов можно использовать поле subject . :sanitize можно добавить к subject для строки темы, подходящей для имени файла.

contents:body

Оставшаяся часть фиксации или сообщение тега, следующее за "subject" .

contents:signature

Необязательная подпись тега GPG.

contents:lines=N

Первые строки сообщения N .

Кроме того, трейлеры, интерпретируемые git-interpret-trailers[1] , получаются как trailers[:options] (или с использованием исторического псевдонима contents:trailers[:options] ). Допустимые значения [:option] см. в разделе trailers документа git-log[1] .

В целях сортировки поля с числовыми значениями сортируются в числовом порядке ( objectsize , authordate , committerdate , creatordate , taggerdate ). Все остальные поля используются для сортировки в порядке их байтовых значений.

Также есть возможность сортировки по версиям, это можно сделать, используя имя поля version:refname или его псевдоним v:refname .

В любом случае имя поля, которое ссылается на поле, неприменимое к объекту, на который ссылается ссылка, не вызывает ошибки. Вместо этого он возвращает пустую строку.

В качестве особого случая для полей типа даты вы можете указать формат даты, добавив : , за которым следует имя формата даты (см. значения, которые принимает опция --date для git-rev-list[1] ).

Некоторые атомы, такие как %(align) и %(if), всегда требуют соответствующего %(end). Мы называем их "opening atoms" и иногда обозначаем %($open).

Когда действует кавычки, специфичные для языка сценариев, все между открывающим атомом верхнего уровня и соответствующим ему %(end) оценивается в соответствии с семантикой открывающего атома, и цитируется только его результат верхнего уровня.

Examples

Пример непосредственного создания форматированного текста. Показать последние 3 отмеченных коммита:

#!/bin/sh

git for-each-ref --count=3 --sort='-*authordate' \
--format='From: %(*authorname) %(*authoremail)
Subject: %(*subject)
Date: %(*authordate)
Ref: %(*refname)

%(*body)
' 'refs/tags'

Простой пример, показывающий использование оболочки eval на выходе, демонстрирующий использование --shell. Перечислите префиксы всех голов:

#!/bin/sh

git for-each-ref --shell --format="ref=%(refname)" refs/heads | \
while read entry
do
        eval "$entry"
        echo `dirname $ref`
done

Немного более подробный отчет по тегам, демонстрирующий, что формат может быть целым скриптом:

#!/bin/sh

fmt='
        r=%(refname)
        t=%(*objecttype)
        T=${r#refs/tags/}

        o=%(*objectname)
        n=%(*authorname)
        e=%(*authoremail)
        s=%(*subject)
        d=%(*authordate)
        b=%(*body)

        kind=Tag
        if test "z$t" = z
        then
                # could be a lightweight tag
                t=%(objecttype)
                kind="Lightweight tag"
                o=%(objectname)
                n=%(authorname)
                e=%(authoremail)
                s=%(subject)
                d=%(authordate)
                b=%(body)
        fi
        echo "$kind $T points at a $t object $o"
        if test "z$t" = zcommit
        then
                echo "The commit was authored by $n $e
at $d, and titled

    $s

Its message reads as:
"
                echo "$b" | sed -e "s/^/    /"
                echo
        fi
'

eval=`git for-each-ref --shell --format="$fmt" \
        --sort='*objecttype' \
        --sort=-taggerdate \
        refs/tags`
eval "$eval"

Пример, показывающий использование %(if)…​%(then)…​%(else)…​%(end). Это добавляет к текущей ветке звездочку.

git for-each-ref --format="%(if)%(HEAD)%(then)* %(else)  %(end)%(refname:short)" refs/heads/

Пример, показывающий использование %(if)…​%(then)…​%(end). При этом печатается имя автора, если оно присутствует.

git for-each-ref --format="%(refname)%(if)%(authorname)%(then) Authored by: %(authorname)%(end)"

Caveats

Обратите внимание, что размеры объектов на диске сообщаются точно, но следует проявлять осторожность, делая выводы о том, какие ссылки или объекты отвечают за использование диска. Размер упакованного не-дельта-объекта может быть намного больше, чем размер объектов, которые дельта-против него, но выбор, какой объект является базовым, а какой дельта, произволен и может быть изменен во время переупаковки.

Также обратите внимание, что в базе данных объектов может присутствовать несколько копий объекта; в этом случае не определено, о каком размере копии или дельта-базе будет сообщено.

Notes

При объединении нескольких фильтров --contains и --no-contains отображаются только ссылки, которые содержат хотя бы один из коммитов --contains и содержат none из коммитов --no-contains .

При объединении нескольких фильтров --merged и --no-merged отображаются только ссылки, доступные хотя бы из одного из коммитов --merged и из none из коммитов --no-merged .
