git-name-rev

Name

git-name-rev — Найти символические имена для заданных версий

Synopsis

git name-rev [--tags] [--refs=<pattern>]
               ( --all | --annotate-stdin | <commit-ish>…)

Description

Находит символические имена, подходящие для человеческого восприятия, для ревизий, заданных в любом формате, который может быть проанализирован git rev-parse .

Options

--tags

Не используйте имена веток, а только теги для именования коммитов.

--refs=<pattern>

Используйте только ссылки, имена которых соответствуют заданному шаблону оболочки. Шаблон может быть именем ветки, именем тега или полным именем ссылки. Если задано несколько раз, используйте ссылки, имена которых соответствуют любому из заданных шаблонов оболочки. Используйте --no-refs , чтобы удалить все предыдущие заданные шаблоны ссылок.

--exclude=<pattern>

Не используйте ссылки, имя которых соответствует заданному шаблону оболочки. Шаблон может быть одним из имени ветки, имени тега или полного имени ссылки. Если указано несколько раз, ссылка будет исключена, если она соответствует любому из заданных шаблонов. При использовании вместе с --refs ссылка будет использоваться как совпадение только в том случае, если она соответствует хотя бы одному шаблону --refs и не соответствует ни одному шаблону --exclude. Используйте --no-exclude , чтобы очистить список шаблонов исключения.

--all

Список всех коммитов, доступных из всех ссылок

--annotate-stdin

Преобразуйте стандартный ввод, заменив все 40-символьные шестнадцатеричные коды SHA-1 (скажем, $hex) на "$hex ($rev_name)" . При использовании с --name-only замените на "$rev_name" , полностью опуская $hex. В старых версиях Git эта опция называлась --stdin .

For example:

$ cat sample.txt

An abbreviated revision 2ae0a9cb82 will not be substituted.
The full name after substitution is 2ae0a9cb8298185a94e5998086f380a355dd8907,
while its tree object is 70d105cc79e63b81cfdcb08a15297c23e60b07ad

$ git name-rev --annotate-stdin <sample.txt

An abbreviated revision 2ae0a9cb82 will not be substituted.
The full name after substitution is 2ae0a9cb8298185a94e5998086f380a355dd8907 (master),
while its tree object is 70d105cc79e63b81cfdcb08a15297c23e60b07ad

$ git name-rev --name-only --annotate-stdin <sample.txt

An abbreviated revision 2ae0a9cb82 will not be substituted.
The full name after substitution is master,
while its tree object is 70d105cc79e63b81cfdcb08a15297c23e60b07ad
--name-only

Вместо того, чтобы печатать SHA-1 и имя, печатайте только имя. Если указано с --tags, обычный префикс тега "tags/" также опускается из имени, что более точно соответствует выводу git-describe .

--no-undefined

Умрет с кодом ошибки != 0, когда ссылка не определена, вместо печати undefined .

--always

Показывать уникальный сокращенный объект фиксации в качестве резервного варианта.

Examples

Учитывая коммит, выясните, где он находится относительно локальных ссылок. Допустим, кто-то написал вам об этом фантастическом коммите 33db5f4d9027a10e477ccf054b2c1ab94f74c85a. Конечно, вы просматриваете коммит, но это говорит вам только о том, что произошло, но не о контексте.

Введите git name-rev :

% git name-rev 33db5f4d9027a10e477ccf054b2c1ab94f74c85a
33db5f4d9027a10e477ccf054b2c1ab94f74c85a tags/v0.99~940

Теперь вы стали мудрее, потому что знаете, что это произошло за 940 ревизий до версии 0.99.

Еще одна приятная вещь, которую вы можете сделать, это:

% git log | git name-rev --annotate-stdin
