scalar

Name

скаляр — инструмент для управления большими репозиториями Git .

Synopsis

scalar clone [--single-branch] [--branch <main-branch>] [--full-clone]
        [--[no-]src] <url> [<enlistment>]
scalar list
scalar register [<enlistment>]
scalar unregister [<enlistment>]
scalar run ( all | config | commit-graph | fetch | loose-objects | pack-files ) [<enlistment>]
scalar reconfigure [ --all | <enlistment> ]
scalar diagnose [<enlistment>]
scalar delete <enlistment>

Description

Scalar — это инструмент управления репозиториями, который оптимизирует Git для использования в больших репозиториях. Scalar повышает производительность за счет настройки расширенных параметров Git , поддержки репозиториев в фоновом режиме и помогает сократить объем данных, передаваемых по сети.

Важной концепцией скаляров является привязка: это каталог верхнего уровня проекта. Обычно он содержит подкаталог src/ , который представляет собой рабочее дерево Git . Это способствует разделению отслеживаемых файлов (внутри src/ ) и неотслеживаемых файлов, таких как артефакты сборки (вне src/ ). При регистрации существующего рабочего дерева Git в Scalar, имя которого отличается от src , присоединение будет идентично рабочему дереву.

Команда scalar реализует различные подкоманды и разные параметры в зависимости от подкоманды. За исключением clone , list и reconfigure --all , все подкоманды предполагается запускать в списке.

Следующие параметры могут быть указаны в подкоманде before :

-C <directory>

Перед запуском подкоманды измените рабочий каталог. Этот вариант имитирует аналогичный вариант git[1] .

-c <key>=<value>

На время выполнения указанной подкоманды настройте этот параметр. Этот вариант имитирует аналогичный вариант git[1] .

Commands

Clone

клонировать [<опции>] <url> [<присоединение>]

Клонирует указанный репозиторий, аналогично git-clone[1] . По умолчанию клонируются только объекты фиксации и дерева. После завершения рабочее дерево будет расположено по адресу <enlistment>/src .

Функция разреженной проверки включена (кроме случаев запуска с --full-clone ), и присутствуют только файлы в каталоге верхнего уровня. Используйте git sparse-checkout set , чтобы расширить набор каталогов, которые вы хотите просмотреть, или git sparse-checkout disable , чтобы расширить его до всех файлов (более подробную информацию см. в git-sparse-checkout[1] ). Вы можете исследовать подкаталоги за пределами вашей разреженной проверки, используя git ls-tree HEAD[:<directory>] .

-b <name>
--branch <name>

Вместо проверки ветки, на которую указывает HEAD клонированного репозитория, проверьте ветку <name> .

--[no-]single-branch

Клонируйте только историю, ведущую к вершине одной ветки, указанной либо опцией --branch , либо на которую указывает HEAD удаленной основной ветки.

Дальнейшие выборки в результирующий репозиторий будут обновлять только ветку удаленного отслеживания для той ветки, которая использовалась для первоначального клонирования. Если HEAD на удаленном компьютере не указывал ни на одну ветку при создании клона --single-branch , ветка удаленного отслеживания не создается.

--[no-]src

По умолчанию scalar clone помещает клонированный репозиторий в каталог <entlistment>/src . Используйте --no-src , чтобы поместить клонированный репозиторий непосредственно в каталог <enlistment> .

--[no-]full-clone

По умолчанию инициализируется разреженная проверка. Такое поведение можно отключить через --full-clone .

List

list

Список списков, которые в настоящее время зарегистрированы Scalar. Эту подкоманду не обязательно выполнять внутри списка.

Register

register [<enlistment>]

Добавляет репозиторий включения в список зарегистрированных репозиториев и запускает фоновое обслуживание. Если <enlistment> не указан, регистрируется список, связанный с текущим рабочим каталогом.

Примечание. Когда эта подкоманда вызывается в рабочем дереве с именем src/ , его родительский каталог считается скалярным списком. Если рабочее дерево not называется src/ , оно само будет считаться скалярным набором.

Unregister

unregister [<enlistment>]

Удалите указанный репозиторий из списка репозиториев, зарегистрированных в Scalar, и остановите плановое фоновое обслуживание.

Run

скалярный запуск (все | конфигурация | график фиксации | выборка | свободные объекты | пакетные файлы) [<enlistment>]

Запустите данную задачу обслуживания (или все задачи, если был указан all ). За исключением all и config , эта подкоманда просто передается git-maintenance[1] (сопоставляет fetch с prefetch и pack-files с incremental-repack ).

Эти задачи запускаются автоматически в рамках планового обслуживания, как только репозиторий регистрируется в Scalar. Поэтому нет необходимости запускать эту подкоманду вручную.

Задача config специфична для Scalar и настраивает все эти самоуверенные настройки по умолчанию, которые позволяют Git работать более эффективно с большими репозиториями. Поскольку эта задача запускается автоматически как часть scalar clone , явный вызов этой задачи требуется редко.

Reconfigure

После обновления Scalar или когда конфигурация набора Scalar была каким-то образом повреждена или изменена по ошибке, эта подкоманда позволяет перенастроить список.

При использовании опции --all все списки, зарегистрированные в настоящее время в Scalar, будут переконфигурированы. Используйте эту опцию после каждого обновления Scalar.

Diagnose

diagnose [<enlistment>]

При сообщении о проблемах со Scalar часто бывает полезно предоставить информацию, собранную этой командой, включая журналы и определенную статистику, описывающую форму данных текущего списка.

Результатом этой команды является файл .zip , который записывается в каталог, расположенный рядом с рабочим деревом в каталоге src .

Delete

delete <enlistment>

Эта подкоманда позволяет удалить существующий список Scalar из вашей локальной файловой системы, отменив регистрацию репозитория.
