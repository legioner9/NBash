git-diagnose

Name

git-diagnose — создать zip-архив с диагностической информацией.

Synopsis

git diagnose [(-o | --output-directory) <path>] [(-s | --suffix) <format>]
               [--mode=<mode>]

Description

Собирает подробную информацию о компьютере пользователя, клиенте Git и состоянии репозитория и упаковывает эту информацию в zip-архив. Сгенерированный архив затем можно, например, передать в список рассылки Git , чтобы помочь в устранении проблемы или использовать в качестве справочного материала для независимой отладки.

По умолчанию в архив фиксируется следующая информация:

  • git version --build-options

  • Путь к корню репозитория

  • Доступное дисковое пространство в файловой системе

  • Имя и размер каждого паковочного файла, включая файлы в альтернативных хранилищах объектов.

  • Общее количество незакрепленных объектов, а также количество с разбивкой по подкаталогу .git/objects .

Дополнительную информацию можно собрать, выбрав другой режим диагностики с помощью опции --mode .

Этот инструмент отличается от git-bugreport[1] тем, что он собирает гораздо более подробную информацию, уделяя больше внимания сообщению о размере и форме данных содержимого репозитория.

Options

-o <path>
--output-directory <path>

Поместите полученный архив диагностики в <path> вместо текущего каталога.

-s <format>
--suffix <format>

Укажите альтернативный суффикс имени архива диагностики, чтобы создать файл с именем git-diagnostics-<formatted suffix> . Это должно иметь форму строки формата strftime(3); будет использоваться текущее местное время.

--mode=(stats|all)

Укажите тип диагностики, которую необходимо собрать. Поведение git diagnose по умолчанию эквивалентно --mode=stats .

Опция --mode=all собирает все, что включено в --mode=stats , а также копии каталогов .git , .git/hooks , .git/info , .git/logs и .git/objects/info . Эта дополнительная информация может быть конфиденциальной, поскольку ее можно использовать для восстановления полного содержимого диагностированного репозитория. Пользователи должны проявлять осторожность при совместном использовании архива, созданного с помощью --mode=all .
