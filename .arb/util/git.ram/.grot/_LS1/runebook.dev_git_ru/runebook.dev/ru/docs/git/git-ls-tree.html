git-ls-tree

Name

git-ls-tree — вывести список содержимого объекта дерева

Synopsis

git ls-tree [-d] [-r] [-t] [-l] [-z]
            [--name-only] [--name-status] [--object-only] [--full-name] [--full-tree] [--abbrev[=<n>]] [--format=<format>]
            <tree-ish> [<path>…​]

Description

Перечисляет содержимое данного объекта дерева, аналогично тому, что "/bin/ls -a" делает в текущем рабочем каталоге. Обратите внимание, что:

  • поведение немного отличается от поведения "/bin/ls" тем, что <path> обозначает просто список шаблонов для сопоставления, например, поэтому указание имени каталога (без -r ) будет вести себя по-другому, и порядок аргументов не имеет значения.

  • поведение аналогично поведению "/bin/ls" в том смысле, что <path> используется относительно текущего рабочего каталога. Например, когда вы находитесь в каталоге sub , в котором есть каталог dir , вы можете запустить git ls-tree -r HEAD dir , чтобы просмотреть содержимое дерева (то есть sub/dir в HEAD ). В этом случае вы не хотите предоставлять дерево, которое не находится на корневом уровне (например, git ls-tree -r HEAD:sub dir ), так как это приведет к запросу sub/sub/dir в коммите HEAD . Однако текущий рабочий каталог можно игнорировать, передав параметр --full-tree.

Options

<tree-ish>

Идентификатор дерева.

-d

Показывать только саму запись дерева, а не ее дочерние элементы.

-r

Рекурсия на поддеревья.

-t

Показывать записи дерева даже при их рекурсивном использовании. Не имеет эффекта, если не был принят -r . -d подразумевает -t .

-l
--long

Показать размер объекта записей blob (файла).

-z

\0 завершение строки при выводе и не заключайте имена файлов в кавычки. Для получения дополнительной информации см. ФОРМАТ ВЫХОДА ниже.

--name-only
--name-status

Укажите только имена файлов (вместо вывода "long" ), по одному в строке. Невозможно объединить с --object-only .

--object-only

Укажите только имена объектов, по одному в строке. Невозможно объединить с --name-only или --name-status . Это эквивалентно указанию --format='%(objectname)' , но как для этой опции, так и для этого конкретного формата команда использует оптимизированный вручную кодовый путь вместо использования общего механизма форматирования.

--abbrev[=<n>]

Вместо отображения полных 40-байтовых строк шестнадцатеричного объекта покажите самый короткий префикс длиной не менее шестнадцатеричных цифр <n> , который однозначно ссылается на объект. Нестандартное количество цифр можно указать с помощью --abbrev=<n>.

--full-name

Вместо отображения имен путей относительно текущего рабочего каталога покажите полные имена путей.

--full-tree

Не ограничивайте список текущим рабочим каталогом. Подразумевается --полное имя.

--format=<format>

Строка, которая интерполирует %(fieldname) на основе отображаемого результата. Он также интерполирует %% в % и %xNN , где NN — шестнадцатеричные цифры, интерполируемые в символ с шестнадцатеричным кодом NN ; например, %x00 интерполирует в \0 (NUL), %x09 в \t (TAB) и %x0a в \n (LF). Если указано, --format нельзя комбинировать с другими опциями изменения формата, включая --long , --name-only и --object-only .

[<path>…​]

Когда пути указаны, покажите их (обратите внимание, что на самом деле это не необработанные имена путей, а скорее список шаблонов, которым нужно соответствовать). В противном случае неявно используется корневой уровень дерева в качестве единственного аргумента пути.

Output format

Выходной формат ls-tree определяется либо опцией --format , либо другими опциями изменения формата, такими как --name-only и т. д. (см. --format выше).

Использование определенных директив --format эквивалентно использованию этих параметров, но вызов полного механизма форматирования может быть медленнее, чем использование соответствующего параметра форматирования.

В тех случаях, когда --format точно соответствует существующей опции, ls-tree будет использовать соответствующий более быстрый путь. Таким образом, формат по умолчанию эквивалентен:

%(objectmode) %(objecttype) %(objectname)%x09%(path)

Этот выходной формат совместим с тем, что ожидает --index-info --stdin или git update-index .

При использовании опции -l формат меняется на

%(objectmode) %(objecttype) %(objectname) %(objectsize:padded)%x09%(path)

Размер объекта, идентифицируемый <objectname>, указывается в байтах и ​​выравнивается по правому краю с минимальной шириной 7 символов. Размер объекта указывается только для записей больших двоичных объектов (файлов); для других записей вместо размера используется символ - .

Без опции -z имена путей с символами "unusual" заключаются в кавычки, как описано для переменной конфигурации core.quotePath (см. git-config[1] ). При использовании -z имя файла выводится дословно, а строка завершается нулевым байтом.

Customized format:

Можно печатать в пользовательском формате, используя опцию --format , которая способна интерполировать различные поля с использованием нотации %(fieldname) . Например, если вас интересуют только поля "objectname" и "path" , вы можете выполнить с конкретным "--format" , например

git ls-tree --format='%(objectname) %(path)' <tree-ish>

Field names

Для интерполяции в результирующий результат можно использовать различные значения из структурированных полей. Для каждой выводящей строки можно использовать следующие имена:

objectmode

Режим объекта.

objecttype

Тип объекта ( commit , blob или tree ).

objectname

Имя объекта.

objectsize[:padded]

Размер объекта blob ( "-" , если это commit или tree ). Он также поддерживает дополненный формат размера "%(objectsize:padded)" .

path

Путь к объекту.
