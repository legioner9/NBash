git-mktag

Name

git-mktag — Создает объект тега с дополнительной проверкой.

Synopsis

git mktag

Description

Считывает содержимое тега со стандартного ввода и создает объект тега. Результатом является идентификатор <object> нового тега.

Эта команда в основном эквивалентна git-hash-object[1] , вызываемой с помощью -t tag -w --stdin . То есть оба они создадут и запишут тег, найденный в my-tag :

git mktag <my-tag
git hash-object -t tag -w --stdin <my-tag

Разница в том, что mktag умрет до записи тега, если тег не пройдет проверку git-fsck[1] .

Проверка "fsck" , выполняемая mktag, является более строгой, чем проверка git-fsck[1] по умолчанию, поскольку все сообщения fsck.<msg-id> преобразуются из предупреждений в ошибки (например, отсутствие строки "tagger" является ошибкой).

Дополнительные заголовки в объекте также являются ошибкой mktag, но игнорируются git-fsck[1] . Эту дополнительную проверку можно отключить, установив соответствующую переменную fsck.<msg-id> :

git -c fsck.extraHeaderEntry=ignore mktag <my-tag-with-headers

Options

--strict

По умолчанию mktag включает режим, эквивалентный git-fsck[1] --strict . Используйте --no-strict , чтобы отключить его.

Tag format

Файл подписи тега, подаваемый на стандартный ввод этой команды, имеет очень простой фиксированный формат: четыре строки

object <hash>
type <typename>
tag <tagname>
tagger <tagger>

за которым следует сообщение произвольной формы optional (некоторые теги, созданные более старым Git , могут не иметь строки tagger ). Сообщение, если оно существует, отделяется от заголовка пустой строкой. Часть сообщения может содержать подпись, которая не важна для самого Git , но ее можно проверить с помощью gpg.
