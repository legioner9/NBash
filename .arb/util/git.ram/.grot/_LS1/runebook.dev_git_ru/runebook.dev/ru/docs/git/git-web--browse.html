git-web--browse

Name

git-web—browse — вспомогательный скрипт Git для запуска веб-браузера.

Synopsis

git web--browse [<options>] (<URL>|<file>)…​

Description

Этот сценарий пытается, насколько это возможно, отобразить URL-адреса и ФАЙЛЫ, передаваемые в качестве аргументов, в виде HTML-страниц на новых вкладках уже открытого веб-браузера.

В настоящее время поддерживаются следующие браузеры (или команды):

  • Firefox (это значение по умолчанию в X Window, если не используется KDE)

  • iceweasel

  • seamonkey

  • iceape

  • хром (также поддерживается как браузер хрома)

  • google-chrome (также поддерживается как chrome)

  • konqueror (это значение по умолчанию в KDE, см. Note about konqueror ниже)

  • opera

  • w3m (это внешняя графическая среда по умолчанию)

  • elinks

  • links

  • lynx

  • dillo

  • открыть (это значение по умолчанию в графическом интерфейсе Mac OS X)

  • start (это значение по умолчанию в MinGW)

  • cygstart (это значение по умолчанию в Cygwin)

  • xdg-open

Также могут быть указаны пользовательские команды.

Options

-b <browser>
--browser=<browser>

Используйте указанный браузер. Он должен быть в списке поддерживаемых браузеров.

-t <browser>
--tool=<browser>

То же, что и выше.

-c <conf.var>
--config=<conf.var>

CONF.VAR ищется в файлах конфигурации Git . Если он установлен, то его значение указывает браузер, который следует использовать.

Configuration variables

CONF.VAR (из опции -c) и web.browser

Веб-браузер можно указать с помощью переменной конфигурации, передаваемой с параметром командной строки -c (или --config), или переменной конфигурации web.browser , если первая не используется.

browser.<tool>.path

Вы можете явно указать полный путь к предпочитаемому вами браузеру, задав переменную конфигурации browser.<tool>.path . Например, вы можете настроить абсолютный путь к Firefox, установив browser.firefox.path . В противном случае git web--browse предполагает, что инструмент доступен в PATH.

browser.<tool>.cmd

Если браузер, указанный в параметрах или переменных конфигурации, не входит в число поддерживаемых, будет найдена соответствующая переменная конфигурации browser.<tool>.cmd . Если эта переменная существует, git web--browse будет рассматривать указанный инструмент как пользовательскую команду и будет использовать команду eval для запуска команды с URL-адресами, переданными в качестве аргументов.

Примечание о победителе

Когда konqueror указан параметром командной строки или переменной конфигурации, мы запускаем kfmclient , чтобы попытаться открыть HTML-страницу руководства уже открытого konqueror на новой вкладке, если это возможно.

Для единообразия мы также попробуем такой трюк, если для browser.konqueror.path установлено что-то вроде A_PATH_TO/konqueror . Это значит, что вместо него мы попробуем запустить A_PATH_TO/kfmclient .

Если вы действительно хотите использовать konqueror , вы можете использовать что-то вроде следующего:

        [web]
                browser = konq

        [browser "konq"]
                cmd = A_PATH_TO/konqueror

Примечание о git-config --global

Обратите внимание, что эти переменные конфигурации, вероятно, следует устанавливать с помощью флага --global , например так:

$ git config --global web.browser Firefox

поскольку они, вероятно, более специфичны для пользователя, чем для репозитория. См. git-config[1] для получения дополнительной информации об этом.
