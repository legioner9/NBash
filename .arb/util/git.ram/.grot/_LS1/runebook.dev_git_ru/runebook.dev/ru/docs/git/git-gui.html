git-gui

Name

git-gui — портативный графический интерфейс для Git .

Synopsis

git gui [<command>] [<arguments>]

Description

Графический пользовательский интерфейс для Git на основе Tcl/Tk. git gui позволяет пользователям вносить изменения в свой репозиторий, делая новые коммиты, изменяя существующие, создавая ветки, выполняя локальные слияния и загружая/передавая в удаленные репозитории.

В отличие от gitk , git gui фокусируется на создании коммитов и аннотациях отдельных файлов и не показывает историю проекта. Однако он предоставляет действия меню для запуска сеанса gitk изнутри git gui .

Известно, что git gui работает во всех популярных системах UNIX, Mac OS X и Windows (как под Cygwin, так и под MSYS). Насколько это возможно, соблюдаются рекомендации по пользовательскому интерфейсу, специфичные для ОС, что делает git gui довольно естественным интерфейсом для пользователей.

Commands

blame

Запустите программу просмотра файлов указанной версии (или рабочего каталога, если не указан).

browser

Запустите браузер дерева, отображающий все файлы в указанном коммите. Файлы, выбранные через браузер, открываются в средстве просмотра обвинений.

citool

Запустите git gui и сделайте ровно одну фиксацию перед выходом и возвратом в оболочку. Интерфейс ограничен только фиксацией действий, что немного сокращает время запуска приложения и упрощает строку меню.

version

Отображение текущей версии git gui .

Examples

git gui blame Makefile

Покажите содержимое файла Makefile в текущем рабочем каталоге и предоставьте аннотации как для исходного автора каждой строки, так и для того, кто переместил строку в ее текущее местоположение. Незафиксированный файл аннотируется, а незафиксированные изменения (если таковые имеются) явно приписываются Not Yet Committed .

git gui blame v0.99.8 Makefile

Покажите содержимое Makefile в версии v0.99.8 и предоставьте аннотации для каждой строки. В отличие от приведенного выше примера, файл читается из базы данных объектов, а не из рабочего каталога.

git gui blame --line=100 Makefile

Загружает аннотации, как описано выше, и автоматически прокручивает представление до центра строки 100 .

git gui citool

Сделайте один коммит и вернитесь в оболочку, когда он будет завершен. Эта команда возвращает ненулевой код выхода, если окно было закрыто каким-либо способом, кроме фиксации.

git gui citool --amend

Автоматически входит в режим интерфейса Amend Last Commit .

git gui citool --nocommit

Ведите себя как обычный citool, но вместо фиксации просто завершите работу с нулевым кодом выхода. Он по-прежнему проверяет, что индекс не содержит несвязанных записей, поэтому вы можете использовать его как версию git-mergetool[1] с графическим интерфейсом.

git citool

То же, что git gui citool (выше).

git gui browser maint

Показать браузер дерева ветки maint . Файлы, выбранные в браузере, можно просмотреть с помощью внутреннего средства просмотра обвинений.

See also

gitk[1]

Браузер репозитория Git . Показывает ветки, историю коммитов и различия в файлах. gitk — это утилита, запускаемая действиями визуализации репозитория git gui .

Other

git gui фактически поддерживается как независимый проект, но для удобства конечных пользователей стабильные версии распространяются как часть пакета Git .

Официальный репозиторий проекта git gui можно найти по адресу:
