git-remote-ext

Name

git-remote-ext — мост интеллектуального транспорта к внешней команде.

Synopsis

git remote add <nick> "ext::<command>[ <arguments>…​]"

Description

Этот удаленный помощник использует указанный <command> для подключения к удаленному серверу Git .

Предполагается, что данные, записанные в стандартный ввод указанного <command> , отправляются на сервер git://, git-upload-pack, git-receive-pack или git-upload-archive (в зависимости от ситуации), а данные читаются из стандартного вывода Предполагается, что <команда> получена от той же службы.

Команда и аргументы разделяются неэкранированным пробелом.

Следующие последовательности имеют особое значение:

'% '

Буквальное пространство в команде или аргументе.

%%

Буквальный знак процента.

%s

Заменяется именем (receive-pack, upload-pack или upload-archive) службы, которую Git хочет вызвать.

%S

Заменяется длинным именем (git-receive-pack, git-upload-pack или git-upload-archive) службы, которую Git хочет вызвать.

%G (должны быть первыми символами аргумента)

Этот аргумент не будет передан <command> . Вместо этого помощник запустится с отправки запросов на обслуживание git:// на удаленную сторону, при этом для поля службы будет установлено соответствующее значение, а для поля репозитория — остальная часть аргумента. По умолчанию такой запрос не отправляется.

Это полезно, если удаленная сторона — это сервер git://, доступ к которому осуществляется через какой-либо туннель.

%V (должны быть первыми символами аргумента)

Этот аргумент не будет передан <command> . Вместо этого он устанавливает поле vhost в запросе службы git:// (остальную часть аргумента). По умолчанию vhost не отправляется в таком запросе (если он отправлен).

Environment variables

GIT_TRANSLOOP_DEBUG

Если установлено, выводит отладочную информацию о различных операциях чтения/записи.

Переменные среды, передаваемые команде

GIT_EXT_SERVICE

Установите длинное имя (git-upload-pack и т. д.) сервисного помощника, который необходимо вызвать.

GIT_EXT_SERVICE_NOPREFIX

Установите длинное имя (upload-pack и т. д.) службы, которую необходимо вызвать.

Examples

Этот удаленный помощник прозрачно используется Git при использовании таких команд, как "git fetch <URL>" , "git clone <URL>" , , "git push <URL>" или "git remote add <nick> <URL>" , где <URL> начинается с ext:: . Примеры:

"ext::ssh -i /home/foo/.ssh/somekey [email protected] %S foo/repo"

Аналогично хосту.example:foo/repo, но используйте /home/foo/.ssh/somekey в качестве пары ключей и пользователя в качестве пользователя на удаленной стороне. Это позволяет избежать необходимости редактировать .ssh/config.

"ext::socat -t3600 - ABSTRACT-CONNECT:/git-server %G/somerepo"

Представляет репозиторий с путем /somerepo, доступным по протоколу git по адресу абстрактного пространства имен / git -server.

"ext::git-server-alias foo %G/repo"

Представляет репозиторий с путем /repo, доступ к которому осуществляется с помощью вспомогательной программы "git-server-alias foo" . Путь к репозиторию и тип запроса передаются не в командной строке, а как часть потока протокола, как обычно в протоколе git://.

"ext::git-server-alias foo %G/repo %Vfoo"

Представляет репозиторий с путем /repo, доступ к которому осуществляется с помощью вспомогательной программы "git-server-alias foo" . Имя хоста удаленного сервера, передаваемое в потоке протокола, будет "foo" (это позволяет нескольким виртуальным серверам Git совместно использовать адрес канального уровня).

"ext::git-server-alias foo %G/repo% with% spaces %Vfoo"

Представляет репозиторий с путем /repo with spaces , доступ к которому осуществляется с помощью вспомогательной программы "git-server-alias foo" . Имя хоста удаленного сервера, передаваемое в потоке протокола, будет "foo" (это позволяет нескольким виртуальным серверам Git совместно использовать адрес канального уровня).

"ext::git-ssl foo.example /bar"

Представляет репозиторий, доступ к которому осуществляется с помощью вспомогательной программы "git-ssl foo.example /bar" . Тип запроса помощник может определить с помощью переменных среды (см. выше).
