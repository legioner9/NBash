git-ls-files

Name

git-ls-files — Показать информацию о файлах в индексе и рабочем дереве.

Synopsis

git ls-files [-z] [-t] [-v] [-f]
                [-c|--cached] [-d|--deleted] [-o|--others] [-i|--ignored]
                [-s|--stage] [-u|--unmerged] [-k|--killed] [-m|--modified]
                [--resolve-undo]
                [--directory [--no-empty-directory]] [--eol]
                [--deduplicate]
                [-x <pattern>|--exclude=<pattern>]
                [-X <file>|--exclude-from=<file>]
                [--exclude-per-directory=<file>]
                [--exclude-standard]
                [--error-unmatch] [--with-tree=<tree-ish>]
                [--full-name] [--recurse-submodules]
                [--abbrev[=<n>]] [--format=<format>] [--] [<file>…​]

Description

Эта команда объединяет список файлов в индексе с фактическим списком рабочих каталогов и показывает различные их комбинации.

Для определения того, какие файлы отображаются, можно использовать несколько флагов, и каждый файл можно распечатать несколько раз, если в индексе имеется несколько записей или если для соответствующих параметров выбора файлов применимо несколько статусов.

Options

-c
--cached

Показать все файлы, кэшированные в индексе Git, то есть все отслеживаемые файлы. (Это значение по умолчанию, если не указаны параметры -c/-s/-d/-o/-u/-k/-m/--resolve-undo.)

-d
--deleted

Показать файлы с неиндексированным удалением

-m
--modified

Показать файлы с неиндексированной модификацией (обратите внимание, что неиндексированное удаление также считается неиндексированной модификацией).

-o
--others

Показывать другие (т.е. неотслеживаемые) файлы в выводе

-i
--ignored

Показывать в выводе только игнорируемые файлы. Должен использоваться либо с явным -c , либо с -o . При отображении файлов в индексе (т. е. при использовании с -c ) печатайте только те файлы, которые соответствуют шаблону исключения. При отображении файлов "other" (т. е. при использовании с -o ) показывайте только те файлы, которые соответствуют шаблону исключения. Стандартные правила игнорирования не активируются автоматически; следовательно, требуется хотя бы одна из опций --exclude* .

-s
--stage

Показывать биты режима промежуточного содержимого, имя объекта и номер этапа в выходных данных.

--directory

Если весь каталог классифицируется как "other" , отображайте только его имя (с косой чертой в конце), а не все его содержимое. Не имеет эффекта без -o/--others.

--no-empty-directory

Не указывайте пустые каталоги. Не имеет никакого эффекта без --directory.

-u
--unmerged

Показывать информацию о несвязанных файлах в выходных данных, но не показывать другие отслеживаемые файлы (forces --stage, overrides --cached).

-k
--killed

Показывать неотслеживаемые файлы в файловой системе, которые необходимо удалить из-за конфликтов файлов/каталогов, чтобы отслеживаемые файлы можно было записать в файловую систему.

--resolve-undo

Показывать файлы, имеющие информацию о разрешении-отмене в индексе, вместе с информацией о разрешении-отмене. (информация разрешения-отмены используется для реализации "git checkout -m $PATH" , т.е. для воссоздания конфликтов слияния, которые были случайно разрешены)

-z

\0 завершение строки при выводе и не заключайте имена файлов в кавычки. См. ВЫВОД ниже для получения дополнительной информации.

--deduplicate

Когда отображаются только имена файлов, подавите дубликаты, которые могут возникнуть из-за наличия нескольких этапов во время слияния или одновременного использования параметров --deleted и --modified . Если используется любая из опций -t , --unmerged или --stage , эта опция не имеет никакого эффекта.

-x <pattern>
--exclude=<pattern>

Пропускать неотслеживаемые файлы, соответствующие шаблону. Обратите внимание, что шаблон представляет собой шаблон подстановочного знака оболочки. Дополнительную информацию см. в разделе ИСКЛЮЧЕНИЕ ШАБЛОНОВ ниже.

-X <file>
--exclude-from=<file>

Прочитать шаблоны исключения из <file>; 1 на строку.

--exclude-per-directory=<file>

Прочтите дополнительные шаблоны исключения, которые применяются только к каталогу и его подкаталогам в <file>. Устарело; вместо этого используйте --exclude-standard.

--exclude-standard

Добавьте стандартные исключения Git : .git/info/exclude, .gitignore в каждом каталоге и глобальный файл исключений пользователя.

--error-unmatch

Если какой-либо <файл> не отображается в индексе, рассматривайте это как ошибку (возврат 1).

--with-tree=<tree-ish>

При использовании --error-unmatch для расширения предоставленных пользователем аргументов <file> (т.е. шаблона пути) до путей, представьте, что пути, которые были удалены в индексе, поскольку именованный <tree-ish> все еще присутствует. Использование этой опции с опциями -s или -u не имеет никакого смысла.

-t

Показывать теги статуса вместе с именами файлов. Обратите внимание, что для целей сценариев git-status[1] , --porcelain и git-diff-files[1] , --name-status почти всегда являются лучшими альтернативами; пользователям следует обратить внимание на git-status[1] --short или git-diff[1] --name-status для более удобных альтернатив.

Эта опция указывает причину отображения каждого имени файла в виде тега состояния (за которым следует пробел, а затем имя файла). Теги состояния представляют собой одиночные символы из следующего списка:

H

отслеживаемый файл, который не является ни объединенным, ни пропущенным рабочим деревом

S

отслеживаемый файл —skip-worktree

M

отслеживаемый файл, который не объединен

R

отслеживаемый файл с неиндексированным удалением/удалением

C

отслеживаемый файл с неустановленной модификацией/изменением

K

неотслеживаемые пути, являющиеся частью конфликтов файлов и каталогов, которые не позволяют извлечь отслеживаемые файлы.

?

untracked file

U

файл с информацией о разрешении-отмене

-v

Аналогичен -t , но для файлов, помеченных как assume unchanged , используйте строчные буквы (см. git-update-index[1] ).

-f

Аналогичен -t , но для файлов, помеченных как fsmonitor valid , используйте строчные буквы (см. git-update-index[1] ).

--full-name

При запуске из подкаталога команда обычно выводит пути относительно текущего каталога. Эта опция заставляет выводить пути относительно верхнего каталога проекта.

--recurse-submodules

Рекурсивно вызывает ls-файлы для каждого активного подмодуля в репозитории. В настоящее время поддерживается только режимы --cached и --stage.

--abbrev[=<n>]

Вместо отображения полных 40-байтовых строк шестнадцатеричного объекта покажите самый короткий префикс длиной не менее шестнадцатеричных цифр <n> , который однозначно ссылается на объект. Нестандартное количество цифр можно указать с помощью --abbrev=<n>.

--debug

После каждой строки, описывающей файл, добавьте дополнительные данные о его записи в кэше. Это предназначено для отображения как можно большего количества информации для ручной проверки; точный формат может измениться в любое время.

--eol

Показать <eolinfo> и <eolattr> файлов. <eolinfo> — это идентификатор содержимого файла, используемый Git , когда атрибут "text" равен "auto" (или не установлен и core.autocrlf не имеет значения false). <eolinfo> — это "-text" , "none" , "lf" , "crlf" , "mixed" или «».

«» означает, что файл не является обычным файлом, его нет в индексе или он недоступен в рабочем дереве.

<eolattr> — это атрибут, который используется при извлечении или фиксации. Поддерживаются либо "", " -text ", " text ", " text=auto ", " text eol=lf ", " text eol=crlf ". Since Git 2.10 " text=auto eol=lf " and " text=auto eol=crlf».

Как <eolinfo> в индексе ( "i/<eolinfo>" ), так и в рабочем дереве ( "w/<eolinfo>" ) отображаются для обычных файлов, за которыми следует ( "attr/<eolattr>" ).

--sparse

Если индекс разреженный, покажите разреженные каталоги, не расширяя их до содержащихся в нем файлов. Разреженные каталоги обозначаются косой чертой в конце, например "x/" для разреженного каталога "x" .

--format=<format>

Строка, которая интерполирует %(fieldname) на основе отображаемого результата. Он также интерполирует %% в % и %xx , где xx — шестнадцатеричные цифры, интерполируемые в символ с шестнадцатеричным кодом xx ; например, %00 интерполирует до \0 (NUL), %09 до \t (TAB) и %0a до \n (LF). --format нельзя комбинировать с -s , -o , -k , -t , --resolve-undo и --eol .

--

Не интерпретируйте дополнительные аргументы как варианты.

<file>

Файлы для показа. Если файлы не указаны, отображаются все файлы, соответствующие другим указанным критериям.

Output

git ls-files просто выводит имена файлов, если не указано --stage ; в этом случае он выводит:

[<tag> ]<mode> <object> <stage> <file>

git ls-files --eol покажет i/<eolinfo><SPACES>w/<eolinfo><SPACES>attr/<eolattr><SPACE*><TAB><file>

git ls-files --unmerged и git ls-files --stage можно использовать для просмотра подробной информации о несвязанных путях.

Для несвязанного пути вместо записи одной пары режим/SHA-1 индекс записывает до трех таких пар; один из дерева O на этапе 1, A на этапе 2 и B на этапе 3. Эта информация может использоваться пользователем (или фарфором), чтобы увидеть, что в конечном итоге должно быть записано на пути. (дополнительную информацию о состоянии см. в git-read-tree[1] )

Без опции -z имена путей с символами "unusual" заключаются в кавычки, как описано для переменной конфигурации core.quotePath (см. git-config[1] ). При использовании -z имя файла выводится дословно, а строка завершается нулевым байтом.

Можно печатать в пользовательском формате, используя опцию --format , которая способна интерполировать различные поля с использованием нотации %(fieldname) . Например, если вас интересуют только поля "objectname" и "path" , вы можете выполнить с конкретным "--format" , например

git ls-files --format='%(objectname) %(path)'

Field names

Способ отображения каждого пути можно настроить с помощью параметра --format=<format> , где %(имя поля) в строке <format> для различных аспектов записи индекса интерполируется. Подразумеваются следующие "fieldname" :

objectmode

Режим файла, который записывается в индекс.

objecttype

Тип объекта файла, который записывается в индекс.

objectname

Имя файла, которое записывается в индекс.

objectsize[:padded]

Размер объекта файла, который записан в индексе ( "-" , если объект commit или tree ). Он также поддерживает расширенный формат размера "%(objectsize:padded)" .

stage

Стадия файла, которая записывается в индекс.

eolinfo:index
eolinfo:worktree

<eolinfo> (см. описание опции --eol ) содержимого индекса или рабочего дерева для пути.

eolattr

<eolattr> (см. описание опции --eol ), применяемый к пути.

path

Путь к файлу, который записан в индексе.

Exclude patterns

git ls-files может использовать список "exclude patterns" при обходе дерева каталогов и поиске файлов, чтобы показать, когда указаны флаги --others или --ignored. gitignore[5] определяет формат шаблонов исключения.

Как правило, вам следует просто использовать --exclude-standard, но по историческим причинам шаблоны исключения можно указать в следующих местах по порядку:

  1. Флаг командной строки --exclude=<шаблон> указывает один шаблон. Шаблоны располагаются в том же порядке, в котором они появляются в командной строке.

  2. Флаг командной строки --exclude-from=<file> указывает файл, содержащий список шаблонов. Узоры располагаются в том же порядке, в котором они появляются в файле.

  3. Флаг командной строки --exclude-per-directory=<name> указывает имя файла в каждом каталоге, который проверяет git ls-files , обычно .gitignore . Файлы в более глубоких каталогах имеют приоритет. Выкройки располагаются в том же порядке, в котором они появляются в файлах.

Шаблон, указанный в командной строке с помощью --exclude или прочитанный из файла, указанного с помощью --exclude-from, относится к вершине дерева каталогов. Шаблон, считанный из файла, указанного --exclude-per-directory, относится к каталогу, в котором находится файл шаблона.
