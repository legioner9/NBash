git-request-pull

Name

git-request-pull — генерирует сводку ожидающих изменений

Synopsis

git request-pull [-p] <start> <URL> [<end>]

Description

Создайте запрос, попросив ваш вышестоящий проект внести изменения в свое дерево. Запрос, выводимый на стандартный вывод, начинается с описания ветки, суммирует изменения и указывает, откуда их можно извлечь.

Ожидается, что вышестоящий проект будет иметь фиксацию с именем <start> , а выходные данные запрашивают интеграцию изменений, внесенных вами с момента этой фиксации, вплоть до фиксации с именем <end> , путем посещения репозитория с именем <URL> .

Options

-p

Включите текст патча в выходные данные.

<start>

Обязайтесь начать с. Это имя фиксации, которая уже находится в исходной истории.

<URL>

URL-адрес репозитория, из которого нужно извлечь.

<end>

Зафиксировать завершение (по умолчанию HEAD). Это имя фиксации в конце истории, которую вы просите извлечь.

Если репозиторий с именем <URL> имеет фиксацию на кончике ссылки, которая отличается от локальной ссылки, вы можете использовать синтаксис <local>:<remote> , чтобы указать его локальное имя, двоеточие : и его удаленное имя.

Examples

Представьте, что вы построили свою работу над веткой master на основе версии v1.0 и хотите, чтобы она была интегрирована в проект. Сначала вы отправляете это изменение в свой репозиторий public , чтобы другие могли его увидеть:

git push https://git.ko.xz/project master

Затем вы запускаете эту команду:

git request-pull v1.0 https://git.ko.xz/project master

который создаст запрос к вышестоящему серверу, суммирующий изменения между выпуском v1.0 и вашим master , чтобы извлечь его из вашего репозитория public .

Если вы отправили свои изменения в ветку, имя которой отличается от того, которое у вас есть локально, например

git push https://git.ko.xz/project master:for-linus

тогда вы можете попросить, чтобы его вытащили

git request-pull v1.0 https://git.ko.xz/project master:for-linus
