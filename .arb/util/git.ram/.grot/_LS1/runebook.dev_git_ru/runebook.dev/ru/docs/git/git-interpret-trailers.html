git-interpret-trailers

Name

git-interpret-trailers — Добавлять или анализировать структурированную информацию в сообщениях фиксации.

Synopsis

git interpret-trailers [--in-place] [--trim-empty]
                        [(--trailer (<key>|<keyAlias>)[(=|:)<value>])…​]
                        [--parse] [<file>…​]

Description

Добавьте или проанализируйте строки trailer , похожие на заголовки электронного письма RFC 822, в конце свободной части сообщения фиксации. Например, в следующем сообщении фиксации

subject

Lorem ipsum dolor sit amet, consectetur adipiscing elit.

Signed-off-by: Alice <alice@example.com>
Signed-off-by: Bob <bob@example.com>

последние две строки, начинающиеся с "Signed-off-by" , — это прицепы.

Эта команда считывает сообщения фиксации либо из аргументов <file>, либо из стандартного ввода, если <file> не указан. Если указан --parse , выходные данные состоят из проанализированных трейлеров, поступающих из входа, без влияния на них каких-либо параметров командной строки или переменных конфигурации.

В противном случае эта команда применяет переменные конфигурации trailer.* (которые потенциально могут добавлять новые трейлеры, а также перемещать их), а также любые аргументы командной строки, которые могут переопределить переменные конфигурации (например, --trailer=... , который также может добавлять новые трейлеры) к каждому входу. файл. Результат выводится на стандартный вывод.

Эта команда также может работать с выводом git-format-patch[1] , который более сложен, чем простое сообщение о фиксации. А именно, такой вывод включает сообщение фиксации (как указано выше), разделительную линию "---" и часть исправления. Для этих входов части делителя и патча не изменяются этой командой и выдаются как есть на выходе, если не указан --no-divider .

Некоторые переменные конфигурации контролируют способ применения аргументов --trailer к каждому входу и способ изменения любого существующего концевика во входе. Они также позволяют автоматически добавлять некоторые трейлеры.

По умолчанию аргумент <key>=<value> или <key>:<value> , заданный с помощью --trailer , будет добавлен после существующих трейлеров, только если последний трейлер имеет другую пару (<ключ>, <значение>) (или если существующий трейлер отсутствует). Части <key> и <value> будут обрезаны, чтобы удалить начальные и конечные пробелы, а полученные в результате обрезанные <key> и <value> появятся в выводе следующим образом:

key: value

Это означает, что обрезанные <ключ> и <значение> будут разделены символом ': ' (одно двоеточие, за которым следует один пробел).

Для удобства можно настроить <keyAlias>, чтобы сделать использование --trailer более коротким для ввода в командной строке. Это можно настроить с помощью переменной конфигурации trailer.<keyAlias>.key . <keyAlias> должен быть префиксом полной строки <key>, хотя регистр не имеет значения. Например, если у вас есть

trailer.sign.key "Signed-off-by: "

в вашей конфигурации вам нужно указать только --trailer="sign: foo" в командной строке вместо --trailer="Signed-off-by: foo" .

По умолчанию новый трейлер появится в конце всех существующих трейлеров. Если существующего трейлера нет, новый трейлер появится в конце ввода. Перед новым трейлером будет добавлена ​​пустая строка, если ее еще нет.

Существующие трейлеры извлекаются из входных данных путем поиска группы из одной или нескольких строк, которая (i) представляет собой все трейлеры или (ii) содержит хотя бы один трейлер, созданный Git или настроенный пользователем, и состоит как минимум из трейлеров на 25%. Группе должна предшествовать одна или несколько пустых строк (или строк, содержащих только пробелы). Группа должна находиться либо в конце ввода, либо быть последней строкой без пробелов перед строкой, начинающейся с --- (за которой следует пробел или конец строки).

При чтении трейлеров не должно быть пробелов до или внутри <key>, но между <key> и разделителем допускается любое количество обычных пробелов и символов табуляции. Пробелы могут быть до, внутри или после <value>. <value> может быть разбит на несколько строк, причем каждая последующая строка начинается хотя бы с одного пробела, как "folding" в RFC 822. Пример:

key: This is a very long value, with spaces and
  newlines in it.

Обратите внимание, что трейлеры не соответствуют (и не предназначены для этого) многим правилам для заголовков RFC 822. Например, они не следуют правилу кодирования.

Options

--in-place

Отредактируйте файлы на месте.

--trim-empty

Если часть <value> любого трейлера содержит только пробелы, весь трейлер будет удален из вывода. Это относится как к существующим прицепам, так и к новым прицепам.

--trailer <key>[(=|:)<value>]

Укажите пару (<ключ>, <значение>), которую следует применять в качестве концевика к входным данным. Смотрите описание этой команды.

--where <placement>
--no-where

Укажите, куда будут добавляться все новые трейлеры. Настройка, предоставляемая --where , переопределяет trailer.where и любые применимые переменные конфигурации trailer.<keyAlias>.where и применяется ко всем опциям --trailer до следующего появления --where или --no-where . При обнаружении --no-where удалите эффект любого предыдущего использования --where , чтобы соответствующие переменные конфигурации больше не переопределялись. Возможные варианты размещения: after , before , end или start .

--if-exists <action>
--no-if-exists

Укажите, какое действие будет выполнено, когда во входных данных уже есть хотя бы один трейлер с таким же <key>. Настройка, предоставляемая --if-exists , переопределяет trailer.ifExists и любые применимые переменные конфигурации trailer.<keyAlias>.ifExists и применяется ко всем опциям --trailer до следующего появления --if-exists или --no-if-exists . При обнаружении --no-if-exists удалите эффект любого предыдущего использования --if-exists, чтобы соответствующие переменные конфигурации больше не переопределялись. Возможные действия: addIfDifferent , addIfDifferentNeighbor , add , replace и doNothing .

--if-missing <action>
--no-if-missing

Укажите, какое действие будет выполняться, если во входных данных нет другого трейлера с таким же <key>. Настройка, предоставляемая --if-missing , переопределяет trailer.ifMissing и любые применимые переменные конфигурации trailer.<keyAlias>.ifMissing и применяется ко всем опциям --trailer до следующего появления --if-missing или --no-if-missing . При обнаружении '--no-if-missing удалите эффект любого предыдущего использования '--if-missing, чтобы соответствующие переменные конфигурации больше не переопределялись. Возможные действия: doNothing или add .

--only-trailers

Выводите только трейлеры, а не какие-либо другие части входных данных.

--only-input

Выводить только те трейлеры, которые существуют на входе; не добавляйте ничего из командной строки или путем применения переменных конфигурации trailer.* .

--unfold

Если в трейлере есть значение, занимающее несколько строк (также известное как "folded" ), переформатируйте это значение в одну строку.

--parse

Удобный псевдоним для --only-trailers --only-input --unfold . Это упрощает просмотр только трейлеров, поступающих из входных данных, не влияя на них с помощью каких-либо параметров командной строки или переменных конфигурации, а также делает выходные данные удобными для машины с помощью --unfold.

--no-divider

Не рассматривайте --- как конец сообщения фиксации. Используйте это, если вы знаете, что ваши входные данные содержат только само сообщение фиксации (а не электронное письмо или вывод git format-patch ).

Configuration variables

trailer.separators

Эта опция указывает, какие символы распознаются как разделители концевиков. По умолчанию только : распознается как разделитель концевика, за исключением того, что = всегда принимается в командной строке для совместимости с другими командами git .

Первый символ, указанный в этой опции, будет символом по умолчанию, используемым, если в конфигурации для этого трейлера не указан другой разделитель.

Например, если значение этой опции — "%=$" , то концевиками будут считаться только строки формата <key><sep><value> с <sep>, содержащими % , = или $ , а затем пробелы. А % будет использоваться в качестве разделителя по умолчанию, поэтому по умолчанию концевики будут выглядеть так: <key>% <value> (между ключом и значением появится один знак процента и один пробел).

trailer.where

Эта опция указывает, где будет добавлен новый трейлер.

Это может быть end , который используется по умолчанию, start , after или before .

Если это end , то каждый новый трейлер будет появляться в конце существующих трейлеров.

Если это start , то каждый новый трейлер будет появляться в начале, а не в конце существующих трейлеров.

Если это after , то каждый новый трейлер будет появляться сразу после последнего трейлера с тем же <ключом>.

Если это before , то каждый новый трейлер будет появляться непосредственно перед первым трейлером с тем же <ключом>.

trailer.ifexists

Эта опция позволяет выбрать, какое действие будет выполняться, когда во входных данных уже есть хотя бы один трейлер с таким же <key>.

Допустимые значения для этой опции: addIfDifferentNeighbor (значение по умолчанию), addIfDifferent , add , replace или doNothing .

При использовании addIfDifferentNeighbor новый трейлер будет добавлен только в том случае, если ни один трейлер с той же парой (<ключ>, <значение>) не находится выше или ниже линии, в которую будет добавлен новый трейлер.

При использовании addIfDifferent новый трейлер будет добавлен только в том случае, если во входных данных уже нет трейлера с такой же парой (<ключ>, <значение>).

С помощью add будет добавлен новый трейлер, даже если некоторые трейлеры с той же парой (<ключ>, <значение>) уже присутствуют во входных данных.

При использовании replace существующий трейлер с тем же <ключом> будет удален и добавлен новый трейлер. Удаленный трейлер будет ближайшим (с тем же <ключом>) к тому месту, где будет добавлен новый.

С doNothing ничего делаться не будет; то есть новый трейлер не будет добавлен, если во входных данных уже есть такой же <key>.

trailer.ifmissing

Эта опция позволяет выбрать, какое действие будет выполняться, если во входных данных еще нет трейлера с таким же <ключом>.

Допустимые значения для этой опции: add (значение по умолчанию) и doNothing .

С add будет добавлен новый трейлер.

С doNothing ничего сделать не получится.

trailer.<keyAlias>.key

Определяет <keyAlias> для <key>. <keyAlias> должен быть префиксом (регистр не имеет значения) <key>. Например, в git config trailer.ack.key "Acked-by" "Acked-by" — это <key>, а "ack" — это <keyAlias>. Эта конфигурация позволяет использовать более короткий вызов --trailer "ack:..." в командной строке с использованием "ack" <keyAlias> вместо более длинного --trailer "Acked-by:..." .

В конце <key> может стоять разделитель, а затем несколько пробелов. По умолчанию единственным допустимым разделителем является : , но его можно изменить с помощью переменной конфигурации trailer.separators .

Если в ключе есть разделитель, он переопределяет разделитель по умолчанию при добавлении трейлера.

trailer.<keyAlias>.where

Этот параметр принимает те же значения, что и переменная конфигурации trailer.where , и переопределяет то, что указано в этом параметре для прицепов с указанным <keyAlias>.

trailer.<keyAlias>.ifexists

Этот параметр принимает те же значения, что и переменная конфигурации trailer.ifexists , и переопределяет то, что указано в этом параметре для прицепов с указанным <keyAlias>.

trailer.<keyAlias>.ifmissing

Этот параметр принимает те же значения, что и переменная конфигурации trailer.ifmissing , и переопределяет то, что указано в этом параметре для прицепов с указанным <keyAlias>.

trailer.<keyAlias>.command

Устарело в пользу trailer.<keyAlias>.cmd . Эта опция ведет себя так же, как trailer.<keyAlias>.cmd , за исключением того, что она не передает ничего в качестве аргумента указанной команде. Вместо этого первое вхождение подстроки $ARG заменяется <значением>, которое будет передано в качестве аргумента.

Обратите внимание, что $ARG в пользовательской команде заменяется только один раз и что первоначальный способ замены $ARG небезопасен.

Если для одного и того же <keyAlias> указаны trailer.<keyAlias>.cmd и trailer.<keyAlias>.command , используется trailer.<keyAlias>.cmd , а trailer.<keyAlias>.command игнорируется.

trailer.<keyAlias>.cmd

Эту опцию можно использовать для указания команды оболочки, которая будет вызываться один раз для автоматического добавления трейлера с указанным <keyAlias>, а затем вызываться каждый раз, когда указывается аргумент --trailer <keyAlias>=<value> , чтобы изменить <value> трейлера, который будет использовать этот параметр. производить.

При первом вызове указанной команды для добавления трейлера с указанным <keyAlias> поведение такое же, как если бы в начале команды "git interpret-trailers" был добавлен специальный аргумент --trailer <keyAlias>=<value> , где <value> считается стандартным выводом команды "git interpret-trailers" . команда с обрезанными ведущими и конечными пробелами.

Если в командной строке также передаются некоторые аргументы --trailer <keyAlias>=<value> , команда вызывается еще раз для каждого из этих аргументов с тем же <keyAlias>. И часть <value> этих аргументов, если таковая имеется, будет передана команде в качестве ее первого аргумента. Таким образом, команда может создать <значение>, вычисленное на основе <значения>, переданного в аргументе --trailer <keyAlias>=<value> .

Examples

  • Настройте трейлер sign с ключом Signed-off-by , а затем добавьте два таких трейлера в файл сообщения о фиксации:

    $ git config trailer.sign.key "Signed-off-by"
$ cat msg.txt
subject

body text
$ git interpret-trailers --trailer 'sign: Alice <[email protected]>' --trailer 'sign: Bob <[email protected]>' <msg.txt
subject

body text

Signed-off-by: Alice <alice@example.com>
Signed-off-by: Bob <bob@example.com>

  • Используйте опцию --in-place , чтобы редактировать файл сообщения о фиксации на месте:

    $ cat msg.txt
subject

body text

Signed-off-by: Bob <bob@example.com>
$ git interpret-trailers --trailer 'Acked-by: Alice <[email protected]>' --in-place msg.txt
$ cat msg.txt
subject

body text

Signed-off-by: Bob <bob@example.com>
Acked-by: Alice <alice@example.com>

  • Извлеките последний коммит в виде патча и добавьте к нему Cc и трейлер Reviewed-by :

    $ git format-patch -1
0001-foo.patch
$ git interpret-trailers --trailer 'Cc: Alice <[email protected]>' --trailer 'Reviewed-by: Bob <[email protected]>' 0001-foo.patch >0001-bar.patch

  • Настройте трейлер sign с помощью команды для автоматического добавления «Подписано:» с информацией об авторе только в том случае, если «Подписано:» еще нет, и покажите, как это работает:

    $ cat msg1.txt
subject

body text
$ git config trailer.sign.key "Signed-off-by: "
$ git config trailer.sign.ifmissing add
$ git config trailer.sign.ifexists doNothing
$ git config trailer.sign.cmd 'echo "$(git config user.name) <$(git config user.email)>"'
$ git interpret-trailers --trailer sign <msg1.txt
subject

body text

Signed-off-by: Bob <bob@example.com>
$ cat msg2.txt
subject

body text

Signed-off-by: Alice <alice@example.com>
$ git interpret-trailers --trailer sign <msg2.txt
subject

body text

Signed-off-by: Alice <alice@example.com>

  • Настройте прицеп fix с ключом, содержащим # и без пробела после этого символа, и покажите, как это работает:

    $ git config trailer.separators ":#"
$ git config trailer.fix.key "Fix #"
$ echo "subject" | git interpret-trailers --trailer fix=42
subject

Fix #42

  • Настройте трейлер help с помощью cmd, используя сценарий glog-find-author , который ищет указанную личность автора из журнала git в репозитории git и показывает, как это работает:

    $ cat ~/bin/glog-find-author
#!/bin/sh
test -n "$1" && git log --author="$1" --pretty="%an <%ae>" -1 || true
$ cat msg.txt
subject

body text
$ git config trailer.help.key "Helped-by: "
$ git config trailer.help.ifExists "addIfDifferentNeighbor"
$ git config trailer.help.cmd "~/bin/glog-find-author"
$ git interpret-trailers --trailer="help:Junio" --trailer="help:Couder" <msg.txt
subject

body text

Helped-by: Junio C Hamano <gitster@pobox.com>
Helped-by: Christian Couder <christian.couder@gmail.com>

  • Настройте трейлер ref с помощью cmd, используя сценарий glog-grep , чтобы получить последний соответствующий коммит из журнала git в репозитории git и показать, как это работает:

    $ cat ~/bin/glog-grep
#!/bin/sh
test -n "$1" && git log --grep "$1" --pretty=reference -1 || true
$ cat msg.txt
subject

body text
$ git config trailer.ref.key "Reference-to: "
$ git config trailer.ref.ifExists "replace"
$ git config trailer.ref.cmd "~/bin/glog-grep"
$ git interpret-trailers --trailer="ref:Add copyright notices." <msg.txt
subject

body text

Reference-to: 8bc9a0c769 (Add copyright notices., 2005-04-07)

  • Настройте трейлер see с помощью команды, чтобы показать тему связанного коммита и показать, как он работает:

    $ cat msg.txt
subject

body text

see: HEAD~2
$ cat ~/bin/glog-ref
#!/bin/sh
git log -1 --oneline --format="%h (%s)" --abbrev-commit --abbrev=14
$ git config trailer.see.key "See-also: "
$ git config trailer.see.ifExists "replace"
$ git config trailer.see.ifMissing "doNothing"
$ git config trailer.see.cmd "glog-ref"
$ git interpret-trailers --trailer=see <msg.txt
subject

body text

See-also: fe3187489d69c4 (subject of related commit)

  • Настройте шаблон фиксации с некоторыми трейлерами с пустыми значениями (используя sed для отображения и сохранения конечных пробелов в конце трейлеров), затем настройте перехватчик commit-msg, который использует git interpret-trailers для удаления трейлеров с пустыми значениями и добавления трейлера git-version . :

    $ cat temp.txt
***subject***

***message***

Fixes: Z
Cc: Z
Reviewed-by: Z
Signed-off-by: Z
$ sed -e 's/ Z$/ /' temp.txt > commit_template.txt
$ git config commit.template commit_template.txt
$ cat .git/hooks/commit-msg
#!/bin/sh
git interpret-trailers --trim-empty --trailer "git-version: \$(git describe)" "\$1" > "\$1.new"
mv "\$1.new" "\$1"
$ chmod +x .git/hooks/commit-msg
