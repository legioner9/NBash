gitformat-signature

Name

gitformat-signature — форматы криптографической подписи Git

Synopsis

<[tag|commit] object header(s)>
<over-the-wire protocol>

Description

Git использует криптографические подписи в различных местах, в настоящее время в объектах (теги, фиксации, слияния тегов) и транзакциях (push). В каждом случае команда, которая собирается создать объект или транзакцию, определяет из него полезную нагрузку, вызывает внешнюю программу для получения отдельной подписи для полезной нагрузки ( gpg -bsa в случае подписей PGP) и встраивает подпись в объект. или транзакция.

Подписи начинаются со строки заголовка "ASCII Armor" и заканчиваются завершающей строкой, которая различается в зависимости от типа подписи (выбранного gpg.format , см. git-config[1] ). Это для значений gpg.format :

gpg (PGP)

-----BEGIN PGP SIGNATURE----- и -----END PGP SIGNATURE----- . Или, если gpg приказано создать подписи RFC1991, -----BEGIN PGP MESSAGE----- и -----END PGP MESSAGE----- .

ssh (SSH)

-----BEGIN SSH SIGNATURE----- и -----END SSH SIGNATURE-----

x509 (X.509)

-----BEGIN SIGNED MESSAGE----- и -----END SIGNED MESSAGE-----

Подписи иногда появляются как часть обычных полезных данных (например, к подписанному тегу блок подписи добавляется после полезных данных, к которым применяется подпись), а иногда появляются в значении заголовка объекта (например, фиксация слияния, которая объединила подписанный тег). будет иметь все содержимое тега в заголовке "mergetag" ). В последнем случае применяется обычное правило многострочного форматирования заголовков объектов. Т.е. вторая и последующие строки имеют префикс SP, сигнализирующий о том, что строка продолжается с предыдущей строки.

Это даже true для изначально пустой строки. В следующих примерах конец строки, оканчивающейся пробельной буквой, выделяется знаком $ ; если вы пытаетесь воссоздать эти примеры вручную, не вырезайте и не вставляйте их — они нужны в первую очередь для выделения лишних пробелов в конце некоторых строк.

Подписанные полезные данные и способ внедрения подписи зависят от типа объекта, соответственно. сделка.

Tag signatures

  • создатель: git tag -s

  • полезная нагрузка: объект аннотированного тега

  • встраивание: добавить подпись к неподписанному объекту тега

  • пример: тег signedtag с темой signed tag

object 04b871796dc0420f8e7561a895b52484b701d51a
type commit
tag signedtag

  • создатель: git commit -S

  • полезная нагрузка: объект фиксации

Mergetag signatures

  • Создатель: git merge на подписанном теге

  • полезная нагрузка/встраивание: весь объект подписанного тега встроен в объект фиксации (слияния) в виде записи заголовка mergetag .

  • пример: объединение подписанного тега signedtag , как указано выше.

tree c7b1cff039a93f3600a1d18b82d26688668c7dea
parent c33429be94b5f2d3ee9b0adad223f877f174b05d
parent 04b871796dc0420f8e7561a895b52484b701d51a
author A U Thor <author@example.com> 1465982009 +0000
committer C O Mitter <committer@example.com> 1465982009 +0000
mergetag object 04b871796dc0420f8e7561a895b52484b701d51a
 type commit
