git-show-branch

Name

git-show-branch — Показать ветки и их коммиты

Synopsis

git show-branch [-a | --all] [-r | --remotes] [--topo-order | --date-order]
                [--current] [--color[=<when>] | --no-color] [--sparse]
                [--more=<n> | --list | --independent | --merge-base]
                [--no-name | --sha1-name] [--topics]
                [(<rev> | <glob>)…​]
git show-branch (-g | --reflog)[=<n>[,<base>]] [--list] [<ref>]

Description

Полувизуально показывает график происхождения коммитов, начиная с коммитов, названных с помощью <rev>s или <glob>s (или всех ссылок под refs/heads и/или refs/tags).

Он не может отображать более 29 веток и коммитов одновременно.

Он использует многозначные элементы конфигурации showbranch.default , если в командной строке не указан <rev> или <glob>.

Options

<rev>

Произвольное расширенное выражение SHA-1 (см. gitrevisions[7] ), которое обычно называет заголовок ветки или тег.

<glob>

Шаблон glob, который соответствует именам ветвей или тегов в refs/. Например, если у вас много веток тем в разделе refs/heads/topic, то при указании topic/* будут показаны все из них.

-r
--remotes

Показать ветки удаленного отслеживания.

-a
--all

Покажите как ветки удаленного отслеживания, так и локальные ветки.

--current

При использовании этой опции команда включает текущую ветвь в список оборотов, который будет отображаться, если он не указан в командной строке.

--topo-order

По умолчанию ветки и их коммиты отображаются в обратном хронологическом порядке. Эта опция заставляет их отображаться в топологическом порядке (т. е. коммиты-потомки отображаются перед родительскими).

--date-order

Эта опция аналогична --topo-order в том смысле, что ни один родительский элемент не предшествует всем своим дочерним элементам, но в остальном фиксации упорядочиваются в соответствии с датой их фиксации.

--sparse

По умолчанию в выводе не учитываются слияния, доступные только из одного отображаемого наконечника. Эта опция делает их видимыми.

--more=<n>

Обычно команда останавливает вывод после отображения фиксации, которая является общим предком всех ветвей. Этот флаг сообщает команде, что необходимо выполнить <n> более частых коммитов за пределами этого значения. Если <n> отрицательное, отображаются только заданные <ref>, без отображения дерева предков фиксации.

--list

Синоним --more=-1

--merge-base

Вместо отображения списка коммитов определите возможные базы слияния для указанных коммитов. Все базы слияний будут содержаться во всех указанных коммитах. Это отличается от того, как git-merge-base[1] обрабатывает случай трех и более коммитов.

--independent

Среди заданных <ref> отображайте только те, к которым невозможно получить доступ из любого другого <ref>.

--no-name

Не показывать строки именования для каждого коммита.

--sha1-name

Вместо того, чтобы называть коммиты, используя путь к ним из голов (например, "master~2" означает «прародитель "master" »), назовите их с помощью уникального префикса имен их объектов.

--topics

Показывает только коммиты, которые НЕ находятся в первой заданной ветке. Это помогает отслеживать ветки тем, скрывая любые коммиты, которые уже находятся в основной линии разработки. Если задан "git show-branch --topics master topic1 topic2" , будут показаны версии, предоставленные "git rev-list ^master topic1 topic2" .

-g
--reflog[=<n>[,<base>]] [<ref>]

Показывает <n> последних записей журнала ссылок для данной ссылки. Если указан <base>, <n> записей, начиная с этой записи. <база> может быть указана как количество или дата. Если явный параметр <ref> не указан, по умолчанию используется текущая ветвь (или HEAD , если она отключена).

--color[=<when>]

Раскрасьте знак состояния (один из этих: * ! + - ) каждого коммита, соответствующего ветке, в которой он находится. Значение должно быть всегда (по умолчанию), никогда или автоматически.

--no-color

Отключите цветной вывод, даже если в файле конфигурации по умолчанию указан цветной вывод. То же, что --color=never .

Обратите внимание, что параметры --more, --list, --independent и --merge-base являются взаимоисключающими.

Output

Учитывая N <ref>, первые N строк представляют собой однострочное описание сообщения о фиксации. Заголовок ветки, на который указывает $GIT_DIR/HEAD, имеет префикс звездочки * , тогда как другие заголовки имеют префикс символа ! .

После этих N строк отображается однострочный журнал каждого коммита с отступом N мест. Если коммит находится на I-й ветке, то I-й символ отступа имеет знак + ; в противном случае отображается пробел. Коммиты слияния обозначаются знаком - . Каждый коммит показывает короткое имя, которое можно использовать в качестве расширенного SHA-1 для имени этого коммита.

В следующем примере показаны три ветви: "master" , "fixes" и "mhf" :

$ git show-branch master fixes mhf
* [master] Add 'git show-branch'.
 ! [fixes] Introduce "reset type" flag to "git reset"
  ! [mhf] Allow "+remote:local" refspec to cause --force when fetching.
---
  + [mhf] Allow "+remote:local" refspec to cause --force при извлечении.
  + [mhf~1] Use git-octopus when pulling more than one head.
 +  [fixes] Introduce "reset type" flag to "git reset"
  + [mhf~2] "git fetch --force".
  + [mhf~3] Use .git/remote/origin, not .git/branches/origin.
  + [mhf~4] Make "git pull" and "git fetch" default to origin
  + [mhf~5] Infamous 'octopus merge'
  + [mhf~6] Retire git-parse-remote.
  + [mhf~7] Multi-head fetch.
  + [mhf~8] Start adding the $GIT_DIR/remotes/ support.
*++ [master] Add 'git show-branch'.

Все эти три ветки произошли от общего коммита [master], сообщение о коммите которого — "Add 'git show-branch'" . В ветке "fixes" добавлен один коммит "Introduce " reset type " flag to " git reset "". The " mhf " branch adds many other commits. The current branch is " master".

Examples

Если вы храните свои основные ветки непосредственно в refs/heads , а тематические ветки в его подкаталогах, может помочь следующее в файле конфигурации:

[showbranch]
        default = --topo-order
        default = heads/*

При этом git show-branch без дополнительных параметров будет показывать только первичные ветки. Кроме того, если вы находитесь в своей тематической ветке, она также отображается.

$ git show-branch --reflog="10,1 hour ago" --list master

показывает 10 записей рефлога, начиная с подсказки по состоянию на 1 час назад. Без --list выходные данные также показывают, как эти советы топологически связаны друг с другом.

Configuration

Все, что находится ниже этой строки в этом разделе, выборочно включено из документации git-config[1] . Содержание такое же, как и найденное там:

showBranch.default

Набор ветвей по умолчанию для git-show-branch[1] . См. git-show-branch[1] .
