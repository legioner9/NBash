git-reflog

Name

git-reflog — Управление информацией журнала обновлений

Synopsis

git reflog [show] [<log-options>] [<ref>]
git reflog expire [--expire=<time>] [--expire-unreachable=<time>]
        [--rewrite] [--updateref] [--stale-fix]
        [--сухой пробег | -n] [--verbose] [--all [--single-worktree] | <ссылки>…​]
git reflog delete [--rewrite] [--updateref]
        [--сухой пробег | -n] [--verbose] <ref>@{<спецификатор>}…​
git reflog exists <ref>

Description

Эта команда управляет информацией, записанной в журналах обновлений.

Журналы ссылок, или "reflogs" , фиксируют, когда в локальном репозитории обновлялись подсказки ветвей и другие ссылки. Журналы обновлений полезны в различных командах Git для указания старого значения ссылки. Например, HEAD@{2} означает "where HEAD used to be two moves ago" , master@{one.week.ago} означает "where master used to point to one week ago in this local repository" и так далее. Дополнительную информацию см. в gitrevisions[7] .

Команда принимает различные подкоманды и разные параметры в зависимости от подкоманды:

Субкоманда "show" (которая также используется по умолчанию, при отсутствии каких-либо подкоманд) показывает журнал ссылки, указанной в командной строке (или HEAD по умолчанию). Журнал обновлений охватывает все недавние действия, а также журнал обновлений HEAD записывает переключение ветвей. git reflog show — это псевдоним git log -g --abbrev-commit --pretty=oneline ; дополнительную информацию см. в git-log[1] .

Подкоманда "expire" удаляет старые записи журнала обновлений. Записи старше времени expire или записи старше времени expire-unreachable и недоступные из текущего подсказки удаляются из журнала обновлений. Обычно это не используется конечными пользователями напрямую — вместо этого см. git-gc[1] .

Подкоманда "delete" удаляет отдельные записи из журнала ссылок. Его аргументом должна быть запись exact (например, " git reflog delete master@{2} " ). Эта подкоманда также обычно не используется конечными пользователями напрямую.

Субкоманда "exists" проверяет, есть ли у ссылки журнал повторов. Он завершает работу с нулевым статусом, если журнал ссылок существует, и с ненулевым статусом, если его нет.

Options

Опции для show

git reflog show принимает любой из вариантов, принятых git log .

Опции для expire

--all

Обработка рефлогов всех ссылок.

--single-worktree

По умолчанию, если указан --all , обрабатываются релоги всех рабочих деревьев. Эта опция ограничивает обработку только журналами текущего рабочего дерева.

--expire=<time>

Удалить записи старше указанного времени. Если этот параметр не указан, срок действия берется из параметра конфигурации gc.reflogExpire , который, в свою очередь, по умолчанию равен 90 дням. --expire=all удаляет записи независимо от их возраста; --expire=never отключает сокращение доступных записей (но см. --expire-unreachable ).

--expire-unreachable=<time>

Удалите записи старше <time> , которые недоступны из текущего конца ветки. Если этот параметр не указан, срок действия берется из параметра конфигурации gc.reflogExpireUnreachable , который, в свою очередь, по умолчанию равен 30 дням. --expire-unreachable=all удаляет недоступные записи независимо от их возраста; --expire-unreachable=never отключает раннее удаление недоступных записей (но см. --expire ).

--updateref

Обновите ссылку на значение верхней записи журнала ссылок (т. е. <ref>@{0}), если предыдущая верхняя запись была удалена. (Этот параметр игнорируется для символических ссылок.)

--rewrite

Если предшественница записи журнала обновлений удалена, настройте ее "old" SHA-1 так, чтобы она была равна полю "new" SHA-1 записи, которая теперь ей предшествует.

--stale-fix

Удалите все записи в журнале, указывающие на "broken commits" . Нарушенная фиксация — это фиксация, которую невозможно получить ни по одному из справочных советов и которая прямо или косвенно ссылается на отсутствующую фиксацию, дерево или объект BLOB-объекта.

Это вычисление включает в себя обход всех достижимых объектов, т.е. оно имеет ту же стоимость, что и git prune . В первую очередь он предназначен для исправления повреждений, вызванных сборкой мусора с использованием старых версий Git, которые не защищали объекты, на которые ссылаются журналы обновлений.

-n
--dry-run

На самом деле не удаляйте никакие записи; просто покажите то, что было бы подрезано.

--verbose

Распечатайте дополнительную информацию на экране.

Опции для delete

git reflog delete принимает опции --updateref , --rewrite , -n , --dry-run и --verbose с теми же значениями, что и при их использовании с expire .
