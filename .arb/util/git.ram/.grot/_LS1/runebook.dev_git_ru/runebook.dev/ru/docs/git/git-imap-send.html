git-imap-send

Name

git-imap-send — отправить набор патчей со стандартного ввода в папку IMAP.

Synopsis

git imap-send [-v] [-q] [--[no-]curl]

Description

Эта команда загружает почтовый ящик, созданный с помощью git format-patch , в папку черновиков IMAP. Это позволяет отправлять исправления так же, как и другую электронную почту, при использовании почтовых клиентов, которые не могут напрямую читать файлы почтового ящика. Команда также работает с любым общим почтовым ящиком, в котором электронные письма имеют поля "From" , "Date" и "Subject" в этом порядке.

Типичное использование выглядит примерно так:

git format-patch --signoff --stdout --attach origin | git imap-отправка

Options

-v
--verbose

Be verbose.

-q
--quiet

Be quiet.

--curl

Используйте libcurl для связи с сервером IMAP, если только вы не туннелируете к нему. Игнорируется, если Git был собран без установленной опции USE_CURL_FOR_IMAP_SEND.

--no-curl

Общайтесь с сервером IMAP, используя собственные процедуры IMAP git вместо использования libcurl. Игнорируется, если Git был собран с установленной опцией NO_OPENSSL.

Configuration

Чтобы использовать этот инструмент, для imap.folder и imap.tunnel или imap.host необходимо установить соответствующие значения.

Все, что находится выше этой строки в этом разделе, не включено в документацию git-config[1] . Следующее содержимое такое же, как и найденное там:

imap.folder

Папка, в которую помещаются письма, обычно это папка «Черновики». Например: "INBOX.Drafts" , "INBOX/Drafts" или "[Gmail]/Drafts" . Необходимый.

imap.tunnel

Команда, используемая для настройки туннеля к серверу IMAP, через который будут передаваться команды вместо использования прямого сетевого подключения к серверу. Требуется, если imap.host не установлен.

imap.host

URL-адрес, идентифицирующий сервер. Используйте префикс imap:// для незащищенных соединений и префикс imaps:// для безопасных соединений. Игнорируется, если установлен imap.tunnel, но требуется в противном случае.

imap.user

Имя пользователя, которое будет использоваться при входе на сервер.

imap.pass

Пароль, который будет использоваться при входе на сервер.

imap.port

Целочисленный номер порта для подключения к серверу. По умолчанию 143 для хостов imap:// и 993 для хостов imap://. Игнорируется, если установлен imap.tunnel.

imap.sslverify

boolean для включения/выключения проверки сертификата сервера, используемого соединением SSL/TLS. По умолчанию — true . Игнорируется, если установлен imap.tunnel.

imap.preformattedHTML

boolean для включения/отключения использования кодировки HTML при отправке патча. Патч в кодировке html будет заключен в скобки <pre> и будет иметь тип содержимого text/html. По иронии судьбы, включение этой опции приводит к тому, что Thunderbird отправляет патч в виде простого текстового сообщения с фиксированным форматом. По умолчанию — false .

imap.authMethod

Укажите метод аутентификации для аутентификации на сервере IMAP. Если Git был собран с параметром NO_CURL, или если ваша версия Curl старше 7.34.0, или если вы используете git-imap-send с параметром --no-curl , единственным поддерживаемым методом является CRAM-MD5 . Если этот параметр не установлен, git imap-send использует базовую команду LOGIN в виде открытого текста IMAP.

Examples

Использование туннельного режима:

[imap]
    folder = "INBOX.Drafts"
    tunnel = "ssh -q -C [email protected] /usr/bin/imapd ./Maildir 2> /dev/null"

Использование прямого режима:

[imap]
    folder = "INBOX.Drafts"
    host = imap://imap.example.com
    user = bob
    pass = p4ssw0rd

Использование прямого режима с SSL:

[imap]
    folder = "INBOX.Drafts"
    host = imaps://imap.example.com
    user = bob
    pass = p4ssw0rd
    port = 123
    ; sslVerify =  false
Note
 Вы можете использовать sslVerify=false при устранении неполадок, если вы подозреваете, что причина проблем с подключением заключается в том, что сертификат, который вы используете на сервере private example.com , который вы пытаетесь настроить (или уже настроили), может быть проверен неправильно.

Использование интерфейса IMAP Gmail:

[imap]
        folder = "[Gmail]/Drafts"
        host = imaps://imap.gmail.com
        user = [email protected]
        port = 993
Note
 Вместо этого вам может потребоваться использовать: folder = "[Google Mail]/Drafts" , если вы получите сообщение об ошибке "Folder doesn’t exist" .
Note
 Если в вашей учетной записи Gmail выбран язык, отличный от английского, имя папки "Drafts" будет локализовано.

Когда коммиты будут готовы к отправке, выполните следующую команду:

$ git format-patch --cover-letter -M --stdout origin/master | git imap-отправить

Просто обязательно отключите перенос строк в почтовом клиенте (веб-интерфейс Gmail будет переносить строки несмотря ни на что, поэтому вам нужно использовать настоящий IMAP-клиент).

Caution

Вы по-прежнему несете ответственность за то, чтобы сообщение электронной почты, отправленное вашей почтовой программой, соответствовало стандартам вашего проекта. Многие проекты не любят прикреплять патчи. Некоторые почтовые агенты преобразуют патчи (например, переносят строки и отправляют их как format=flowed) таким образом, что они не работают. Если вы не проверите это, вы получите гневное пламя, которое будет насмехаться над вами.

В частности, известно, что Thunderbird является проблематичным. Пользователи Thunderbird могут посетить эту веб-страницу для получения дополнительной информации: https://kb.mozillazine.org/Plain_text_e-mail_-_Thunderbird#Completely_plain_email .
