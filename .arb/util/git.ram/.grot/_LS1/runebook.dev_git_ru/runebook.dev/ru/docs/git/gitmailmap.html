gitmailmap

Name

gitmailmap — сопоставить имена авторов/коммитаторов и/или адреса электронной почты.

Synopsis

$GIT_WORK_TREE/.mailmap

Description

Если файл .mailmap существует на верхнем уровне репозитория или в месте, указанном параметрами конфигурации mailmap.file или mailmap.blob (см. git-config[1] ), он используется для сопоставления имен авторов и коммиттеров и адресов электронной почты с каноническими настоящими именами и адресами электронной почты.

Syntax

Символ # начинает комментарий до конца строки, пустые строки игнорируются.

В простой форме каждая строка файла состоит из канонического настоящего имени автора, пробелов и адреса электронной почты, используемого в коммите (заключенного в < и > ) для сопоставления с именем. Например:

Proper Name <commit@email.xx>

Более сложные формы:

<proper@email.xx> <commit@email.xx>

что позволяет mailmap заменять только часть коммита, посвященную электронной почте, и:

Proper Name <proper@email.xx> <commit@email.xx>

который позволяет mailmap заменять как имя, так и адрес электронной почты коммита, соответствующий указанному адресу электронной почты коммита, и:

Proper Name <proper@email.xx> Commit Name <commit@email.xx>

который позволяет mailmap заменять как имя, так и адрес электронной почты коммита, совпадающего как с указанным именем коммита, так и с адресом электронной почты.

И электронные письма, и имена сопоставляются без учета регистра. Например, это также будет соответствовать Commit Name <[email protected]> , указанному выше:

Proper Name <proper@email.xx> CoMmIt NaMe <CoMmIt@EmAiL.xX>

Notes

Git не переходит по символическим ссылкам при доступе к файлу .mailmap в рабочем дереве. Это обеспечивает единообразие поведения при доступе к файлу из индекса или дерева, а не из файловой системы.

Examples

Ваша история содержит коммиты двух авторов, Джейн и Джо, чьи имена появляются в репозитории в нескольких формах:

Joe Developer <joe@example.com>
Joe R. Developer <joe@example.com>
Jane Doe <jane@example.com>
Jane Doe <jane@laptop.(none)>
Jane D. <jane@desktop.(none)>

Теперь предположим, что Джо хочет, чтобы его второе имя использовалось в начале, а Джейн предпочитает, чтобы ее фамилия была написана полностью. Файл .mailmap для исправления имен будет выглядеть так:

Joe R. Developer <joe@example.com>
Jane Doe <jane@example.com>
Jane Doe <jane@desktop.(none)>

Обратите внимание, что нет необходимости сопоставлять имя <jane@laptop.(none)> , чтобы исправить только имена. Однако оставлять явно сломанные электронные письма <jane@laptop.(none)> и <jane@desktop.(none)> как есть обычно не то, что вам нужно. Файл .mailmap , который также исправляет это:

Joe R. Developer <joe@example.com>
Jane Doe <jane@example.com> <jane@laptop.(none)>
Jane Doe <jane@example.com> <jane@desktop.(none)>

Наконец, предположим, что у Джо и Джейн был общий адрес электронной почты, но не имя, например, из-за того, что эти два коммита были в истории, созданной системой отчетов об ошибках. Т.е. имена, фигурирующие в истории как:

Joe <bugs@example.com>
Jane <bugs@example.com>

Полный файл .mailmap , который также обрабатывает эти случаи (добавление двух строк к приведенному выше примеру), будет выглядеть так:

Joe R. Developer <joe@example.com>
Jane Doe <jane@example.com> <jane@laptop.(none)>
Jane Doe <jane@example.com> <jane@desktop.(none)>
Joe R. Developer <joe@example.com> Joe <bugs@example.com>
Jane Doe <jane@example.com> Jane <bugs@example.com>
