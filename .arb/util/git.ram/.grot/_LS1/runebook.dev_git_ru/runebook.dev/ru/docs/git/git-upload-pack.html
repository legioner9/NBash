git-upload-pack

Name

git-upload-pack — отправить упакованные объекты обратно в git-fetch-pack.

Synopsis

git-upload-pack [--[no-]strict] [--timeout=<n>] [--stateless-rpc]
                  [--advertise-refs] <directory>

Description

Вызывается git fetch-pack , изучает, каких объектов не хватает другой стороне, и отправляет их после упаковки.

Эта команда обычно не вызывается конечным пользователем напрямую. Пользовательский интерфейс протокола находится на стороне git fetch-pack , и эта пара программ предназначена для использования для получения обновлений из удаленного репозитория. Для операций push см. git send-pack .

Options

--[no-]strict

Не пытайтесь использовать <directory>/.git/, если <directory> не является каталогом Git .

--timeout=<n>

Прервать передачу после <n> секунд бездействия.

--stateless-rpc

Выполните только один цикл чтения-записи с помощью stdin и stdout. Это соответствует модели обработки запросов HTTP POST, где программа может прочитать запрос, записать ответ и должна выйти.

--http-backend-info-refs

Используется git-http-backend[1] для обслуживания запросов $GIT_URL/info/refs?service=git-upload-pack . См. "Smart Clients" в gitprotocol-http[5] и "HTTP Transport" в документации gitprotocol-v2[5] . Также понимается под git-receive-pack[1] .

<directory>

Репозиторий, из которого выполняется синхронизация.

Environment

GIT_PROTOCOL

Внутренняя переменная, используемая для подтверждения проводного протокола. Администраторам сервера может потребоваться настроить некоторые транспорты, чтобы разрешить передачу этой переменной. См. обсуждение в git[1] .
