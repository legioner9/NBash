git-archimport

Name

git-archimport — импортировать репозиторий GNU Arch в Git .

Synopsis

git archimport [-h] [-v] [-o] [-a] [-f] [-T] [-D <depth>] [-t <tempdir>]
               <archive>/<branch>[:<git-branch>]…​

Description

Импортирует проект из одного или нескольких репозиториев GNU Arch. Он будет следовать за ветвями и репозиториями в пространствах имен, определенных указанными параметрами <archive>/<branch>. Если он не может найти удаленную ветку, из которой происходит слияние, он просто импортирует его как обычный коммит. Если он сможет его найти, он пометит его как слияние, когда это возможно (см. обсуждение ниже).

Скрипт ожидает, что вы предоставите корни ключей, с которых он сможет начать импорт из коммита Arch типа initial import или tag . Он будет следовать и импортировать новые ветки в пределах предоставленных корней.

Ожидается, что он будет заниматься только одним проектом. Если он увидит ветки с разными корнями, он откажется бежать. В этом случае отредактируйте параметры <archive>/<branch>, чтобы четко определить объем импорта.

git archimport широко использует tla в фоновом режиме для доступа к репозиторию Arch. Убедитесь, что по пути доступна последняя версия tla . tla должен знать о репозиториях, которые вы передаете git archimport .

При первоначальном импорте git archimport ожидает оказаться в пустом каталоге. Чтобы следить за развитием проекта, использующего Arch, повторно запустите git archimport с теми же параметрами, что и при первоначальном импорте, для выполнения добавочного импорта.

Хотя git archimport пытается создать разумные имена ветвей для импортируемых архивов, также можно указать имена ветвей Git вручную. Для этого напишите имя ветки Git после каждого параметра <archive>/<branch>, разделив его двоеточием. Таким образом, вы можете сократить имена ветвей Arch и преобразовать жаргон Arch в жаргон Git , например, сопоставив ветку "PROJECT--devo--VERSION" с "master" .

Возможно связывание нескольких ветвей Arch с одной веткой Git ; результат будет иметь наибольший смысл только в том случае, если после создания второй ветки в первую ветку не будет внесено никаких коммитов. Тем не менее, это полезно для преобразования репозиториев Arch, которые периодически менялись.

Merges

Данные слияния патчей из Arch также используются для обозначения слияний в Git . Git не особо заботится об отслеживании исправлений и рассматривает слияние только тогда, когда ветка включает в себя все коммиты с момента их разветвления. Конечным результатом является то, что Git будет иметь хорошее представление о том, насколько далеко разошлись ветви. Таким образом, в процессе импорта некоторые метаданные обмена патчами теряются.

К счастью, когда вы попытаетесь объединить ветки, импортированные из Arch, Git найдет хорошую базу для слияния и имеет хорошие шансы идентифицировать патчи, которые были переданы вне последовательности между ветвями.

Options

-h

Display usage.

-v

Verbose output.

-T

Множество тегов. Создаст тег для каждого коммита, отражающий имя коммита в репозитории Arch.

-f

Используйте стратегию быстрого импорта набора обновлений. Это может быть значительно быстрее для больших деревьев, но не позволяет обрабатывать переименования каталогов или изменения разрешений. Стратегия по умолчанию медленная и безопасная.

-o

Используйте это для совместимости с именами ветвей старого стиля, использовавшимися в более ранних версиях git archimport . Имена ветвей в старом стиле выглядели как категория-ветвь, тогда как имена ветвей в новом стиле выглядели как архив, категория-ветвь-версия. В обоих случаях имена, указанные в командной строке, будут иметь приоритет над автоматически сгенерированными.

-D <depth>

Следуйте предкам слияния и попытайтесь импортировать деревья, которые были объединены. Укажите глубину больше 1, если журналы исправлений были удалены.

-a

Попытайтесь автоматически зарегистрировать архивы на http://mirrors.sourcecontrol.net . Это особенно полезно при использовании опции -D.

-t <tmpdir>

Переопределить временный каталог по умолчанию.

<archive>/<branch>

Идентификатор <архив>/<ветвь> в формате, понятном tla log .
