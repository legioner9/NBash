git-for-each-repo

Name

git-for-each-repo — запустить команду Git для списка репозиториев.

Synopsis

git for-each-repo --config=<config> [--] <arguments>

Description

Запустите команду Git для списка репозиториев. Аргументы после известных опций или индикатора -- используются в качестве аргументов подпроцесса Git .

ЭТА КОМАНДА ЭКСПЕРИМЕНТАЛЬНАЯ. ПОВЕДЕНИЕ МОЖЕТ ИЗМЕНИТЬСЯ.

Например, мы могли бы запустить обслуживание каждого из списка репозиториев, хранящегося в переменной конфигурации maintenance.repo , используя

git for-each-repo --config=maintenance.repo запуск обслуживания

Это запустит git -C <repo> maintenance run для каждого значения <repo> в многозначной переменной конфигурации maintenance.repo .

Options

--config=<config>

Используйте данную переменную конфигурации как многозначный список, хранящий имена абсолютных путей. Перебирайте этот список путей для запуска заданных аргументов.

Эти значения конфигурации загружаются из системной, глобальной и локальной конфигурации Git , если они доступны. Если git for-each-repo запускается в каталоге, который не является репозиторием Git , то используется только системная и глобальная конфигурация.

Subprocess behavior

Если какой-либо подпроцесс git -C <repo> <arguments> возвращает ненулевой код выхода, то процесс git for-each-repo возвращает этот код выхода без запуска дополнительных подпроцессов.

Каждый подпроцесс git -C <repo> <arguments> наследует стандартные файловые дескрипторы stdin , stdout и stderr .
