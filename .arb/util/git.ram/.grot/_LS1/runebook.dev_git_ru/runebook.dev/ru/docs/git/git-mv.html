git-mv

Name

git-mv — переместить или переименовать файл, каталог или символическую ссылку

Synopsis

git mv [<options>] <source>…<destination>

Description

Переместите или переименуйте файл, каталог или символическую ссылку.

git mv [-v] [-f] [-n] [-k] <source> <destination>
git mv [-v] [-f] [-n] [-k] <source> ... <destination directory>

В первой форме он переименовывает <source>, который должен существовать и быть файлом, символической ссылкой или каталогом, в <destination>. Во второй форме последним аргументом должен быть существующий каталог; данные источники будут перемещены в этот каталог.

Индекс обновляется после успешного завершения, но изменение все равно необходимо зафиксировать.

Options

-f
--force

Принудительное переименование или перемещение файла, даже если <пункт назначения> существует.

-k

Пропускайте действия по перемещению или переименованию, которые могут привести к возникновению ошибки. Ошибка возникает, когда источник не существует и не контролируется Git, или когда он перезаписывает существующий файл, если не указан -f .

-n
--dry-run

Ничего не делать; только покажи, что будет

-v
--verbose

Сообщайте имена файлов по мере их перемещения.

Submodules

Перемещение субмодуля с помощью gitfile (что означает, что они были клонированы с помощью Git версии 1.7.8 или новее) обновит настройки gitfile и core.worktree, чтобы субмодуль работал в новом месте. Он также попытается обновить настройку submodule.<name>.path в файле gitmodules[5] и проиндексировать этот файл (если не используется -n).

Bugs

Каждый раз, когда обновление суперпроекта перемещает заполненный подмодуль (например, при переключении между фиксациями до и после перемещения), устаревшая проверка подмодуля останется в старом месте, а в новом месте появится пустой каталог. Чтобы снова заполнить подмодуль в новом месте, пользователю впоследствии придется запустить "git submodule update" . Удаление старого каталога безопасно только в том случае, если он использует git-файл, иначе история подмодуля также будет удалена. Оба шага станут устаревшими, когда будет реализовано рекурсивное обновление подмодуля.
