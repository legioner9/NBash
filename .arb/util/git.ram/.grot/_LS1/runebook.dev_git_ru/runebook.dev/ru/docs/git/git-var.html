git-var

Name

git-var — показать логическую переменную Git

Synopsis

git var (-l | <variable>)

Description

Печатает логическую переменную Git . Выход с кодом 1, если переменная не имеет значения.

Options

-l

Отобразите логические переменные. Кроме того, также перечислены все переменные файла конфигурации Git .git/config. (Однако функция вывода списка переменных конфигурации устарела в пользу git config -l .)

Examples

$ git var GIT_AUTHOR_IDENT
Eric W. Biederman <ebiederm@lnxi.com> 1121223278 -0600

Variables

GIT_AUTHOR_IDENT

Автор фрагмента кода.

GIT_COMMITTER_IDENT

Человек, который поместил кусок кода в Git.

GIT_EDITOR

Текстовый редактор для использования командами Git . Значение предназначено для интерпретации оболочкой при его использовании. Примеры: ~/bin/vi , $SOME_ENVIRONMENT_VARIABLE , "C:\Program Files\Vim\gvim.exe" --nofork . Порядок предпочтения: переменная среды $GIT_EDITOR , затем конфигурация core.editor , затем $VISUAL , затем $EDITOR , а затем значение по умолчанию, выбранное во время компиляции, обычно это vi .

GIT_SEQUENCE_EDITOR

Текстовый редактор, используемый для редактирования файла todo во время работы git rebase -i . Как и GIT_EDITOR , значение предназначено для интерпретации оболочкой при его использовании. В порядке предпочтения — переменная среды $GIT_SEQUENCE_EDITOR , затем конфигурация sequence.editor , а затем значение git var GIT_EDITOR .

GIT_PAGER

Просмотрщик текста для использования командами Git (например, less ). Значение предназначено для интерпретации оболочкой. Порядок предпочтений: переменная среды $GIT_PAGER , затем конфигурация core.pager , затем $PAGER и затем значение по умолчанию, выбранное во время компиляции (обычно less ).

GIT_DEFAULT_BRANCH

Имя первой ветки, созданной во вновь инициализированных репозиториях.

GIT_SHELL_PATH

Путь к двоичному файлу, предоставляющему оболочку POSIX для команд, использующих оболочку.

GIT_ATTR_SYSTEM

Путь к системному файлу gitattributes[5] , если он включен.

GIT_ATTR_GLOBAL

Путь к глобальному (для каждого пользователя) файлу gitattributes[5] .

GIT_CONFIG_SYSTEM

Путь к файлу конфигурации системы, если он включен.

GIT_CONFIG_GLOBAL

Путь к глобальным файлам конфигурации (для каждого пользователя), если таковые имеются.

Большинство значений пути содержат только одно значение. Однако некоторые из них могут содержать несколько значений, которые разделены символами новой строки и перечислены в порядке от высшего к низшему приоритету. Вызывающие программы должны быть готовы к тому, что любое такое значение пути будет содержать несколько элементов.

Обратите внимание, что пути печатаются, даже если они не существуют, но не в том случае, если они отключены другими переменными среды.
