git-replace

Name

git-replace — создавать, перечислять, удалять ссылки для замены объектов

Synopsis

git replace [-f] <object> <replacement>
git replace [-f] --edit <object>
git replace [-f] --graft <commit> [<parent>…​]
git replace [-f] --convert-graft-file
git replace -d <object>…​
git replace [--format=<format>] [-l [<pattern>]]

Description

Добавляет ссылку replace в пространство имен refs/replace/ .

Имя ссылки replace — это SHA-1 заменяемого объекта. Содержимое ссылки replace — это SHA-1 замещающего объекта.

Заменяемый объект и замещающий объект должны быть одного типа. Это ограничение можно обойти с помощью -f .

Если не указан -f , ссылка replace еще не должна существовать.

Других ограничений на замещаемые и замещающие объекты нет. Коммиты слияния могут быть заменены коммитами без слияния и наоборот.

Ссылки на замену будут использоваться по умолчанию всеми командами Git , за исключением тех, которые выполняют обход достижимости (удаление, пакетная передача и fsck).

Можно отключить использование ссылок замены для любой команды, используя опцию --no-replace-objects сразу после git .

Например, если коммит foo был заменен на коммит bar :

$ git --no-replace-objects cat-file commit foo

показывает информацию о коммите foo , при этом:

$ git cat-file commit foo

показывает информацию о фиксации bar .

Переменную среды GIT_NO_REPLACE_OBJECTS можно настроить для достижения того же эффекта, что и параметр --no-replace-objects .

Options

-f
--force

Если существует существующая ссылка на замену для того же объекта, она будет перезаписана (вместо сбоя).

-d
--delete

Удалить существующие ссылки замены для данных объектов.

--edit <object>

Редактируйте содержимое объекта в интерактивном режиме. Существующее содержимое <object> печатается во временный файл, для файла запускается редактор, а результат анализируется для создания нового объекта того же типа, что и <object>. Затем создается ссылка на замену, чтобы заменить <object> вновь созданным объектом. См. git-var[1] для получения подробной информации о том, как будет выбран редактор.

--raw

При редактировании предоставляйте необработанное содержимое объекта, а не красиво напечатанное. В настоящее время это влияет только на деревья, которые будут отображаться в двоичной форме. С этим сложнее работать, но он может помочь при ремонте дерева, которое настолько повреждено, что его невозможно красиво напечатать. Обратите внимание, что вам может потребоваться настроить редактор для чистого чтения и записи двоичных данных.

--graft <фиксация> [<родитель>…​]

Создайте коммит трансплантации. Новый коммит создается с тем же содержимым, что и <commit>, за исключением того, что его родителями будут [<parent>…​] вместо родителей <commit>. Затем создается ссылка на замену, чтобы заменить <commit> вновь созданным коммитом. Используйте --convert-graft-file для преобразования файла $GIT_DIR/info/grafts и вместо этого используйте ссылки для замены.

--convert-graft-file

Создает трансплантационные фиксации для всех записей в $GIT_DIR/info/grafts и удаляет этот файл в случае успеха. Цель состоит в том, чтобы помочь пользователям отказаться от устаревшего файла трансплантации.

-l <pattern>
--list <pattern>

Список ссылок на замену для объектов, соответствующих заданному шаблону (или всех, если шаблон не указан). Ввод "git replace" без аргументов также выводит список всех ссылок на замену.

--format=<format>

При листинге используйте указанный <формат>, который может быть одним из short , medium и long . Если этот параметр опущен, по умолчанию используется формат short .

Formats

Доступны следующие форматы:

  • short : <заменено sha1>

  • medium : <замененный sha1> → <замененный sha1>

  • long : <замененный sha1> (<заменяемый тип>) → <замененный sha1> (<тип замены>)

Создание объектов замены

git-hash-object[1] , git-rebase[1] и git-filter-repo , среди других команд git , можно использовать для создания замещающих объектов из существующих объектов. Опцию --edit также можно использовать с git replace для создания объекта замены путем редактирования существующего объекта.

Если вы хотите заменить множество больших двоичных объектов, деревьев или коммитов, которые являются частью строки коммитов, вы можете просто создать замещающую строку коммитов, а затем заменить только коммит на кончике целевой строки коммитов на коммит в позиции кончик строки замены коммитов.

Bugs

Сравнение BLOB-объектов или деревьев, которые были заменены, с теми, которые их заменяют, не будет работать должным образом. А использование git reset --hard для возврата к замененному коммиту приведет к перемещению ветки к заменяющему коммиту вместо замененного коммита.

При использовании git rev-list могут возникнуть и другие проблемы, связанные с ожидающими объектами.
